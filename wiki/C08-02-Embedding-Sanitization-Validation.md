# C8.2: Embedding Sanitization & Validation

> [Back to C08 Index](C08-Memory-and-Embeddings.md)
> **Last Researched:** 2026-03-29

## Purpose

Pre-screen content before vectorization; treat memory writes as untrusted inputs; prevent ingestion of unsafe payloads. Embedding ingestion pipelines are a critical trust boundary — content that passes through them enters the system's long-term memory and influences all future retrieval. As of early 2026, PII detection and encoding validation are well-supported, but outlier detection and contradiction checking require custom implementations. The ACM AISec data loader study (74.4% ASR across 357 scenarios) showed that format-level attacks (font poisoning, homoglyphs) bypass all tested loaders, and A-MemGuard found even LLM-based classifiers miss 66% of poisoned entries. Inference-time defenses like SDAG (Feb 2026) and hybrid BM25+vector retrieval offer promising complementary approaches.

| **8.2.1** | **Verify that** regulated data and sensitive fields are detected prior to embedding and are masked, tokenized, transformed, or dropped based on policy. | 1 | PII/PHI/PCI data leakage via embeddings — once sensitive text is embedded, the original content can potentially be reconstructed via inversion attacks, and the vector store becomes a regulated data repository subject to GDPR/CCPA deletion requirements. Maps to OWASP LLM02:2025. OWASP LLM08:2025 formally recognizes that embedding inversion attacks can recover 50-70% of original input words if vectors are compromised. Vec2Text research reports reconstruction of up to 92% of 32-token inputs from T5-based embeddings. | Review the ingestion pipeline for DLP/PII scanning stages before the embedding step. Test with synthetic documents containing known PII patterns (SSNs, credit card numbers, email addresses). Verify sensitive fields are masked or dropped in both the stored text chunks and metadata. | Microsoft Presidio, AWS Comprehend, and Google DLP API can be integrated into ingestion pipelines. The challenge is that NER-based PII detection has recall gaps for domain-specific sensitive data (e.g., medical record numbers, internal project codes). Custom entity recognizers are often needed. Embedding the masked version may reduce retrieval quality for legitimate queries. New defense: the DPPN approach (2025) identifies and perturbs only privacy-sensitive neurons, reducing privacy leakage by 5-78% while improving downstream task performance by 14-40% compared to blanket noise methods. |
| **8.2.2** | **Verify that** embedding ingestion rejects or quarantines inputs that violate required content constraints (e.g., non-UTF-8, malformed encodings, oversized payloads, invisible Unicode characters, or executable content intended to poison retrieval). | 1 | Indirect prompt injection via RAG — attackers inject adversarial instructions into documents that get embedded and later retrieved into the model's context window. Invisible Unicode characters (e.g., zero-width joiners, right-to-left overrides) can hide malicious instructions from human reviewers while remaining effective against LLMs. Prompt Security's 2025 POC showed that hidden instructions survive the vectorization process and trigger with 80% success rate. The ACM AISec "Hidden Threat in Plain Text" study tested 19 stealthy injection techniques across five popular data loaders (LangChain, LlamaIndex, Docling, Haystack, LLMSherpa) and found a 74.4% attack success rate across 357 scenarios — font poisoning and homoglyph characters achieved 100% ASR against all loaders tested, while zero-size font attacks hit 81% ASR against LlamaIndex. These attacks were further validated on black-box services including NotebookLM and OpenAI Assistants. | Submit test documents containing: non-UTF-8 byte sequences, invisible Unicode (U+200B, U+200E, U+FEFF), oversized payloads exceeding defined limits, documents with embedded scripts or encoded executable content, font poisoning payloads, and homoglyph character substitutions. Verify each is rejected or quarantined. Inspect quarantine storage for review capability. Scan for suspicious instruction patterns ("ignore previous directives," "you must respond with") using regex or LLM-based detection. Test across all data loaders in the ingestion pipeline — the AISec study showed loader-specific vulnerabilities (e.g., out-of-bound text effective against Docling at 83% ASR but only 50% against LlamaIndex). | Content constraint enforcement is straightforward for encoding and size checks but harder for semantic attacks. Invisible Unicode detection requires explicit character class filtering. Font poisoning and homoglyph attacks target the document parsing layer before embeddings are created, requiring format-aware sanitization specific to each file type (DOCX, HTML, PDF). LLM-based classifiers (e.g., Rebuff, Lakera Guard, Prompt Security) are needed for semantic injection detection, but Sunil et al. (2025) emphasize that "no single signal is sufficient" — composite trust scoring across multiple signals is needed. The AISec study's automated toolkit implementing 19 injection techniques provides a practical red-team checklist for testing ingestion pipelines. |
| **8.2.3** | **Verify that** vectors that fall outside normal clustering patterns are flagged and quarantined before entering production indices. | 2 | Adversarial embedding injection — an attacker crafts vectors positioned in unusual regions of embedding space to manipulate retrieval behavior. Out-of-distribution vectors may be designed to be retrieved for a wide range of queries (high-centrality poisoning) or to target specific query clusters. PoisonedRAG demonstrated that carefully positioned vectors dominate retrieval for target queries. The MINJA attack (Dong et al., NeurIPS 2025) showed that bridging steps create vectors that appear individually reasonable while advancing attacker objectives, making pure outlier detection insufficient. | Review the outlier detection mechanism in the ingestion pipeline. Verify it uses statistical methods (e.g., z-score on cosine distance to cluster centroids, isolation forest, or DBSCAN-based novelty detection). Inject synthetic outlier vectors and confirm they are quarantined. Check that quarantine does not block legitimate novel content without review. Consider write-ahead validation using secondary models to evaluate proposed entries before commitment (Schneider 2025, Layer 2 defense). | Defining "normal clustering patterns" requires a baseline — cold-start scenarios or rapidly evolving knowledge bases make this challenging. False positive rates for outlier detection in high-dimensional spaces can be high. Dimensionality reduction (UMAP/t-SNE) before outlier detection is common but lossy. Consider adaptive baselines that update with verified-clean ingestion batches. MINJA's progressive shortening strategy means poisoned entries increasingly resemble legitimate content over iterative refinement, challenging static outlier thresholds. CorruptRAG (Jan 2026) showed a single optimized document can achieve 98.2% attack success, meaning outlier detection must catch individual vectors, not just clusters. Perplexity filtering (flagging statistically unnatural text) and cross-encoder re-ranking are emerging complementary defenses. |
| **8.2.4** | **Verify that** an agent's own outputs are not automatically written back into its trusted memory without explicit validation (such as content-origin checks or write-authorization controls that verify the content's source before committing writes). | 2 | Memory poisoning feedback loops — if an agent can write its own (potentially hallucinated or manipulated) outputs back into its memory, an attacker can use a single successful prompt injection to permanently corrupt the agent's knowledge base. This creates a persistent backdoor that survives conversation resets. OWASP ASI06 (Top 10 for Agentic Applications 2026) formally recognizes memory and context poisoning as a top agentic risk. The January 2026 "ZombieAgent" POC (Radware) demonstrated that ChatGPT's connector and memory features can be combined to make indirect prompt injection persistent and cross-session, spreading through email attachments. Unit 42 (October 2025) showed Amazon Bedrock Agents' session summarization could be hijacked to inject persistent instructions via forged XML tags. | Review agent architecture for write-back paths from agent output to memory stores. Verify that content-origin tagging distinguishes user-provided content, retrieved content, and agent-generated content. Test that agent-generated content requires explicit approval or a separate validation pipeline before memory write. Test for delayed tool invocation attacks (Rehberger, January 2025) where conditional instructions embedded in chat context trigger memory writes when users respond with natural affirmations. | This is critical for autonomous agents (e.g., AutoGPT-style systems) that have self-modification capabilities. LangChain's memory modules (ConversationBufferMemory, ConversationSummaryMemory) write back by default without origin checks. Custom guardrails are needed. The distinction between "agent output" and "validated knowledge" must be architecturally enforced. MINJA (NeurIPS 2025) demonstrated >95% injection success rate through query-only interaction without direct memory access, meaning even read-only agent interfaces can lead to memory poisoning. A-MemGuard research found that even advanced LLM-based memory classifiers miss 66% of poisoned entries, highlighting the difficulty of post-hoc detection. Mem0.ai published AI memory security best practices recommending write-ahead validation. |
| **8.2.5** | **Verify that** new content written to memory is checked for contradictions with what is already stored and that conflicts trigger alerts. | 3 | Knowledge base corruption and slow-burn poisoning — an attacker gradually introduces contradictory information to degrade the reliability of RAG responses. Without contradiction detection, conflicting facts coexist silently, and the model may retrieve either version unpredictably, undermining trust in the system. Microsoft's February 2026 report on "AI Recommendation Poisoning" documented over 50 distinct examples of this attack deployed by 31 real companies across 14 industries in a 60-day review, demonstrating slow-burn poisoning is actively exploited in the wild. Context manipulation attacks (as described by Zealynx 2025) gradually establish false assumptions that influence AI behavior over time, distinct from direct prompt injection. | Review contradiction detection implementation (e.g., NLI-based entailment checking, semantic similarity with polarity analysis). Test by ingesting documents that directly contradict existing stored content. Verify alerts fire and conflicting content is flagged for human review rather than silently merged. | Automated contradiction detection at scale is an unsolved problem in NLP. Current approaches using Natural Language Inference (NLI) models (e.g., DeBERTa fine-tuned on MNLI) work for explicit contradictions but miss subtle or contextual conflicts. Cross-document consistency checking is computationally expensive — O(n^2) pairwise comparisons in the worst case. Practical implementations focus on checking new content against a curated "golden" knowledge set. Memory poisoning attacks are temporally decoupled — poison planted today may execute weeks later when semantically triggered by unrelated interactions, making real-time contradiction detection the only viable prevention point. |
