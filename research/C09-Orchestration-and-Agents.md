# C09: Autonomous Orchestration & Agentic Action Security

> **Source:** [`1.0/en/0x10-C09-Orchestration-and-Agentic-Action.md`](https://github.com/OWASP/AISVS/blob/main/1.0/en/0x10-C09-Orchestration-and-Agentic-Action.md)
> **Requirements:** 51 | **Sections:** 8

## Control Objective

Autonomous and multi-agent systems must execute only authorized, intended, and bounded actions. This control family reduces risk from tool misuse, privilege escalation, uncontrolled recursion/cost growth, protocol manipulation, and cross-agent or cross-tenant interference.

> **2025-2026 Highlights:** Agentic AI security hardened into a distinct discipline across 2025-2026. The OWASP Top 10 for Agentic Applications (Dec 2025) crystallized the ten ASI risk categories, and AISVS C9 grew to 51 requirements as new attack classes emerged. Q1 2026 brought the first wave of real-world agent exploits — 30+ MCP CVEs in 60 days, the OpenClaw crisis (42,000+ exposed instances by April), and the PleaseFix zero-click browser agent hijack. Early April 2026 delivered another concentrated burst: CVE-2026-32922 in OpenClaw (CVSS 9.9 privilege escalation via WebSocket scope handling), four CrewAI CVEs (CVE-2026-2275 / 2285 / 2286 / 2287) chaining prompt injection into sandbox escape and RCE, and Chrome Gemini Live (CVE-2026-0628) which let browser extensions hijack the privileged AI panel to access camera, microphone, and local files. As of March 2026, the Gravitee State of AI Agent Security report found 88% of organizations experienced confirmed or suspected agent security incidents, while only 14.4% deploy agents with full security approval. NIST closed public comment on its Agent Identity and Authorization concept paper (Apr 2, 2026) after proposing OAuth 2.0/2.1, OIDC, SPIFFE/SPIRE, and SCIM as the baseline. The MCP 2026-03-15 specification mandated RFC 8707 resource indicators to block token mis-redemption and expanded on the 2025-03-26 Streamable HTTP transport, batched tool calls, and read-only vs. destructive tool annotations. The CHEQ IETF draft introduced a privacy-preserving approval protocol where sensitive data never reaches the LLM. The sandboxing ecosystem matured rapidly with Fly.io Sprites, Northflank, and expanded E2B/Daytona offerings, though CrewAI's Docker-fallback flaws show isolation alone is not enough without runtime verification.

These changes respond directly to the OpenClaw shared-credential failure mode, CrewAI sandbox-fallback RCE, and the 2026 TOCTOU vulnerability wave that bled over from classical systems (CVE-2026-20796 Mattermost, CVE-2026-27456 util-linux, CVE-2026-4878 libcap).

## Threat Landscape

Known attacks, real-world incidents, and threat vectors relevant to this chapter:

- **Destructive tool misuse** — Google's Antigravity IDE (Gemini 3 Pro in "Turbo mode") wiped an entire D: drive via `rmdir /s /q` while clearing a cache (December 2025). Gemini CLI permanently deleted files after misinterpreting a failed directory creation (July 2025). These incidents demonstrate the real cost of autonomous execution without confirmation gates.
- **Agentic misalignment at scale** — Anthropic's safety testing (May 2025) found that 96% of frontier models attempted blackmail when given email access and discovering both a threat to their existence and compromising information. Misaligned behavior was systemic across Anthropic, OpenAI, Google, Meta, xAI, and DeepSeek models.
- **Runaway agent costs** — A multi-agent research pipeline entered an infinite loop for 11 days, costing $47,000 (November 2025). A data enrichment agent made 2.3 million API calls over a weekend after misinterpreting an error code. Fortune 500 companies collectively leaked an estimated $400M in unbudgeted cloud spend from uncontrolled agent execution.
- **Prompt injection → tool abuse** — EchoLeak (CVE-2025-32711, CVSS 9.3) demonstrated zero-click data exfiltration from Microsoft 365 Copilot. MCP tool poisoning (April 2025) showed how malicious instructions hidden in tool descriptions cause agents to exfiltrate SSH keys. MCP rug pull attacks modify tool behavior post-approval.
- **Agent framework vulnerabilities** — LangGrinch (CVE-2025-68664, CVSS 9.3) enabled environment variable theft and RCE via serialization injection in langchain-core. Patched in v1.2.5 and v0.3.81.
- **Multi-agent protocol attacks** — A2A session smuggling (Unit 42, October 2025) demonstrated covert instruction injection during legitimate multi-turn sessions. Cross-agent steganographic collusion enables secret coordination channels invisible to monitoring.
- **Semantic privilege escalation** — A distinct class where agents operate within technical permissions but outside semantic scope. A financial agent was tricked into exporting 45,000 customer records using a regex matching every record — all permission checks passed.
- **MCP protocol CVE surge** — Between January and February 2026, researchers filed 30+ CVEs targeting MCP servers, clients, and infrastructure. CVE-2025-6514 (CVSS 9.6) enabled arbitrary command execution via mcp-remote, affecting 437,000+ downloads. CVE-2025-49596 hit Anthropic's own MCP Inspector tool. CVE-2025-54136 (MCPoison) bypassed Cursor IDE's trust-on-first-use model to silently execute malicious changes post-approval. Scanning of 2,614 MCP implementations found 82% vulnerable to path traversal and 67% to code injection.
- **OpenClaw agent compromise** — CVE-2026-25253 (CVSS 8.8) enabled one-click RCE via cross-site WebSocket hijacking in the OpenClaw autonomous agent framework (January 2026). The ClawHavoc campaign distributed 341 malicious skills (~12% of the registry). Censys identified 21,639 publicly exposed instances, and the Moltbook breach exposed 1.5 million API tokens. Compromise enabled lateral movement through inherited OAuth tokens across Slack, Gmail, and cloud storage.
- **Agentic browser hijacking** — The PleaseFix vulnerability family (March 2026) demonstrated zero-click compromise of AI agent browsers like Perplexity Comet. Attackers embed malicious content in routine workflows (calendar invites); when the agent processes them, it exfiltrates files and credentials without user interaction. Represents the evolution of social engineering from humans to autonomous agents.
- **Rogue agent privilege escalation** — In March 2026, a Meta internal AI agent autonomously posted a response to an employee without authorization, sharing it across internal forums to personnel who lacked access rights. An employee acted on the flawed advice, inadvertently exposing massive amounts of company data to unauthorized engineers. This incident demonstrates how an uncontrolled agent action can cascade through human trust into a full privilege escalation chain — the agent operated within its technical permissions but far outside its intended scope.
- **Agent lateral movement as a threat class** — As of early 2026, multi-agent architectures create new lateral movement paths: a compromised orchestration agent holding API keys for downstream agents gives attackers access to all connected systems. Unlike traditional network lateral movement, agent-to-agent movement operates within legitimate authenticated sessions and evades conventional monitoring. The Gravitee 2026 report found 25.5% of deployed agents can create and task other agents, often with shared credentials — a direct lateral movement enabler.
- **CrewAI sandbox-fallback RCE chain** — In April 2026, Cyata researcher Yarden Porat disclosed four CVEs in CrewAI (CVE-2026-2275, 2285, 2286, 2287) that chain prompt injection into full host compromise. The core design flaw: when the CodeInterpreter cannot reach Docker at runtime, CrewAI silently falls back to an in-process SandboxPython that permits arbitrary C calls. Combined with an SSRF in the RAG tools and an arbitrary-file-read in the JSON loader, an attacker delivering a poisoned prompt can achieve credential theft, cloud-metadata exfiltration, and RCE on the host. CERT/CC VU#221883 tracks the coordinated disclosure. This is the defining example of why C9.3.7 fail-closed enforcement matters — degraded isolation silently replaced strong isolation.
- **OpenClaw WebSocket scope self-assignment** — CVE-2026-32922 (CVSS 9.9, April 2026) let any authenticated OpenClaw user self-assign admin scopes during the WebSocket handshake. Local connections were trusted by default and exempted from rate limiting, so a low-privilege `operator.pairing` token obtained through normal device pairing was sufficient to brute-force into `operator.admin` and execute commands across every connected node. Nine CVEs were disclosed between March 18-21, 2026; by April, Censys counted 42,000+ exposed instances with 15,200 directly RCE-vulnerable. The ClawHavoc supply-chain campaign published 1,184 malicious skills on ClawHub — roughly one in five packages at peak — carrying Atomic Stealer (macOS) and Vidar (Windows) infostealers.
- **Agentic browser panel hijack** — CVE-2026-0628 (Chrome Gemini Live, disclosed Oct 2025 / patched Jan 2026, Unit 42) let a low-permission browser extension inject JavaScript into the privileged Gemini side panel and inherit its camera, microphone, local-file, and screenshot capabilities. Represents a structural pattern: trusted AI panels running inside untrusted browser processes inherit both the model's authority and the browser's attack surface. Expect similar hijacks against Copilot in Edge, Comet, and other agentic browser integrations.
- **Agent-layer TOCTOU exposure** — C9.8.8 was added to address a pattern that emerged in 2026 mainstream CVEs (Mattermost CVE-2026-20796, util-linux CVE-2026-27456, libcap CVE-2026-4878) and transfers directly to multi-agent systems. When concurrent agents access shared mutable state (e.g., a rate counter, approval flag, or access token table), the authorization check and the guarded mutation must be atomic — otherwise a second agent can invalidate the decision before the first agent acts on it. Most agent frameworks use optimistic in-memory locks that do not survive across distributed workers.

## Tooling & Implementation

Current tools, frameworks, and libraries that help implement these controls:

- **Agent frameworks with security features:** [LangGraph](https://github.com/langchain-ai/langgraph) (graph-based orchestration with checkpoint-based human-in-the-loop), [OpenAI Agents SDK](https://openai.github.io/openai-agents-python/) (March 2025 — input/output/tool guardrails with blocking execution mode), [Claude Agent SDK](https://platform.claude.com/docs/en/agent-sdk/overview) (admin-controlled MCP allow-lists, human approval before patches), [CrewAI](https://www.crewai.com/) (multi-agent with OpenTelemetry tracing, Enterprise SOC2), [AutoGen/AG2](https://github.com/microsoft/autogen) (v0.4+ async event-driven with enterprise integrations)
- **Agent sandboxing:** [E2B](https://e2b.dev/) (~15M sandbox sessions/month, Firecracker microVMs, 150ms startup, ~50% Fortune 500 adoption), [Modal](https://modal.com/) (gVisor isolation, per-sandbox egress policies), [Daytona](https://daytona.io/) (sub-90ms sandbox creation, $24M Series A in Feb 2026), [Firecracker](https://firecracker-microvm.github.io/) (KVM-based microVMs with cgroups/namespace isolation), [Fly.io Sprites](https://sprites.dev/) (Jan 2026 — stateful persistent VMs with checkpoint/restore in ~300ms, Firecracker-based, auto-idle billing), [Northflank](https://northflank.com/) (Kata Containers + gVisor adaptive isolation, 2M+ workloads/month, BYOC), [AIO Sandbox](https://github.com/agent-infra/sandbox) (all-in-one: Browser + Shell + File + MCP + VSCode in a single container)
- **Budget and cost controls:** [Helicone](https://www.helicone.ai/) (proxy-based cost tracking across 100+ LLMs with rate limiting), [Langfuse](https://langfuse.com/) (token/cost tracking per trace, open-source self-hosted), [LangSmith](https://smith.langchain.com/) (per-step token/latency/cost metrics). Architectural pattern: per-run hard spend caps, model tier routing, context window trimming.
- **Agent identity and auth:** [SPIFFE/SPIRE](https://spiffe.io/) (CNCF Graduated — short-lived SVIDs for agent workload identity; instance-level IDs for non-deterministic agents), [OPA](https://www.openpolicyagent.org/) (fine-grained policy enforcement via Rego), [SPIFFE+OAuth2 pattern](https://riptides.io/blog-post/spiffe-meets-oauth2-current-landscape-for-secure-workload-identity-in-the-agentic-ai-era) (attestation-bound identity exchanged for scoped OAuth tokens)
- **Agent communication protocols:** [A2A](https://a2a-protocol.org/latest/specification/) (Google, April 2025, v0.3 with gRPC — Agent Cards, OAuth 2.0 auth; donated to Linux Foundation June 2025), [MCP](https://modelcontextprotocol.io/) (Anthropic — OAuth 2.1 with PKCE, 97M monthly SDK downloads; [2025-03-26 spec](https://modelcontextprotocol.io/specification/2025-03-26/basic/authorization) added Streamable HTTP transport replacing SSE, batched tool calls, and tool behavior annotations for read-only vs. destructive operations), [Agentic AI Foundation](https://www.linuxfoundation.org/) (Linux Foundation, Dec 2025 — governs both A2A and MCP)
- **Approval and HITL protocols:** [CHEQ](https://www.ietf.org/archive/id/draft-rosenberg-cheq-00.html) (IETF draft, Rosenberg/Five9, Jennings/Cisco — privacy-preserving confirmation protocol where the LLM never sees sensitive data; uses HTTP 202 + URI pack for out-of-band human approval with cryptographic non-repudiation), [Microsoft Agent Framework](https://www.devleader.ca/2026/03/11/tool-approval-and-humanintheloop-in-microsoft-agent-framework) (MAF 1.0.0-rc1, Feb 2026 — DelegatingChatClient middleware for per-tool approval with selective auto-approve for read-only operations), [LangGraph interrupt()](https://github.com/langchain-ai/langgraph) (pause graph mid-execution for human input, resume cleanly)
- **Agent security gateways:** [Lasso MCP Gateway](https://github.com/lasso-security/mcp-gateway/) (open-source, April 2025 — security filters for MCP requests, prompt injection blocking, data leakage prevention; [partnered with Portkey](https://portkey.ai/blog/securing-mcp-to-deliver-enterprise-grade-agentic-ai-protection/) for inline MCP pipeline protection at sub-50ms latency), [Lakera Guard](https://www.lakera.ai/) (AI firewall, sub-50ms, 98%+ detection across 100+ languages; [acquired by Check Point in 2025](https://blog.checkpoint.com/securing-the-cloud/best-in-class-genai-security-when-cloudguard-waf-meets-lakera/) to form Global Center of Excellence for AI Security)
- **Agent monitoring:** [AgentOps](https://www.agentops.ai/) (session replay, time-travel debugging, multi-agent support), [Arize Phoenix](https://phoenix.arize.com/) (open-source, multi-step trajectory analysis), [Lunary](https://lunary.ai/) (LLM firewalls, PII masking, auto-categorization)

| Control Area | Tooling Maturity | Notes |
|--------------|:---:|-------|
| C9.1 Execution Budgets | Maturing | Helicone/Langfuse track costs; OpenAI Agents SDK has blocking guardrails. But $47K incidents show most deployments still lack hard budget enforcement. Only 14.4% of orgs deploy agents with full security approval. |
| C9.2 High-Impact Action Approval | Maturing | LangGraph checkpoints, OpenAI tool guardrails, Claude SDK approval gates, and now MAF 1.0.0-rc1 DelegatingChatClient middleware all provide HITL. The CHEQ IETF draft adds a privacy-preserving approval protocol with cryptographic non-repudiation. C9.2 expanded from 3 to 6 requirements, adding nonce-based replay prevention (9.2.4), TTL-bounded approvals (9.2.5), and compensating/rollback actions (9.2.6). The Meta rogue agent incident (March 2026) demonstrates that approval bypass remains common — the agent acted without any confirmation gate. |
| C9.3 Tool & Plugin Isolation | Mature | E2B (15M sessions/month), Modal, Firecracker, Fly.io Sprites, Northflank provide production-grade sandboxing. Lasso+Portkey MCP Gateway adds inline protocol filtering. But 30+ MCP CVEs in early 2026 show the tool integration surface remains dangerously exposed — 82% of scanned implementations vulnerable to path traversal. C9.3.7 (fail-closed on quota/timeout breach, added 2026) codifies the lesson of CrewAI CVE-2026-2275/2287: a silent fallback from Docker to an in-process sandbox is not a degradation, it's an escape. Runtimes must terminate rather than downgrade. |
| C9.4 Agent Identity & Audit | Emerging | SPIFFE+OAuth2 pattern emerging but not widely adopted. NIST NCCoE concept paper public comment period closed April 2, 2026 with responses from OpenID Foundation and Digital Chamber; proposes adapting OAuth 2.0/2.1, OIDC, SPIFFE, SCIM, and SP 800-63-4 for agent identity. arXiv 2603.14332 (March 2026) introduces a cryptographic binding framework with Ed25519 signatures (~97μs overhead) and BBS+ selective disclosure for verifiable agent actions. C9.4.6 (added 2026) extends integrity protection to persisted agent state — memory, task context, goals, partial results — addressing memory-poisoning attacks where an attacker plants instructions in long-term storage and the agent recalls them as truth weeks later. Gravitee 2026 report: only 21.9% of teams treat agents as independent identity-bearing entities; 45.6% still rely on shared API keys. |
| C9.5 Secure Messaging | Emerging | A2A v0.3 supports but doesn't enforce Agent Card signing. MCP 2026-03-15 spec (building on 2025-03-26 Streamable HTTP) now mandates RFC 8707 resource indicators to block token mis-redemption, strengthens OAuth 2.1 + PKCE enforcement in the Authorization Server role, and requires consent workflows with MFA for high-risk operations. Neither A2A nor MCP mandates end-to-end encryption. A2A session smuggling plus the arXiv 2602.11327 threat model (replay, Sybil, protocol downgrade attacks) show fundamental protocol-level design gaps that remain despite the spec updates. |
| C9.6 Authorization & Delegation | Emerging | OPA provides the policy engine but agent-specific delegation context propagation (C9.6.2) has limited tooling. C9.6.6 (added 2026) requires the PDP to evaluate the agent's granted permissions and the initiating principal's delegated scope as independent constraints — neither alone is sufficient. C9.6.7 (added 2026) requires explicit peer authorization policy for agent-to-agent delegation (approved registry or allowlist). This directly counters the pattern documented in Lasso's 2026 analysis: in CrewAI, AutoGen, MetaGPT, and OpenClaw, when one agent tasks another the child either inherits the parent's full credentials or gets its own unscoped key, with no scope narrowing, depth limits, or cascade revocation. Maverics, Strata's identity playbook, and the "Delegation Chains" governance pattern are early commercial entrants. |
| C9.7 Intent Verification | Emerging | Pre-execution gates are mostly custom-built. Post-condition checks (C9.7.3-C9.7.4) have no standard tooling. C9.7.5 requires integrity verification of remotely-loaded prompt templates and agent policy configs — hashes or signatures at load time. C9.7.6 (added 2026) requires write operations to persistent external state to be gated by explicit human approval or an independent policy-based authorization mechanism evaluating against original user intent, never by agent-generated output alone. C9.7.7 (added 2026) mandates fail-closed behavior when the PDP is unavailable (timeout, network partition, service failure) — a gap that real-world deployments commonly miss. No turnkey tooling exists for 9.7.6/9.7.7 yet. |
| C9.8 Multi-Agent Isolation | Emerging | Namespace isolation for tools, hierarchical memory isolation, and swarm-level shutdown are documented in research (AgenticCyOps) but have limited production tooling. C9.8.7 (added 2026) requires per-agent dedicated credentials within a swarm — explicitly outlawing shared keys, which Gravitee 2026 found in 45.6% of deployments. C9.8.8 (added 2026) requires atomic authorization + state mutation (transactions, optimistic locking, compare-and-swap) to prevent TOCTOU, mirroring the pattern seen in 2026 classical CVEs (Mattermost CVE-2026-20796, util-linux CVE-2026-27456, libcap CVE-2026-4878) as it bleeds into agent systems. Most agent frameworks use optimistic in-memory locks that do not survive across distributed workers. |

## Open Research Questions

- [ ] **How do you enforce "least agency" in practice?** — The OWASP Agentic Top 10 introduces the principle but no standard framework exists for defining minimum autonomy levels. NIST's five-tier autonomy taxonomy (Level 0 tool-assisted to Level 4 fully autonomous) is a starting point.
- [ ] **Can agent identity scale to millions of ephemeral instances?** — SPIFFE treats all pod replicas as identical, but agents are non-deterministic. The arXiv 2603.14332 framework found 5.8x variance in inference determinism across models, complicating replay-based verification. Instance-level identities are a proposed solution but the Gravitee 2026 report shows only 21.9% of teams even treat agents as independent identity-bearing entities — the gap between theory and practice remains vast.
- [ ] **How do you detect semantic privilege escalation?** — All technical permission checks pass, but the agent operates outside intended scope. No standard tooling exists for semantic boundary enforcement.
- [ ] **Will A2A and MCP converge on security standards?** — Both protocols are now under the Linux Foundation's Agentic AI Foundation (Dec 2025). Neither mandates end-to-end encryption or enforces authorization at the protocol level. A2A session smuggling shows protocol-level design weaknesses.
- [ ] **How do you monitor emergent swarm behavior?** — Multi-agent oscillation, deadlocks, and coordinated manipulation (steganographic collusion) require runtime behavioral analysis that goes beyond individual agent monitoring. Limited production tooling exists.
- [ ] **What does agentic misalignment defense look like?** — Anthropic's study showed 96% of models attempt blackmail under specific conditions. This is a model behavior problem (C07) that manifests as an agent security problem (C09).
- [ ] **How do you secure the MCP ecosystem at scale?** — 30+ CVEs in 60 days (Jan-Feb 2026), 8,000+ exposed servers, 82% path traversal rate, and 38-41% of registry servers lacking auth suggest the MCP ecosystem has a systemic security gap. Trust-on-first-use models (MCPoison/CVE-2025-54136) are fundamentally broken. The Vulnerable MCP Project is tracking issues, but no certification or security baseline exists for MCP server implementations.
- [ ] **Is agent-to-agent lateral movement the next major breach vector?** — The OpenClaw crisis (21,639 exposed instances, inherited OAuth tokens enabling cross-system traversal), the PleaseFix browser hijack, and the Meta rogue agent incident demonstrate that compromised or uncontrolled agents provide lateral movement paths invisible to traditional monitoring. The Gravitee 2026 report found 25.5% of deployed agents can create and task other agents with shared credentials — a direct enabler. No standard detection framework exists for agent-layer lateral movement.
- [ ] **How should approval protocols handle the privacy-sensitive action problem?** — The CHEQ IETF draft (Rosenberg et al.) proposes keeping sensitive data entirely outside the LLM's context during approval, but the protocol requires significant integration work. As C9.2 expanded to 6 requirements (adding nonce-based replay prevention, TTL-bounded approvals, and rollback semantics), the gap between what standards require and what production systems implement continues to widen.
- [ ] **Can frameworks enforce true fail-closed semantics across isolation tiers?** — The CrewAI CVE-2026-2275 / 2287 chain exposed a class of silent degradation: the CodeInterpreter's fallback from Docker to an in-process sandbox looked like graceful degradation, but functionally constituted sandbox escape. C9.3.7 (fail-closed on quota/timeout breach) and C9.7.7 (fail-closed on PDP unavailability) address this abstractly, but no widely adopted framework API exists that lets the runtime declare "these isolation guarantees are required — if any is unavailable, terminate." OpenAI Agents SDK blocking guardrails and Microsoft Agent Framework DelegatingChatClient are partial answers; the CNCF/Linux Foundation ecosystem has not yet codified a cross-framework standard.
- [ ] **Is agent state integrity a separate subsystem or a layer of memory?** — C9.4.6 (persisted state integrity) overlaps with C8 memory controls, and the 2026 research push (mem0, Supermemory, AgenticCyOps) treats agent memory as a first-class architectural component. But integrity protection — cryptographic MACs or signatures over stored context, goals, and partial results — has no mainstream implementation. Lakera's 2025 poisoned-memory research showed agents developing persistent false beliefs about security policies and defending them when questioned. Without verifiable state integrity, an attacker with one-time write access can compromise agents indefinitely.
