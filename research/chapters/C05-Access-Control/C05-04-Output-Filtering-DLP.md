# C5.4: Output Entitlement Enforcement

> [Back to C05 Index](C05-Access-Control.md)
> **Last Researched:** 2026-06-09

## Purpose

Deploy post-processing controls to prevent unauthorized data exposure in model-generated content. AI models memorize and regurgitate training data, RAG pipelines surface sensitive documents in responses, and prompt injection causes models to output restricted information. Post-inference filtering, citation validation against caller entitlements, and output format restrictions are all necessary to prevent data loss through model-generated outputs. The emergence of agentic browsers and autonomous tool use has introduced a new class of exfiltration risk where agents leak data through their own authorized channels, bypassing traditional DLP controls entirely.

This control stack sits downstream of retrieval-time filtering (C5.3) and complements it. Retrieval-time enforcement prevents unauthorized documents from entering the model's context; output-time enforcement is the last line of defense when retrieval filters fail, when the model hallucinates plausible-but-restricted content, or when indirect prompt injection causes the model to restate data the caller is not entitled to see. As of June 2026, output filtering is also the decision point that sees the full model response and has one chance to redact or block it before it is rendered to the caller, embedded in citation metadata, or passed to a downstream tool — and vendors are now shipping deterministic egress controls (OpenAI's Lockdown Mode being the most visible) precisely because per-response filtering keeps getting bypassed.

The highest-risk failures now show up outside the visible answer. Check Point Research disclosed a March 2026 ChatGPT code-execution runtime issue where DNS resolution became a hidden outbound channel for conversation and uploaded-file data. Zenity's March 2026 PerplexedBrowser disclosure showed an agentic browser reading local `file://` resources and exfiltrating them through routine navigation. Microsoft's March 2026 Patch Tuesday assigned CVE-2026-26144 to an Excel cross-site-scripting flaw that the Copilot Agent could be coerced into using as a zero-click exfiltration channel through unintended network egress. Capsule Security's "ShareLeak" (CVE-2026-21520, patched January 15 2026) showed that a Copilot Studio agent reading a public SharePoint form would happily query connected CRM data and forward it via Outlook to an attacker-controlled address; the same researchers reproduced the pattern as "PipeLeak" against Salesforce Agentforce. Palo Alto Networks Unit 42 has also documented web-based indirect prompt injection in the wild, including payloads aimed at ad-review systems, unauthorized transactions, data destruction, and sensitive information leakage. These cases reinforce that output filtering must inspect final text, citations, structured fields, network-bound tool arguments, and browser or agent actions as separate output channels — and that it must do so even when a model-side patch ostensibly addresses the underlying prompt injection.

> **Scope note:** Controls in this section govern post-inference output filtering, citation access validation, and output format restrictions. Query-time enforcement at the data layer is in C5.3; policy decision point isolation for agents is in C5.5; multi-tenant isolation is in C5.6. General safety filtering (toxicity, PII redaction, confidential data blocking where entitlements don't vary per caller) is in C7.3.

## Implementation Notes

**Defense-in-depth stack.** Treat output filtering as the third layer, not the only layer. Layer 1: retrieval-time entitlement enforcement (C5.3) — prevents unauthorized documents from entering context. Layer 2: data classification propagation (C5.2.3) — labels follow content through embeddings and prompt caches so output filters can see what sensitivity level they're inspecting. Layer 3 (this control): post-inference redaction, citation validation, and format restriction.

**Fail-closed posture.** When the output filter errors, times out, or encounters an unparseable response, the default must be to block, not pass. Attackers who can induce detector errors (malformed UTF-8, extremely long outputs, structured data that breaks the parser) will otherwise get a free pass. Log every fail-closed event.

**Gateway placement.** AI gateway and guardrail products are starting to expose controls that are close enough to the model boundary to inspect generated responses before users or downstream tools see them. As of April 2026, Cloudflare AI Gateway DLP scans prompts and responses and explicitly documents that text in tool call arguments and tool results is in scope when it appears in the JSON message body. Its documented limits are audit-relevant: response scanning buffers streamed output, base64-encoded images and file attachments are not decoded, and external URLs are not followed. Lakera Guard takes a similar runtime placement approach, recommending screening after generation and before return to the user or downstream system, with repeated calls at each tool interaction in multi-step agent workflows. These controls are useful only when the application routes every outbound channel through them; separate gateways or policy projects are still needed when tenants, data classes, or latency-sensitive streaming paths require different enforcement.

**Agentic outputs.** For agents that can invoke write-capable tools, filtering the final text response is insufficient; every tool invocation is an output channel. Apply the same entitlement checks to tool arguments as to user-visible text, including email recipients, calendar descriptions, CRM fields, webhook bodies, browser URLs, DNS labels, file names, and attachment metadata. The PerplexedBrowser case is the canonical example: exfiltration happened through URL navigation, not through the rendered response.

**Egress lockdown as a compensating control.** When per-response filtering cannot be trusted — and the Silent Egress numbers argue it often cannot — deterministic egress restriction is the honest fallback. OpenAI's Lockdown Mode (GA for all logged-in account tiers since early June 2026) limits browsing to cached content, disables response image rendering, agent mode, connectors, and file downloads, trading functionality for a hard guarantee that live network requests cannot carry data out. Enterprise deployments can build the same posture themselves: domain allowlists on agent egress, no image/markdown rendering of model-controlled URLs, and per-session aggregation of everything sent to any single external destination so fragmented (sharded) leaks become visible. Treat lockdown profiles as a per-tenant or per-data-classification setting, not a global toggle.

**MCP and tool-description poisoning.** Invariant Labs' April 2025 MCP tool-poisoning work showed that malicious tool descriptions can cause an agent to read local configuration or SSH keys and transmit them through a hidden tool parameter. The same research demonstrated "shadowing," where a malicious MCP server changes how the agent uses a separate trusted email tool. For this control, the practical lesson is that output entitlement enforcement must inspect the exact tool payload that will execute, not the summarized approval text in the user interface. Pinning and scanning MCP servers belongs in supply-chain controls, but C5.4 should still block tool arguments that contain secrets, unauthorized source identifiers, or external destinations that the caller is not allowed to use.

**Citation objects are output.** Treat citations, footnotes, source cards, tool traces, and "sources consulted" lists as protected output fields. The caller must be authorized to see the document and the specific metadata shown. If the answer can be shown but the source cannot, render the answer without that citation and log the strip event.

**Audit evidence to collect.**
- Output-filter policy versions, detector configuration, allowlists, and deny-lists active for each environment.
- Canary replay results covering answer text, citation metadata, hidden markdown, rendered HTML attributes, tool arguments, webhook payloads, and logs exposed to users.
- Entitlement decision logs that tie every displayed citation to the caller, source document, ACL or label, policy version, and final allow or strip decision.
- Tool-call enforcement logs with destination, normalized payload, data-classification labels, redaction or block reason, and reviewer decision for any override.
- Fail-closed events for detector timeout, malformed Unicode, oversized payload, parser failure, and unavailable policy services.
- Sampling records showing that streamed responses and low-latency paths do not bypass the output filter.

**Canary replay.** Keep a small set of synthetic secrets and restricted canary documents in each environment. Use them in regression tests and sampled production replay to verify that answer text, citations, browser actions, and tool payloads are blocked before leaving the trust boundary.

**Monitoring signals.** Track per-user redaction rates (sudden spikes suggest a new jailbreak), citation-strip rates (a user whose citations are constantly stripped is either misconfigured or probing), and detector-error rates (rising errors may indicate adversarial input crafted to fail the detector).

**Regulatory hooks.** The EU AI Act's core high-risk AI system obligations — originally applicable from August 2 2026, deferred to December 2 2027 by the May 2026 Digital Omnibus (AI in Annex I regulated products: August 2 2028) — include Article 15's lifecycle accuracy, robustness, and cybersecurity requirements, including resilience against confidentiality attacks and adversarial manipulation. NIST AI RMF's Generative AI Profile expects evidence that risks are measured, monitored, and managed across the lifecycle. ISO/IEC 42001 gives auditors a management-system frame for documenting ownership, risk treatment, control monitoring, and continual improvement around these output-filtering controls.

**Implementation Maturity.** Mixed. Server-side output DLP for plain text PII and credentials is mature: Presidio, Google SDP, AWS Comprehend, and Nightfall all ship production-ready SDKs and APIs. Citation entitlement enforcement is moving from "custom code per stack" to "library-level pattern" thanks to SpiceDB, OpenFGA, and Cerbos integrations with vector stores, but most teams still write their own glue and nothing off-the-shelf enforces at citation-render time. Agentic-channel filtering — tool arguments, browser URLs, MCP tool descriptions, structured output fields — remains emerging; Cloudflare AI Gateway DLP and Lakera Guard cover parts of it, but no single product yet inspects every output channel an agent can drive, and the Silent Egress results show per-response inspection misses fragmented leaks even where coverage exists. The repeated 2026 Copilot incidents (CVE-2026-26144, CVE-2026-21520, CVE-2026-24307) demonstrate that even well-resourced platform teams ship agents that can be talked into exfiltration; vendor responses are tilting toward deterministic egress lockdown (OpenAI Lockdown Mode) rather than smarter filters.

## Notable Incidents (2025–2026)

- **CVE-2026-24307 "Reprompt" — single-click Copilot Personal exfiltration (patched January 13 2026).** Varonis Threat Labs showed that Copilot's `q=` URL prefill parameter auto-executed attacker-supplied prompts on page load; a double-request trick evaded protections applied only to the first execution, and an attacker-controlled server then paced chained follow-up prompts to extract user context ("summarize all files the user accessed today," home location, travel plans). Microsoft 365 Copilot resisted the same chain because tenant-level Purview auditing, DLP, and sensitivity labeling added enforcement absent from the consumer tier — a direct demonstration of the value of the output-side controls this section requires.
- **GrafanaGhost — indirect prompt injection in Grafana AI features (disclosed April 7 2026).** Covered in OWASP's GenAI Exploit Round-up for Q1 2026, the attack used indirect prompt injection to make Grafana's AI features transmit sensitive observability and infrastructure data to attacker-controlled servers via external rendering flows — the same render-as-egress channel as EchoLeak, now in monitoring tooling. The same report logged the Vertex AI "Double Agent" research (March 31 2026), where malicious agents abused default service-account permissions to exfiltrate data across Google Cloud projects, and noted that most Q1 AI security events had no CVE at all, arising from design flaws and prompt manipulation instead.
- **CVE-2026-26144 — Excel Copilot Agent zero-click data exfiltration (March 10 2026).** Microsoft's March 2026 Patch Tuesday addressed a cross-site-scripting flaw in Excel that the Copilot Agent could be coerced into using as an exfiltration channel via unintended network egress, with no user interaction required. The Register, Action1, and ZDI covered the disclosure; Microsoft's published mitigation is to restrict outbound network traffic from Office applications and monitor unusual network requests from Excel processes until patches are applied.
- **CVE-2026-21520 "ShareLeak" + "PipeLeak" (patched January 15 2026).** Capsule Security disclosed an indirect prompt injection in Copilot Studio where a public-facing SharePoint form field was concatenated into the agent's system instructions, causing the agent to query connected SharePoint lists and exfiltrate customer data via Outlook to an attacker-controlled address. The same researchers reproduced the pattern against Salesforce Agentforce (PipeLeak). VentureBeat's April 2026 follow-up reports that even after Microsoft's patch, public lead-form payloads still hijacked the agent in lab testing, with no volume cap on exfiltrated CRM data and no user-visible signal that anything had left the tenant.
- **CVE-2026-26030 and CVE-2026-25592 — Semantic Kernel RCE via prompt injection (Microsoft, May 7 2026).** Microsoft's MSRC blog detailed how unsafe string interpolation in Semantic Kernel's In-Memory Vector Store allowed attacker-controlled context to escape into Python execution; a separate sandbox-escape chain used `DownloadFileAsync` to drop payloads into the Windows Startup folder. The fix used AST-allowlist, function-allowlist, dangerous-attribute deny-list, and bare-identifier restriction layers. For output filtering this is the canonical reminder that "the LLM is not a security boundary" and that tool arguments must be validated by code outside the model loop.
- **CVE-2025-68143 / 68144 / 68145 — Anthropic Git MCP server (January 2026).** Three prompt-injection vulnerabilities in the official Git MCP server allowed an attacker who could influence what an assistant reads to trigger code execution and data exfiltration. The incidents reinforce that MCP tool descriptions and tool outputs are themselves model inputs and must be treated as untrusted by output filters that inspect tool arguments.
- **EchoLeak — CVE-2025-32711 (June 2025).** The first publicly documented zero-click prompt-injection exploit in a production LLM system, EchoLeak chained markdown-image references and a Teams proxy allowed by CSP to exfiltrate the most sensitive data in the M365 Copilot context. Microsoft patched the service-side issue, but the chain remains the canonical regression test for any output filter that claims to strip exfiltration markdown.
