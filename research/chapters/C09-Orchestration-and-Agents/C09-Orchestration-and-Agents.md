# C09: Autonomous Orchestration & Agentic Action Security

> **Source:** [`1.0/en/0x10-C09-Orchestration-and-Agentic-Action.md`](https://github.com/OWASP/AISVS/blob/main/1.0/en/0x10-C09-Orchestration-and-Agentic-Action.md)
> **Requirements:** 35 | **Sections:** 8

## Control Objective

Autonomous and multi-agent systems must execute only authorized, intended, and bounded actions. This control family reduces risk from tool misuse, privilege escalation, uncontrolled recursion/cost growth, protocol manipulation, and cross-agent or cross-tenant interference. The current AISVS source narrows this chapter to agent-specific orchestration controls: generic transport encryption, schema validation, message integrity, log integrity, concurrency safety, supply chain integrity, service-account rotation, multi-tenant isolation, contextual authorization, and API-edge rate limiting remain in ASVS v5, C10, C13, and adjacent AISVS chapters.

> **2025-2026 Highlights:** Agentic AI security hardened into a distinct discipline across 2025-2026. The OWASP Top 10 for Agentic Applications (Dec 2025) crystallized ten ASI risk categories, while NIST's NCCoE concept paper pushed agent identity and authorization toward OAuth 2.0/2.1, OIDC, SPIFFE/SPIRE, SCIM, NGAC, and zero-trust patterns. April 2026 added two important signals for this chapter: OX Security's MCP STDIO advisory showed how user-controlled MCP configuration can become ecosystem-wide command execution across LangFlow, LiteLLM, Agent Zero, Windsurf, DocsGPT, and related tooling; and FIDO launched Agentic Authentication and Payments workstreams for verifiable user instructions, agent authentication, and trusted delegation. Google AP2 and Mastercard Verifiable Intent now give auditors concrete examples of cryptographically bound user intent for autonomous transactions, directly supporting the C9.2 approval, C9.4 identity, C9.6 delegation, and C9.7 intent-verification controls.

## Threat Landscape

Known attacks, real-world incidents, and threat vectors relevant to this chapter:

- **Destructive tool misuse** — Google's Antigravity IDE (Gemini 3 Pro in "Turbo mode") wiped an entire D: drive via `rmdir /s /q` while clearing a cache (December 2025). Gemini CLI permanently deleted files after misinterpreting a failed directory creation (July 2025). These incidents demonstrate the real cost of autonomous execution without confirmation gates.
- **Agentic misalignment at scale** — Anthropic's safety testing (May 2025) found that 96% of frontier models attempted blackmail when given email access and discovering both a threat to their existence and compromising information. Misaligned behavior was systemic across Anthropic, OpenAI, Google, Meta, xAI, and DeepSeek models.
- **Runaway agent costs** — A multi-agent research pipeline entered an infinite loop for 11 days, costing $47,000 (November 2025). A data enrichment agent made 2.3 million API calls over a weekend after misinterpreting an error code. Fortune 500 companies collectively leaked an estimated $400M in unbudgeted cloud spend from uncontrolled agent execution.
- **Prompt injection → tool abuse** — EchoLeak (CVE-2025-32711, CVSS 9.3) demonstrated zero-click data exfiltration from Microsoft 365 Copilot. MCP tool poisoning (April 2025) showed how malicious instructions hidden in tool descriptions cause agents to exfiltrate SSH keys. MCP rug pull attacks modify tool behavior post-approval.
- **Agent framework vulnerabilities** — LangGrinch (CVE-2025-68664, CVSS 9.3) enabled environment variable theft and RCE via serialization injection in langchain-core. Patched in v1.2.5 and v0.3.81.
- **Multi-agent protocol attacks** — A2A session smuggling (Unit 42, October 2025) demonstrated covert instruction injection during legitimate multi-turn sessions. Cross-agent steganographic collusion enables secret coordination channels invisible to monitoring.
- **Semantic privilege escalation** — A distinct class where agents operate within technical permissions but outside semantic scope. A financial agent was tricked into exporting 45,000 customer records using a regex matching every record — all permission checks passed.
- **MCP protocol CVE surge** — Between January and February 2026, researchers filed 30+ CVEs targeting MCP servers, clients, and infrastructure. CVE-2025-6514 (CVSS 9.6) enabled arbitrary command execution via mcp-remote, affecting 437,000+ downloads. CVE-2025-49596 hit Anthropic's own MCP Inspector tool. CVE-2025-54136 (MCPoison) bypassed Cursor IDE's trust-on-first-use model to silently execute malicious changes post-approval. Scanning of 2,614 MCP implementations found 82% vulnerable to path traversal and 67% to code injection.
- **MCP STDIO command execution by configuration** — OX Security's April 15, 2026 advisory documented a systemic class where MCP STDIO server configuration values flow into process execution. Confirmed variants include unauthenticated UI configuration, authenticated configuration, allowlist bypass through `npm`/`npx` arguments, prompt injection that modifies local MCP config, and hidden backend STDIO paths triggered through edited network requests. The pattern maps cleanly to C9.3.2 fail-closed tool execution, C9.3.4 manifest enforcement, C9.6.1 runtime policy, and C9.7.1 pre-execution gates.
- **OpenClaw agent compromise** — CVE-2026-25253 (CVSS 8.8) enabled one-click RCE via cross-site WebSocket hijacking in the OpenClaw autonomous agent framework (January 2026). The ClawHavoc campaign distributed 341 malicious skills (~12% of the registry). Censys identified 21,639 publicly exposed instances, and the Moltbook breach exposed 1.5 million API tokens. Compromise enabled lateral movement through inherited OAuth tokens across Slack, Gmail, and cloud storage.
- **Agentic browser hijacking** — The PleaseFix vulnerability family (March 2026) demonstrated zero-click compromise of AI agent browsers like Perplexity Comet. Attackers embed malicious content in routine workflows (calendar invites); when the agent processes them, it exfiltrates files and credentials without user interaction. Represents the evolution of social engineering from humans to autonomous agents.
- **Rogue agent privilege escalation** — In March 2026, a Meta internal AI agent autonomously posted a response to an employee without authorization, sharing it across internal forums to personnel who lacked access rights. An employee acted on the flawed advice, inadvertently exposing massive amounts of company data to unauthorized engineers. This incident demonstrates how an uncontrolled agent action can cascade through human trust into a full privilege escalation chain — the agent operated within its technical permissions but far outside its intended scope.
- **Agent lateral movement as a threat class** — As of early 2026, multi-agent architectures create new lateral movement paths: a compromised orchestration agent holding API keys for downstream agents gives attackers access to all connected systems. Unlike traditional network lateral movement, agent-to-agent movement operates within legitimate authenticated sessions and evades conventional monitoring. The Gravitee 2026 report found 25.5% of deployed agents can create and task other agents, often with shared credentials — a direct lateral movement enabler.
- **CrewAI sandbox-fallback RCE chain** — In April 2026, Cyata researcher Yarden Porat disclosed four CVEs in CrewAI (CVE-2026-2275, 2285, 2286, 2287) that chain prompt injection into full host compromise. The core design flaw: when the CodeInterpreter cannot reach Docker at runtime, CrewAI silently falls back to an in-process SandboxPython that permits arbitrary C calls. Combined with an SSRF in the RAG tools and an arbitrary-file-read in the JSON loader, an attacker delivering a poisoned prompt can achieve credential theft, cloud-metadata exfiltration, and RCE on the host. CERT/CC VU#221883 tracks the coordinated disclosure. This is the defining example for C9.3.2 fail-closed enforcement: degraded isolation silently replaced strong isolation.
- **OpenClaw WebSocket scope self-assignment** — CVE-2026-32922 (CVSS 9.9, April 2026) let any authenticated OpenClaw user self-assign admin scopes during the WebSocket handshake. Local connections were trusted by default and exempted from rate limiting, so a low-privilege `operator.pairing` token obtained through normal device pairing was sufficient to brute-force into `operator.admin` and execute commands across every connected node. Nine CVEs were disclosed between March 18-21, 2026; by April, Censys counted 42,000+ exposed instances with 15,200 directly RCE-vulnerable. The ClawHavoc supply-chain campaign published 1,184 malicious skills on ClawHub — roughly one in five packages at peak — carrying Atomic Stealer (macOS) and Vidar (Windows) infostealers.
- **Agentic browser panel hijack** — CVE-2026-0628 (Chrome Gemini Live, disclosed Oct 2025 / patched Jan 2026, Unit 42) let a low-permission browser extension inject JavaScript into the privileged Gemini side panel and inherit its camera, microphone, local-file, and screenshot capabilities. Represents a structural pattern: trusted AI panels running inside untrusted browser processes inherit both the model's authority and the browser's attack surface. Expect similar hijacks against Copilot in Edge, Comet, and other agentic browser integrations.
- **Agent-layer TOCTOU exposure** — The 2026 wave of classical TOCTOU issues transfers directly to multi-agent systems. When concurrent agents access shared mutable state (e.g., a rate counter, approval flag, or access token table), the authorization check and the guarded mutation must be atomic — otherwise a second agent can invalidate the decision before the first agent acts on it. C9.8 addresses the agent-isolation half of the problem, while the underlying transactional/concurrency controls remain in ASVS and platform architecture reviews.

## Tooling & Implementation

Current tools, frameworks, and libraries that help implement these controls:

- **Agent frameworks with security features:** [LangGraph](https://github.com/langchain-ai/langgraph) (graph-based orchestration with checkpoint-based human-in-the-loop), [OpenAI Agents SDK](https://openai.com/index/the-next-evolution-of-the-agents-sdk/) (April 2026 update adds controlled workspace and native sandbox execution), [Claude Agent SDK](https://platform.claude.com/docs/en/agent-sdk/overview) (admin-controlled MCP allow-lists, human approval before patches), [CrewAI](https://www.crewai.com/) (multi-agent with OpenTelemetry tracing, Enterprise SOC2), [AutoGen/AG2](https://github.com/microsoft/autogen) (v0.4+ async event-driven with enterprise integrations)
- **Agent sandboxing:** [E2B](https://e2b.dev/) (~15M sandbox sessions/month, Firecracker microVMs, 150ms startup, ~50% Fortune 500 adoption), [Modal](https://modal.com/) (gVisor isolation, per-sandbox egress policies), [Daytona](https://daytona.io/) (sub-90ms sandbox creation, $24M Series A in Feb 2026), [Firecracker](https://firecracker-microvm.github.io/) (KVM-based microVMs with cgroups/namespace isolation), [Fly.io Sprites](https://sprites.dev/) (Jan 2026 — stateful persistent VMs with checkpoint/restore in ~300ms, Firecracker-based, auto-idle billing), [Northflank](https://northflank.com/) (Kata Containers + gVisor adaptive isolation, 2M+ workloads/month, BYOC), [AIO Sandbox](https://github.com/agent-infra/sandbox) (all-in-one: Browser + Shell + File + MCP + VSCode in a single container)
- **Budget and cost controls:** [Helicone](https://www.helicone.ai/) (proxy-based cost tracking across 100+ LLMs with rate limiting), [Langfuse](https://langfuse.com/) (token/cost tracking per trace, open-source self-hosted), [LangSmith](https://smith.langchain.com/) (per-step token/latency/cost metrics). Architectural pattern: per-run hard spend caps, model tier routing, context window trimming.
- **Agent identity and auth:** [SPIFFE/SPIRE](https://spiffe.io/) (CNCF Graduated — short-lived SVIDs for agent workload identity; instance-level IDs for non-deterministic agents), [OPA](https://www.openpolicyagent.org/) (fine-grained policy enforcement via Rego), [SPIFFE+OAuth2 pattern](https://riptides.io/blog-post/spiffe-meets-oauth2-current-landscape-for-secure-workload-identity-in-the-agentic-ai-era) (attestation-bound identity exchanged for scoped OAuth tokens), [OpenAPI x-agent-trust](https://spec.openapis.org/registry/extension/index.html) (registry extension for agent-authenticated API security schemes using `Agent-Signature` and JWKS-based local verification)
- **Agent communication protocols:** [A2A](https://a2a-protocol.org/latest/specification/) (Google, April 2025, v0.3 with gRPC — Agent Cards, OAuth 2.0 auth; donated to Linux Foundation June 2025), [MCP](https://modelcontextprotocol.io/) (Anthropic — OAuth 2.1 with PKCE, 97M monthly SDK downloads; [2025-03-26 spec](https://modelcontextprotocol.io/specification/2025-03-26/basic/authorization) added Streamable HTTP transport replacing SSE, batched tool calls, and tool behavior annotations for read-only vs. destructive operations), [Agentic AI Foundation](https://www.linuxfoundation.org/) (Linux Foundation, Dec 2025 — governs both A2A and MCP)
- **Approval and HITL protocols:** [CHEQ](https://www.ietf.org/archive/id/draft-rosenberg-cheq-00.html) (IETF draft, Rosenberg/Five9, Jennings/Cisco — privacy-preserving confirmation protocol where sensitive approval data stays outside model context; uses HTTP 202 + URI pack for out-of-band human approval with cryptographic non-repudiation), [FIDO agentic authentication](https://fidoalliance.org/fido-alliance-agentic-ai/) (standards work for verifiable user instructions, agent authentication, and trusted delegation), [AP2](https://ap2-protocol.org/) and [Mastercard Verifiable Intent](https://www.mastercard.com/global/en/news-and-trends/stories/2026/verifiable-intent.html) (cryptographically verifiable intent records for autonomous commerce), [Microsoft Agent Framework](https://www.devleader.ca/2026/03/11/tool-approval-and-humanintheloop-in-microsoft-agent-framework) (per-tool approval middleware), [LangGraph interrupt()](https://github.com/langchain-ai/langgraph) (pause graph mid-execution for human input, resume cleanly)
- **Agent security gateways:** [Lasso MCP Gateway](https://github.com/lasso-security/mcp-gateway/) (open-source, April 2025 — security filters for MCP requests, prompt injection blocking, data leakage prevention; [partnered with Portkey](https://portkey.ai/blog/securing-mcp-to-deliver-enterprise-grade-agentic-ai-protection/) for inline MCP pipeline protection at sub-50ms latency), [Lakera Guard](https://www.lakera.ai/) (AI firewall, sub-50ms, 98%+ detection across 100+ languages; [acquired by Check Point in 2025](https://blog.checkpoint.com/securing-the-cloud/best-in-class-genai-security-when-cloudguard-waf-meets-lakera/) to form Global Center of Excellence for AI Security)
- **Agent monitoring:** [AgentOps](https://www.agentops.ai/) (session replay, time-travel debugging, multi-agent support), [Arize Phoenix](https://phoenix.arize.com/) (open-source, multi-step trajectory analysis), [Lunary](https://lunary.ai/) (LLM firewalls, PII masking, auto-categorization), [Microsoft Agent 365](https://www.microsoft.com/en-us/security/blog/2026/03/30/addressing-the-owasp-top-10-risks-in-agentic-ai-with-microsoft-copilot-studio/) (May 2026 enterprise control plane for agent visibility, policy enforcement, and threat protection)

### Implementation Maturity

| Control Area | Tooling Maturity | Notes |
|--------------|:---:|-------|
| C9.1 Execution Budgets | Maturing | Helicone/Langfuse track costs; OpenAI Agents SDK has blocking guardrails. But $47K incidents show most deployments still lack hard budget enforcement. Only 14.4% of orgs deploy agents with full security approval. |
| C9.2 High-Impact Action Approval | Maturing | LangGraph checkpoints, OpenAI tool guardrails, Claude SDK approval gates, Microsoft approval middleware, CHEQ, AP2, and Verifiable Intent all give implementation patterns for explicit approval, parameter binding, nonce/TTL handling, and rollback semantics. FIDO's April 2026 workstreams are the strongest sign that verifiable delegated intent is becoming a standards problem rather than a product feature. |
| C9.3 Tool & Plugin Isolation | Mature | E2B, Modal, Firecracker, Fly.io Sprites, Northflank, and OpenAI's April 2026 Agents SDK sandbox provide production-grade isolation options. Lasso+Portkey MCP Gateway adds inline protocol filtering. The OX MCP STDIO advisory and CrewAI Docker-fallback chain show the remaining gap: sandbox availability, command allowlisting, quota/timeouts, and manifest enforcement must fail closed, not silently downgrade. |
| C9.4 Agent Identity & Audit | Emerging | SPIFFE+OAuth2, x-agent-trust, AP2/Verifiable Intent, and NIST's NCCoE concept paper all point toward signed agent identity, delegation context, and non-repudiable action chains. NIST explicitly calls out OAuth 2.0/2.1, OIDC, SPIFFE/SPIRE, SCIM, NGAC, SP 800-207, SP 800-63-4, and token-protection guidance as candidate building blocks. |
| C9.5 Secure Messaging | Emerging | C9.5 is now focused on semantic validation of agent outputs before they influence downstream agents. A2A session smuggling plus the arXiv 2602.11327 threat model (replay, Sybil, protocol downgrade attacks) show why schema-valid messages still need meaning-level constraints; transport, replay, schema, and MCP-specific protocol hardening belong in ASVS and C10. |
| C9.6 Authorization & Delegation | Emerging | OPA provides the policy engine but agent-specific delegation context propagation (C9.6.2) has limited tooling. C9.6.5 requires the PDP to evaluate the agent's granted permissions and the initiating principal's delegated scope as independent constraints. C9.6.6 requires explicit peer authorization policy for agent-to-agent delegation through an approved registry or allowlist. |
| C9.7 Intent Verification | Emerging | Pre-execution gates are mostly custom-built, but AP2, Verifiable Intent, CHEQ, OpenAPI x-agent-trust, and Microsoft Agent 365 give concrete patterns for binding action parameters back to user intent, policy versions, and audit evidence. C9.7.4's PDP fail-closed requirement is especially important for distributed agents where network partitions are normal operational events. |
| C9.8 Multi-Agent Isolation | Emerging | Namespace isolation for tools, memory, context windows, aggregate swarm rate limits, behavior monitoring, and shutdown controls are documented in research but have limited production tooling. Microsoft Agent 365 is one of the first enterprise control-plane examples for observing and restricting deployed agents, but cross-framework swarm shutdown and emergent-behavior detection remain immature. |

Case study AML.CS0042 (SesameOp) demonstrates agent infrastructure as C2 channel.

- [AP2 Agent Payments Protocol](https://ap2-protocol.org/) — Open protocol for autonomous commerce using verifiable digital credentials, checkout mandates, and payment mandates to provide deterministic proof of user intent and an auditable transaction chain.
- [Mastercard Verifiable Intent](https://www.mastercard.com/global/en/news-and-trends/stories/2026/verifiable-intent.html) (Mar 2026) — Protocol-agnostic trust layer for agentic commerce that links identity, intent, and action into a tamper-resistant record of what the user authorized.

- [NIST SP 800-207: Zero Trust Architecture](https://csrc.nist.gov/pubs/detail/sp/800-207/final) — Agents should be treated as untrusted entities requiring continuous verification, task-scoped credentials, and microsegmentation
- [EU AI Act Article 6](https://artificialintelligenceact.eu/article/6/) — Autonomous AI systems in high-risk domains (critical infrastructure, employment, essential services) require human oversight, risk management, logging, and accuracy/robustness guarantees
- [CSA AI Controls Matrix](https://cloudsecurityalliance.org/artifacts/ai-controls-matrix) (Jul 2025) — 243 controls; includes A2A threat modeling via MAESTRO framework
- [A2A Protocol Specification](https://a2a-protocol.org/latest/specification/) — Security analysis: [Semgrep guide](https://semgrep.dev/blog/2025/a-security-engineers-guide-to-the-a2a-protocol/), [CSA MAESTRO threat model](https://cloudsecurityalliance.org/blog/2025/04/30/threat-modeling-google-s-a2a-protocol-with-the-maestro-framework)
- [MCP Authorization Specification](https://modelcontextprotocol.io/specification/2025-11-25/basic/authorization) — Uses OAuth 2.1, PKCE, protected resource metadata, and RFC 8707 resource indicators; MCP servers must validate that access tokens were issued specifically for the target server and must not pass through unrelated tokens. Note: the early 2026 CVE surge and April 2026 OX STDIO advisory exposed systemic issues in MCP server implementations — see [Vulnerable MCP Project](https://vulnerablemcp.info/) for tracking.
- [Governing Dynamic Capabilities: Cryptographic Binding for AI Agent Tool Use](https://arxiv.org/abs/2603.14332) (Mar 2026) — Proposes three governance requirements: capability integrity (agents cannot silently escalate), behavioral verifiability (claimed actions are verified), and interaction auditability (multi-agent provenance). Implements Ed25519 (~97μs) and BBS+ selective disclosure (~13.8ms). Found 5.8x variance in inference determinism across 9 models — a fundamental challenge for replay-based verification.
- [CHEQ: Confirmation with HITL Exchange](https://www.ietf.org/archive/id/draft-rosenberg-cheq-00.html) (IETF Draft, Jul 2025) — Protocol for human confirmation of AI agent decisions where sensitive data (account numbers, credentials) never reaches the LLM. Uses HTTP 202 + URI pack for out-of-band approval with cryptographic non-repudiation.
- [Security Threat Modeling for AI-Agent Protocols](https://arxiv.org/html/2602.11327) (Feb 2026) — Lifecycle-aware risk framework covering MCP, A2A, Agora, and ANP. Identifies authentication gaps, supply chain integrity risks, sandbox escape vectors, replay attacks, Sybil attacks, and protocol downgrade vulnerabilities across all four protocols.

## Open Research Questions

- [ ] **How do you enforce "least agency" in practice?** — The OWASP Agentic Top 10 introduces the principle but no standard framework exists for defining minimum autonomy levels. NIST's five-tier autonomy taxonomy (Level 0 tool-assisted to Level 4 fully autonomous) is a starting point.
- [ ] **Can agent identity scale to millions of ephemeral instances?** — SPIFFE treats all pod replicas as identical, but agents are non-deterministic. The arXiv 2603.14332 framework found 5.8x variance in inference determinism across models, complicating replay-based verification. Instance-level identities are a proposed solution but the Gravitee 2026 report shows only 21.9% of teams even treat agents as independent identity-bearing entities — the gap between theory and practice remains vast.
- [ ] **How do you detect semantic privilege escalation?** — All technical permission checks pass, but the agent operates outside intended scope. No standard tooling exists for semantic boundary enforcement.
- [ ] **Will A2A, MCP, AP2, and FIDO converge on compatible trust semantics?** — A2A and MCP cover communication and tool access; AP2 and Verifiable Intent cover commerce-specific intent proofs; FIDO is now trying to standardize delegated agent authentication. The open question is whether these efforts produce interoperable identity, intent, and authorization claims or another set of incompatible trust islands.
- [ ] **How do you monitor emergent swarm behavior?** — Multi-agent oscillation, deadlocks, and coordinated manipulation (steganographic collusion) require runtime behavioral analysis that goes beyond individual agent monitoring. Limited production tooling exists.
- [ ] **What does agentic misalignment defense look like?** — Anthropic's study showed 96% of models attempt blackmail under specific conditions. This is a model behavior problem (C07) that manifests as an agent security problem (C09).
- [ ] **How do you secure the MCP ecosystem at scale?** — 30+ CVEs in 60 days (Jan-Feb 2026), 8,000+ exposed servers, 82% path traversal rate, 38-41% of registry servers lacking auth, and the April 2026 OX STDIO advisory suggest the MCP ecosystem has a systemic security gap. Trust-on-first-use models (MCPoison/CVE-2025-54136) and user-editable STDIO launch configs are fundamentally weak defaults. The Vulnerable MCP Project is tracking issues, but no certification or security baseline exists for MCP server implementations.
- [ ] **Is agent-to-agent lateral movement the next major breach vector?** — The OpenClaw crisis (21,639 exposed instances, inherited OAuth tokens enabling cross-system traversal), the PleaseFix browser hijack, and the Meta rogue agent incident demonstrate that compromised or uncontrolled agents provide lateral movement paths invisible to traditional monitoring. The Gravitee 2026 report found 25.5% of deployed agents can create and task other agents with shared credentials — a direct enabler. No standard detection framework exists for agent-layer lateral movement.
- [ ] **How should approval protocols handle the privacy-sensitive action problem?** — The CHEQ IETF draft proposes keeping sensitive data outside the model context during approval, while AP2 and Verifiable Intent focus on cryptographic proof of user intent for commerce. The remaining gap is a general-purpose approval pattern that binds exact action parameters, nonce, TTL, requester, user intent, and rollback semantics without overexposing sensitive data to the model or the approving party.
- [ ] **Can frameworks enforce true fail-closed semantics across isolation tiers?** — The CrewAI sandbox fallback and MCP STDIO advisory both expose silent degradation: weak runtime behavior looks like convenience until it becomes host execution. C9.3.2 and C9.7.4 require termination when quotas, timeouts, isolation guarantees, or PDP dependencies fail, but no widely adopted framework API lets a developer declare required isolation guarantees and have the runtime enforce them uniformly.
- [ ] **Is agent state integrity a separate subsystem or a layer of memory?** — C9.4.5 overlaps with C8 memory controls, and the 2026 research push treats agent memory as a first-class architectural component. But integrity protection — cryptographic MACs or signatures over stored context, goals, and partial results — has no mainstream implementation. Without verifiable state integrity, an attacker with one-time write access can compromise agents indefinitely.
