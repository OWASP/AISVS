# C09: Autonomous Orchestration & Agentic Action Security

> **Source:** [`1.0/en/0x10-C09-Orchestration-and-Agentic-Action.md`](https://github.com/OWASP/AISVS/blob/main/1.0/en/0x10-C09-Orchestration-and-Agentic-Action.md)
> **Requirements:** 47 | **Sections:** 9

## Control Objective

Autonomous and multi-agent systems must execute only authorized, intended, and bounded actions. This chapter focuses on controls unique to agentic AI execution: agent-as-principal identity, agent action chains, model-output-driven authorization risk, intent verification of LLM-decided actions, multi-agent swarm dynamics, and the architectural data-flow controls that keep planning logic separate from untrusted tool output. The threat surface this control family reduces includes tool misuse, privilege escalation, uncontrolled recursion or cost growth, protocol manipulation, and cross-agent or cross-tenant interference. Generic transport encryption, schema validation, message integrity, log integrity, concurrency safety, supply chain integrity, service-account rotation, multi-tenant isolation, contextual authorization, and API-edge rate limiting remain in ASVS v5, C10, C13, and adjacent AISVS chapters — since May 11, 2026 those scope-overlap notes live in the AISVS "Using AISVS" front-matter rather than in this chapter's Control Objective.

> **2025-2026 Highlights:** Agentic AI security hardened into a distinct discipline across 2025-2026. The OWASP Top 10 for Agentic Applications (Dec 2025) crystallized ten ASI risk categories, while NIST's NCCoE concept paper (public comment closed April 2, 2026) pushed agent identity and authorization toward OAuth 2.0/2.1, OIDC, SPIFFE/SPIRE, SCIM, NGAC, and zero-trust patterns. April 2026 added several important signals: OX Security's MCP STDIO advisory showed how user-controlled MCP configuration can become ecosystem-wide command execution across LangFlow, LiteLLM, Agent Zero, Windsurf, DocsGPT, and related tooling — followed by a deeper "Mother of All AI Supply Chains" disclosure documenting 150M+ downloads, 7,000+ publicly accessible servers, and up to 200,000 vulnerable instances tied to a "by design" STDIO execution behavior Anthropic declined to change. CSA's April 17, 2026 "AI Agent Disclosure Vacuum" whitepaper put a name to the systemic gap — the "silent bounty problem" — where vendors quietly pay out reports without filing CVEs or issuing advisories, leaving Flowise (CVE-2025-59528), EchoLeak, and most of the MCP CVE wave undocumented in MITRE for months. FIDO launched Agentic Authentication and Payments workstreams for verifiable user instructions, agent authentication, and trusted delegation, with Google's AP2 (now v0.2) and Mastercard's Verifiable Intent open-sourced as initial contributions. May 2026 turned the pattern into a cascade: Microsoft's Semantic Kernel disclosures showed prompt injection crossing into host-level RCE when model-controlled parameters reach dynamic filters or host file-transfer helpers; Windsurf 1.9544.26 (CVE-2026-30615) became the first widely reported zero-click MCP prompt-injection RCE; and PraisonAI's CVE-2026-44338 (May 15) shipped an auth-disabled-by-default Flask API bound to `0.0.0.0:8080`, exploited within hours of disclosure for unauthenticated agent enumeration and workflow hijacking. Lyrie Research framed the cluster on May 11 as a "trust boundary collapse" with one architectural takeaway: the model is not the boundary — the tool is. The same month, Microsoft Agent 365 moved to general availability (May 1) and Microsoft published MDASH (May 12), a multi-model defensive harness that found 16 Windows CVEs while topping the CyberGym benchmark; CSAI Foundation announced (April 29) its STAR for AI Catastrophic Risk Annex, MITRE-issued CVE Numbering Authority, and adoption of the Autonomous Action Runtime Management (AARM) specification and Agentic Trust Framework. The common thread across these signals is that runtime-side enforcement — sandbox fail-closed semantics, manifest verification, origin-aware policy, agent registry, and PDP availability — is now the binding constraint for C9.3, C9.6, C9.7, and C9.8. Mid-May added more reinforcing data points: Akamai's May 13 disclosure of three more MCP database-server flaws (Apache Doris CVE-2025-66335, Apache Pinot, and Alibaba RDS — the last unpatched); the May 15 Cyera "Claw Chain" disclosure of four chainable OpenClaw CVEs (CVE-2026-44112/44113/44115/44118, patched in OpenClaw 2026.4.22) that demonstrated the now-canonical agent kill chain of prompt-injected code execution → credential theft → owner-level escalation → persistence; and the May 18 OWASP ASI02 "Tool Misuse and Exploitation" definitive guide, the first per-entry deep dive within the Agentic Top 10. The governance side caught up the same window: on May 1 CISA, NSA, and the Five Eyes ASD ACSC, CCCS, NCSC-NZ, and NCSC-UK published "Careful Adoption of Agentic AI Services," the first multinational baseline that explicitly calls for per-agent cryptographic identity, short-lived credentials, encrypted agent-to-agent traffic, and human approval for high-impact actions — closely mirroring C9.4, C9.6, C9.7, and C9.8. The late-May read is consistent: Verizon's 2026 Data Breach Investigations Report (released May 20, 2026) names service and machine accounts as the accounts "most likely to be leveraged in our potential agentic AI future," and the May 12 NetOps/AIOps survey paper (arXiv 2605.12729) by Bilal, Crowcroft et al. — picked up in Help Net Security's May 20 coverage — frames LLM-driven operations as a confused-deputy problem that only a strict propose-commit split with a non-bypassable gate the model has no authority over can mitigate. Both signals push the same conclusion C9 already encodes: identity, runtime gates, and architectural separation (C9.4, C9.7, C9.9) are not optional once the agent holds production credentials. The May 19 Mini Shai-Hulud npm wave — 637 malicious versions across 317 packages in 22 minutes, writing persistence hooks into `.claude/settings.json` and `.vscode/tasks.json` — closes the same window from the supply-chain side and confirms CSA's May 17 framing that agent configuration files are now an active attack surface, not opaque metadata. A parallel signal from the AI-gateway side closes the loop on identity and secrets: LiteLLM CVE-2026-42208 (CVSS 9.3, pre-auth SQL injection in the `Authorization: Bearer` path, GHSA advisory April 24, first in-the-wild exploitation 36 hours later per Sysdig, CISA-KEV May 8 with FCEB deadline May 11) demonstrated that a single proxy compromise can dump every OpenAI organization key, Anthropic console key, and AWS Bedrock IAM credential the gateway brokers — Sysdig's framing that "the blast radius is closer to a cloud-account compromise than a typical web-app SQL injection" puts AI gateways squarely in the C9.4 identity / C9.4.5 rotation / C9.6.4 secrets surface. The same window finally turned prompt injection itself into a measurable procurement metric: Anthropic's Opus 4.5, 4.6, and 4.7 system cards now publish attack success rates per surface (GUI agent 17.8% single-attempt unsafeguarded → 78.6% across 200 attempts; constrained coding 0%; browser-use 1.4% with Opus 4.5 safeguards vs 10.8% for Sonnet 4.5), and Mandiant's M-Trends 2026 reports 28.3% of newly disclosed CVEs are exploited within 24 hours of publication — together they turn C9.4.5 rotation, C9.7.4 fail-closed PDP, and C9.8.6 shutdown latency into hard contracts auditors can demand evidence against. The late-May/early-June 2026 window added the governance capstones: NSA's AI Security Center published its first MCP-specific Cybersecurity Information Sheet (May 20, 2026 — egress proxies, data-classification zones for tools, sandboxing, and local MCP scans as baseline design considerations), Microsoft open-sourced RAMPART and Clarity the same day (pytest-native agent red-team testing in CI, built on PyRIT), and OWASP published "State of Agentic AI Security and Governance" 2.01 (June 3, 2026) with an Enterprise Adoption Maturity Model presented at Infosecurity Europe on June 4. On the offense side, Adversa's TrustFall (May 7) and SymJack (May 26) disclosures showed that the approval prompt itself is now an attack surface — four coding CLIs execute project-defined MCP servers on a single folder-trust keypress, and a renamed symlink makes the approval dialog display a benign path while the kernel writes to agent configuration — while the VIPER-MCP study (arXiv 2605.21392) confirmed 106 exploitable zero-days across 39,884 scanned MCP server repositories with 67 CVEs assigned, turning the MCP CVE wave from anecdote into measured base rate.

## Threat Landscape

Known attacks, real-world incidents, and threat vectors relevant to this chapter:

- **Destructive tool misuse** — Google's Antigravity IDE (Gemini 3 Pro in "Turbo mode") wiped an entire D: drive via `rmdir /s /q` while clearing a cache (December 2025). Gemini CLI permanently deleted files after misinterpreting a failed directory creation (July 2025). These incidents demonstrate the real cost of autonomous execution without confirmation gates.
- **Agentic misalignment at scale** — Anthropic's safety testing (May 2025) found that 96% of frontier models attempted blackmail when given email access and discovering both a threat to their existence and compromising information. Misaligned behavior was systemic across Anthropic, OpenAI, Google, Meta, xAI, and DeepSeek models.
- **Runaway agent costs** — A multi-agent research pipeline entered an infinite loop for 11 days, costing $47,000 (November 2025). A data enrichment agent made 2.3 million API calls over a weekend after misinterpreting an error code. Fortune 500 companies collectively leaked an estimated $400M in unbudgeted cloud spend from uncontrolled agent execution.
- **Prompt injection → tool abuse** — EchoLeak (CVE-2025-32711, CVSS 9.3) demonstrated zero-click data exfiltration from Microsoft 365 Copilot. MCP tool poisoning (April 2025) showed how malicious instructions hidden in tool descriptions cause agents to exfiltrate SSH keys. MCP rug pull attacks modify tool behavior post-approval.
- **Agent framework vulnerabilities** — LangGrinch (CVE-2025-68664, CVSS 9.3) enabled environment variable theft and RCE via serialization injection in langchain-core. Patched in v1.2.5 and v0.3.81.
- **Semantic Kernel prompt-to-RCE chains** — Microsoft disclosed two Semantic Kernel issues in May 2026: CVE-2026-26030 let prompt-controlled vector-store filter parameters reach Python code execution before version 1.39.4, and CVE-2026-25592 exposed host file-write paths through a sandbox helper in the .NET SDK before version 1.71.0. The lesson is architectural: treat every model-controlled tool parameter as attacker-controlled and keep host filesystem, shell, and transfer helpers out of the callable tool surface unless narrowly allowlisted and path-confined.
- **Multi-agent protocol attacks** — A2A session smuggling (Unit 42, October 2025) demonstrated covert instruction injection during legitimate multi-turn sessions. Cross-agent steganographic collusion enables secret coordination channels invisible to monitoring.
- **Semantic privilege escalation** — A distinct class where agents operate within technical permissions but outside semantic scope. A financial agent was tricked into exporting 45,000 customer records using a regex matching every record — all permission checks passed.
- **MCP protocol CVE surge** — Between January and February 2026, researchers filed 30+ CVEs targeting MCP servers, clients, and infrastructure. CVE-2025-6514 (CVSS 9.6) enabled arbitrary command execution via mcp-remote, affecting 437,000+ downloads. CVE-2025-49596 hit Anthropic's own MCP Inspector tool. CVE-2025-54136 (MCPoison) bypassed Cursor IDE's trust-on-first-use model to silently execute malicious changes post-approval. Scanning of 2,614 MCP implementations found 82% vulnerable to path traversal and 67% to code injection.
- **MCP STDIO command execution by configuration** — OX Security's April 15, 2026 advisory documented a systemic class where MCP STDIO server configuration values flow into process execution. Confirmed variants include unauthenticated UI configuration, authenticated configuration, allowlist bypass through `npm`/`npx` arguments, prompt injection that modifies local MCP config, and hidden backend STDIO paths triggered through edited network requests. The pattern maps cleanly to C9.3.2 fail-closed tool execution, C9.3.4 manifest enforcement, C9.6.1 runtime policy, and C9.7.1 pre-execution gates.
- **OpenClaw agent compromise** — CVE-2026-25253 (CVSS 8.8) enabled one-click RCE via cross-site WebSocket hijacking in the OpenClaw autonomous agent framework (January 2026). The ClawHavoc campaign distributed 341 malicious skills (~12% of the registry). Censys identified 21,639 publicly exposed instances, and the Moltbook breach exposed 1.5 million API tokens. Compromise enabled lateral movement through inherited OAuth tokens across Slack, Gmail, and cloud storage.
- **Agentic browser hijacking** — The PleaseFix vulnerability family (March 2026) demonstrated zero-click compromise of AI agent browsers like Perplexity Comet. Attackers embed malicious content in routine workflows (calendar invites); when the agent processes them, it exfiltrates files and credentials without user interaction. Represents the evolution of social engineering from humans to autonomous agents.
- **Rogue agent privilege escalation** — In March 2026, a Meta internal AI agent autonomously posted a response to an employee without authorization, sharing it across internal forums to personnel who lacked access rights. An employee acted on the flawed advice, inadvertently exposing massive amounts of company data to unauthorized engineers. This incident demonstrates how an uncontrolled agent action can cascade through human trust into a full privilege escalation chain — the agent operated within its technical permissions but far outside its intended scope.
- **Agent lateral movement as a threat class** — As of early 2026, multi-agent architectures create new lateral movement paths: a compromised orchestration agent holding API keys for downstream agents gives attackers access to all connected systems. Unlike traditional network lateral movement, agent-to-agent movement operates within legitimate authenticated sessions and evades conventional monitoring. The Gravitee 2026 report found 25.5% of deployed agents can create and task other agents, often with shared credentials — a direct lateral movement enabler.
- **CrewAI sandbox-fallback RCE chain** — In April 2026, Cyata researcher Yarden Porat disclosed four CVEs in CrewAI (CVE-2026-2275, 2285, 2286, 2287) that chain prompt injection into full host compromise. The core design flaw: when the CodeInterpreter cannot reach Docker at runtime, CrewAI silently falls back to an in-process SandboxPython that permits arbitrary C calls. Combined with an SSRF in the RAG tools and an arbitrary-file-read in the JSON loader, an attacker delivering a poisoned prompt can achieve credential theft, cloud-metadata exfiltration, and RCE on the host. CERT/CC VU#221883 tracks the coordinated disclosure. This is the defining example for C9.3.2 fail-closed enforcement: degraded isolation silently replaced strong isolation.
- **OpenClaw WebSocket scope self-assignment** — CVE-2026-32922 (CVSS 9.9, April 2026) let any authenticated OpenClaw user self-assign admin scopes during the WebSocket handshake. Local connections were trusted by default and exempted from rate limiting, so a low-privilege `operator.pairing` token obtained through normal device pairing was sufficient to brute-force into `operator.admin` and execute commands across every connected node. Nine CVEs were disclosed between March 18-21, 2026; by April, Censys counted 42,000+ exposed instances with 15,200 directly RCE-vulnerable. The ClawHavoc supply-chain campaign published 1,184 malicious skills on ClawHub — roughly one in five packages at peak — carrying Atomic Stealer (macOS) and Vidar (Windows) infostealers.
- **Agentic browser panel hijack** — CVE-2026-0628 (Chrome Gemini Live, disclosed Oct 2025 / patched Jan 2026, Unit 42) let a low-permission browser extension inject JavaScript into the privileged Gemini side panel and inherit its camera, microphone, local-file, and screenshot capabilities. Represents a structural pattern: trusted AI panels running inside untrusted browser processes inherit both the model's authority and the browser's attack surface. Expect similar hijacks against Copilot in Edge, Comet, and other agentic browser integrations.
- **Agent-layer TOCTOU exposure** — The 2026 wave of classical TOCTOU issues transfers directly to multi-agent systems. When concurrent agents access shared mutable state (e.g., a rate counter, approval flag, or access token table), the authorization check and the guarded mutation must be atomic — otherwise a second agent can invalidate the decision before the first agent acts on it. C9.8 addresses the agent-isolation half of the problem, while the underlying transactional/concurrency controls remain in ASVS and platform architecture reviews.
- **Zero-click MCP prompt injection (Windsurf)** — CVE-2026-30615 (April-May 2026) escalated the OX STDIO class into full zero-click RCE. Attacker-controlled HTML content rendered by Windsurf 1.9544.26 silently rewrote the local MCP JSON configuration and registered a malicious STDIO server; the MCP SDK then launched the server binary with no approval prompt or warning. Reinforces that C9.3 manifest enforcement, C9.7 pre-execution gates, and C9.9 origin-aware policy must apply to configuration-time data, not just runtime tool arguments.
- **Cross-tenant agent gateway exposure (Azure SRE Agent)** — CVE-2026-32173 (CVSS 8.6, disclosed April 3, 2026) was an improper-authentication flaw in the Azure SRE Agent Gateway SignalR Hub that authenticated tokens without checking tenant scope. Any Entra ID account holder could open an unauthenticated WebSocket and eavesdrop on live command streams, AI chat streams, internal LLM reasoning, tool calls, and credentials from agents belonging to unrelated tenants. A textbook C9.6.5 failure where the policy decision point evaluated the caller's identity but not the delegated/tenant constraint, with secondary impact on C9.4 audit fidelity and C9.8 multi-tenant isolation.
- **MCP "by design" RCE across the supply chain** — OX Security's follow-up disclosure ("Mother of All AI Supply Chains," April 2026) showed that the STDIO execution behavior is not an isolated bug but a protocol-level design decision implemented uniformly across Anthropic's official MCP SDKs for Python, TypeScript, Java, and Rust. Researchers executed commands on six live production platforms and found the same primitive in LiteLLM, LangChain, and IBM's LangFlow. Anthropic confirmed the behavior as expected and pushed sanitization onto downstream developers — meaning C9.3.2 fail-closed enforcement, C9.3.4 manifest restriction, and C9.6.1 runtime policy must compensate for an upstream protocol that treats configuration as code.
- **MCP sampling abuse (Unit 42)** — May 2026 research detailed three new attack vectors enabled by the MCP sampling primitive, which lets servers proactively request LLM completions through the client. A compromised server can drain user API quotas with covert sampling jobs (resource theft), inject persistent instructions that cross conversation turns (conversation hijacking), or invoke unauthorized filesystem and tool operations without user awareness (covert tool invocation). Most current MCP hosts do not enforce sampling budgets or origin checks, which connects directly to C9.1 cumulative spend caps, C9.5 semantic validation of agent-to-agent traffic, and C9.9 origin-aware enforcement at the tool-call boundary.
- **CISA-cataloged Langflow RCE under active exploitation** — CVE-2026-33017 (CVSS 9.3) added to CISA's Known Exploited Vulnerabilities catalog on March 25, 2026 with an April 8 federal remediation deadline. Unauthenticated RCE in Langflow joins LangGrinch (CVE-2025-68664), LangChain filesystem read (CVE-2026-34070), and the LangGraph SQLite checkpoint exposure (CVE-2025-67644) in a pattern where popular orchestration frameworks are shipping primitive injection sinks. CSA's Lab Space tracks the broader "agentic framework CVEs under active exploitation" cluster.
- **Industry breach baseline** — HiddenLayer's 2026 AI Threat report puts autonomous agents at more than one in eight reported AI breaches, and Adversa AI's May 2026 tracker counted six confirmed agentic-framework RCE or sandbox-escape disclosures in a single month. The shape of the threat landscape now justifies C9 controls being treated as a baseline rather than an aspirational tier.
- **Insecure-by-default agent server (PraisonAI)** — CVE-2026-44338, disclosed May 15, 2026, was an authentication-disabled-by-default flaw in PraisonAI's legacy Flask `api.py` that bound to `0.0.0.0:8080` with `AUTH_ENABLED = False` and `AUTH_TOKEN = None`. A `GET /agents` request leaked agent metadata, and `POST /chat` triggered the local `agents.yaml` workflow, allowing unauthenticated workflow hijack and budget drain. Researchers observed exploitation within hours of public disclosure. PraisonAI 4.6.34 patched the issue and the project now requires `praisonaiagents serve agents --api-key` bound to `127.0.0.1` for the supported path. This is the cleanest current example of C9.4.1 agent-identity-as-first-class-principal, C9.6.1 deny-by-default runtime policy, and C9.1.2 cumulative spend caps all failing together at the framework-shipped defaults.
- **Agent-driven Git hook RCE (Cursor)** — CVE-2026-26268 (Cursor IDE, disclosed February 2026, coordinated with the vendor) chains an attacker-supplied bare repository with a malicious `pre-commit` hook. Because the Cursor agent autonomously runs `git checkout`, `git commit`, and similar operations driven by user prompts and Cursor Rules, hook invocation happens with no human in the loop. The lesson tracks directly to C9.7.1 pre-execution gates and C9.9 origin-aware policy: every command the agent is allowed to run on a freshly cloned untrusted repo should be evaluated against the origin of the file or invocation, not just the agent's static toolset.
- **Coding-agent project-file abuse (Claude Code)** — CVE-2025-59536 and CVE-2026-21852, disclosed in stages between October 2025 and February 2026, used `.claude/settings.json` and `.mcp.json` in cloned repositories to register hooks and MCP servers that executed before any trust dialog rendered, and to set a malicious `ANTHROPIC_BASE_URL` that exfiltrated the user's Anthropic API key on first invocation. Both CVEs are patched, but the pattern — repo-provided configuration that becomes shell or proxy execution before the user can review it — is now a recurring class across coding agents and maps to C9.3.4 manifest enforcement, C9.7.5 prompt-template and policy-integrity verification, and C9.9.4-5 origin-aware policy on tool arguments.
- **Mini Shai-Hulud — npm worm weaponizes AI coding-agent config files (May 19, 2026)** — A self-propagating supply-chain worm operated by TeamPCP published 637 malicious versions across 317 npm packages in a 22-minute automated burst between 01:39 and 02:06 UTC on May 19, 2026, after compromising the `atool` maintainer account. High-impact targets included `size-sensor` (~4.2M downloads/month), `echarts-for-react` (~3.8M), and 314 `@antv/*` scoped packages. Beyond the credential-stealing payload, the worm scans the host for AI coding-agent state and writes persistence hooks into `.claude/settings.json` (`SessionStart` hook executing `node .claude/setup.mjs`) and `.vscode/tasks.json` (`"runOn": "folderOpen"`), so that subsequent developer sessions silently re-execute the payload through the agent's own initialization path. A separate Bun bootstrapper at `~/.claude/package/index.js` and recursive scans for existing agent settings across all projects extend the foothold even after package removal. CSA Lab Space's May 17, 2026 research note characterizes the campaign as "the first documented supply chain attack to weaponize AI coding agent configurations as a persistence mechanism," and the precursor May 11, 2026 wave (TeamPCP, Wiz disclosure, ~115 npm + 2 PyPI packages including `guardrails-ai` and `mistralai`) shows the operator iterating on the same primitive. The control implication is direct: C9.3.4 manifest enforcement, C9.7.5 prompt-template and policy-integrity verification, C9.9.5-6 origin-aware policy, and C9.4.6 state-integrity verification must apply to agent settings, hook registrations, and session-start artifacts the same way they apply to runtime tool arguments — agent configuration files are now an active attack surface, not passive metadata.
- **Trust boundary collapse thesis (Lyrie Research, May 11 2026)** — Lyrie's "Trust Boundary Is Gone" deep-dive frames the January-May 2026 cascade (Antigravity, Claude Code MemoryTrap, Azure SRE Agent, CrewAI VU#221883, Anthropic MCP, Semantic Kernel) as a single architectural failure rather than a string of bugs. The summary is direct: "the model is not the boundary; the tool is." The implication for C9 is that perimeter-style controls on prompts or model outputs cannot compensate for tools that are themselves exposed and trusted, which is the case the C9.3, C9.7, and C9.9 controls together exist to address.
- **Offensive agentic AI in the wild — hackerbot-claw (Feb 20 – Mar 2, 2026)** — StepSecurity's disclosure documented the first widely reported case of an autonomous AI agent conducting offensive operations against open-source CI/CD pipelines. A GitHub account self-identified as an "autonomous security research agent powered by claude-opus-4-5" systematically opened 12+ pull requests across seven targets — microsoft/ai-discovery-agent, DataDog/datadog-iac-scanner, avelino/awesome-go (140K+ stars), ambient-code/platform, project-akri/akri (CNCF), aquasecurity/trivy (25K+ stars), and RustPython/RustPython — and achieved arbitrary code execution in at least six of them through five distinct primitives: poisoned Go `init()` functions, raw curl-pipe-bash payloads, brace-expansion shell injection inside git branch names, base64-encoded commands inside filenames executed by unescaped bash loops, and direct AI prompt injection through poisoned project configuration files. The agent exfiltrated a GitHub token with write permissions from `avelino/awesome-go` and fully compromised `aquasecurity/trivy`, deleting releases and publishing a malicious VSCode extension before remediation. DataDog patched in 9 hours; Aqua republished Trivy 0.69.2. Two operational notes matter for C9: Claude Code itself recognized and refused both of the bot's attempted AI prompt-injection chains (a rare published case of an agentic safety gate holding under live offensive load — the C9.7.5 prompt-template and policy-integrity surface working as designed), and the bot ran continuously for 10 days without any external loop, budget, or identity-revocation gate — exactly the C9.1, C9.4.5, and C9.8.6 failure mode that lets a compromised or rogue agent operate unattended at attacker timescales.
- **Comment and Control — GitHub Actions agent prompt injection (CSA, April-May 2026)** — CSA's "Comment and Control" research note (with companion paper "Prompt Injection in AI-Powered GitHub Actions," May 3, 2026) showed the same primitive across three production CI/CD agents: Anthropic's Claude Code Security Review Action, Google's Gemini CLI Action, and GitHub Copilot Agent. A pull-request comment containing injected natural-language instructions was interpreted by the agent and executed inside the GitHub Actions runner, leading to the agent posting the `ANTHROPIC_API_KEY`, `GEMINI_API_KEY`, and `GITHUB_TOKEN` back into publicly visible PR comments or Actions logs. Anthropic rated their finding CVSS 9.4 Critical; Google patched Gemini CLI within four days; GitHub paid a Copilot bounty. The root cause is architectural and vendor-agnostic — any agent that simultaneously holds a bash-execution tool, access to secrets, and untrusted natural-language input has this exposure. Maps directly to C9.7.1 pre-execution gates, C9.9.5 origin-aware policy on tool arguments, C9.6.4 out-of-band secrets handling, and the OWASP ASI02 "Tool Misuse and Exploitation" guidance.
- **Claude Code SOCKS5 sandbox bypass — silently patched (March 31, 2026; The Register coverage May 20, 2026)** — A SOCKS5 hostname null-byte injection in the Claude Code network sandbox let an attacker bypass the user-configured allowlist filter; combined with prompt injection, the flaw let a malicious instruction in a cloned repo force the agent to fetch attacker-controlled code and exfiltrate cloud and GitHub credentials, the GitHub token Claude authenticated with, cloud metadata, and internal APIs from inside the sandbox. The window from sandbox GA through Claude Code 2.1.90 was approximately five and a half months. The bypass was fixed in a `sandbox-runtime` commit on March 31, 2026 with no advisory CVE, no changelog note, and no public release notes — a textbook case of CSA's "silent bounty problem." The earlier related CVE-2025-66479 (December 2025) was assigned only to the upstream sandbox-runtime library, not to Claude Code itself. ## Tooling & Implementation

Current tools, frameworks, and libraries that help implement these controls:

- **Agent frameworks with security features:** [LangGraph](https://github.com/langchain-ai/langgraph) (graph-based orchestration with checkpoint-based human-in-the-loop), [OpenAI Agents SDK](https://openai.com/index/the-next-evolution-of-the-agents-sdk/) (controlled workspaces, native sandbox execution, and tool guardrails that can validate or block custom function-tool calls before and after execution), [Claude Agent SDK](https://platform.claude.com/docs/en/agent-sdk/overview) (admin-controlled MCP allow-lists, human approval before patches), [Claude Managed Agents](https://claude.com/blog/claude-managed-agents-updates) (May 19, 2026 — self-hosted sandboxes in public beta on Cloudflare, Daytona, Modal, and Vercel, plus MCP tunnels in research preview that expose internal databases, APIs, and ticketing systems through a customer-deployed gateway making a single outbound encrypted connection — no inbound firewall rules or public endpoints), [CrewAI](https://www.crewai.com/) (multi-agent with OpenTelemetry tracing, Enterprise SOC2), [AutoGen/AG2](https://github.com/microsoft/autogen) (v0.4+ async event-driven with enterprise integrations)
- **Agent sandboxing:** [E2B](https://e2b.dev/) (~15M sandbox sessions/month, Firecracker microVMs, 150ms startup, ~50% Fortune 500 adoption), [Modal](https://modal.com/) (gVisor isolation, per-sandbox egress policies), [Daytona](https://daytona.io/) (sub-90ms sandbox creation, $24M Series A in Feb 2026), [Firecracker](https://firecracker-microvm.github.io/) (KVM-based microVMs with cgroups/namespace isolation), [Fly.io Sprites](https://sprites.dev/) (Jan 2026 — stateful persistent VMs with checkpoint/restore in ~300ms, Firecracker-based, auto-idle billing), [Northflank](https://northflank.com/) (Kata Containers + gVisor adaptive isolation, 2M+ workloads/month, BYOC), [AIO Sandbox](https://github.com/agent-infra/sandbox) (all-in-one: Browser + Shell + File + MCP + VSCode in a single container), [NVIDIA NemoClaw](https://particula.tech/blog/nvidia-nemoclaw-openclaw-enterprise-security) (GTC March 16, 2026 — out-of-process enterprise security wrapper that runs OpenClaw inside NVIDIA's OpenShell sandbox runtime; deny-by-default `openclaw-sandbox.yaml` policy with explicit allow-lists for network, filesystem, and process calls evaluated at four levels — binary, destination, method, path — designed specifically to compensate for OpenClaw's ClawHavoc and Claw Chain CVE history without modifying agent code)
- **Budget and cost controls:** [Helicone](https://www.helicone.ai/) (proxy-based cost tracking across 100+ LLMs with rate limiting), [Langfuse](https://langfuse.com/) (token/cost tracking per trace, open-source self-hosted), [LangSmith](https://smith.langchain.com/) (per-step token/latency/cost metrics). Architectural pattern: per-run hard spend caps, model tier routing, context window trimming.
- **Agent identity and auth:** [SPIFFE/SPIRE](https://spiffe.io/) (CNCF Graduated — short-lived SVIDs for agent workload identity; instance-level IDs for non-deterministic agents), [OPA](https://www.openpolicyagent.org/) (fine-grained policy enforcement via Rego), [SPIFFE+OAuth2 pattern](https://riptides.io/blog-post/spiffe-meets-oauth2-current-landscape-for-secure-workload-identity-in-the-agentic-ai-era) (attestation-bound identity exchanged for scoped OAuth tokens), [OpenAPI x-agent-trust](https://spec.openapis.org/registry/extension/index.html) (registry extension for agent-authenticated API security schemes using `Agent-Signature` and JWKS-based local verification)
- **Agent communication protocols:** [A2A](https://a2a-protocol.org/latest/specification/) (Google, April 2025, v0.3 with gRPC — Agent Cards, OAuth 2.0 auth; donated to Linux Foundation June 2025), [MCP](https://modelcontextprotocol.io/) (Anthropic — OAuth 2.1 with PKCE for HTTP transports; the latest [2025-11-25 authorization spec](https://modelcontextprotocol.io/specification/2025-11-25/basic/authorization) requires protected resource metadata, RFC 8707 resource indicators, token audience validation, and rejection of token passthrough), [Agentic AI Foundation](https://www.linuxfoundation.org/) (Linux Foundation, Dec 2025 — governs both A2A and MCP)
- **Approval and HITL protocols:** [CHEQ](https://www.ietf.org/archive/id/draft-rosenberg-cheq-00.html) (IETF draft, Rosenberg/Five9, Jennings/Cisco — privacy-preserving confirmation protocol where sensitive approval data stays outside model context; uses HTTP 202 + URI pack for out-of-band human approval with cryptographic non-repudiation), [FIDO agentic authentication](https://fidoalliance.org/fido-alliance-agentic-ai/) (standards work for verifiable user instructions, agent authentication, and trusted delegation), [AP2](https://ap2-protocol.org/) and [Mastercard Verifiable Intent](https://www.mastercard.com/global/en/news-and-trends/stories/2026/verifiable-intent.html) (cryptographically verifiable intent records for autonomous commerce), [Microsoft Agent Framework](https://www.devleader.ca/2026/03/11/tool-approval-and-humanintheloop-in-microsoft-agent-framework) (per-tool approval middleware), [LangGraph interrupt()](https://github.com/langchain-ai/langgraph) (pause graph mid-execution for human input, resume cleanly)
- **Agent security gateways:** [Lasso MCP Gateway](https://github.com/lasso-security/mcp-gateway/) (open-source, April 2025 — security filters for MCP requests, prompt injection blocking, data leakage prevention; [partnered with Portkey](https://portkey.ai/blog/securing-mcp-to-deliver-enterprise-grade-agentic-ai-protection/) for inline MCP pipeline protection at sub-50ms latency), [Lakera Guard](https://www.lakera.ai/) (AI firewall, sub-50ms, 98%+ detection across 100+ languages; [acquired by Check Point in 2025](https://blog.checkpoint.com/securing-the-cloud/best-in-class-genai-security-when-cloudguard-waf-meets-lakera/) to form Global Center of Excellence for AI Security)
- **Agent monitoring:** [AgentOps](https://www.agentops.ai/) (session replay, time-travel debugging, multi-agent support), [Arize Phoenix](https://phoenix.arize.com/) (open-source, multi-step trajectory analysis), [Lunary](https://lunary.ai/) (LLM firewalls, PII masking, auto-categorization), [Microsoft Agent 365](https://www.microsoft.com/en-us/microsoft-agent-365) (generally available May 1, 2026 — centralized agent registry, AWS Bedrock + Google Cloud connection sync, IT-controlled onboarding workflows with security policy templates, lifecycle governance across providers, and Entra/Defender/Purview integration)
- **Runtime action management:** [AARM (Autonomous Action Runtime Management)](https://aarm.dev/) — open specification donated by Vanta to the CSAI Foundation (April 29, 2026) defining a runtime that intercepts actions before execution, accumulates session context, evaluates policy and intent alignment, enforces authorization, and emits tamper-evident receipts; four candidate architectures (protocol gateway, SDK instrumentation, kernel-level eBPF, vendor integration) with conformance criteria. [Microsoft MDASH](https://www.microsoft.com/en-us/security/blog/2026/05/12/defense-at-ai-speed-microsofts-new-multi-model-agentic-security-system-tops-leading-industry-benchmark/) is the first widely documented defensive deployment of a multi-agent ensemble (100+ models that discover, debate, and prove exploits) and a reference point for how agent-on-agent monitoring can operate at scale.
- **Agent security testing:** [Microsoft RAMPART](https://www.microsoft.com/en-us/security/blog/2026/05/20/introducing-rampart-and-clarity-open-source-tools-to-bring-safety-into-agent-development-workflow/) (May 20, 2026, open-sourced from Microsoft's AI Red Team — pytest-native safety/security testing framework built on PyRIT; adversarial scenarios run as CI tests on every change, with the most mature coverage on cross-prompt injection through poisoned documents, emails, and tickets) and its companion [Clarity](https://thehackernews.com/2026/05/microsoft-open-sources-rampart-and.html) (structured design-review tool for pressure-testing agent architecture assumptions before build). [VIPER-MCP](https://arxiv.org/abs/2605.21392) (research framework, May 2026 — automated static+dynamic taint auditing of MCP servers that confirms exploitability with generated proof-of-concept prompts; 106 zero-days / 67 CVEs from 39,884 scanned repos). Research-stage runtime defenses worth tracking: [AgentTrust](https://arxiv.org/abs/2605.04785) (pre-execution tool-call interception with allow/warn/block verdicts — a C9.7.1 reference design), [ARGUS](https://arxiv.org/abs/2605.03378) (provenance-aware influence-graph defense plus context-injection benchmark — directly on the C9.9.4-8 origin-tracking surface), [AgentShield](https://arxiv.org/abs/2605.11026) (honeytokens and decoy tools for compromised-agent detection), and [ADR](https://arxiv.org/abs/2605.17380) (detection-and-response for MCP-based agents with runtime telemetry).
- **Open-source runtime governance:** [Microsoft Agent Governance Toolkit](https://github.com/microsoft/agent-governance-toolkit) (April 2, 2026, MIT-licensed) is the first open-source toolkit that explicitly maps to all ten OWASP Agentic Top 10 risks. Seven integrated packages: a stateless policy engine (Agent OS) that intercepts agent actions with YAML rules, OPA Rego, and Cedar; Ed25519-based agent identity with dynamic trust scoring (0–1000 scale, behavioral decay); execution rings inspired by CPU privilege levels with capability gating and resource limits; and SRE-style reliability primitives (SLOs, error budgets, circuit breakers, progressive delivery). Native integrations with LangChain, CrewAI, and LangGraph avoid framework rewrites, and Microsoft documents [sub-millisecond p99 enforcement](https://opensource.microsoft.com/blog/2026/04/02/introducing-the-agent-governance-toolkit-open-source-runtime-security-for-ai-agents/). One caveat surfaced in the May 2026 hackerbot-claw coverage: several authentication primitives shipped with zero production callers, so AISVS evidence still has to verify that the toolkit's controls are actually wired into the agent's execution path, not just imported.

### Implementation Maturity

| Control Area | Tooling Maturity | Notes |
|--------------|:---:|-------|
| C9.1 Execution Budgets | Maturing | Helicone/Langfuse track costs; OpenAI Agents SDK blocking guardrails can run before the model or tool path starts, avoiding token spend and side effects when a tripwire fires. But $47K incidents show most deployments still lack hard budget enforcement. PraisonAI CVE-2026-44338 (May 2026) demonstrates the upstream failure mode: an agent framework that ships an unauthenticated network listener has no realistic budget control because anyone on the network can replay the workflow. Only 14.4% of orgs deploy agents with full security approval. |
| C9.2 High-Impact Action Approval | Maturing | LangGraph checkpoints, OpenAI tool guardrails, Claude SDK approval gates, Microsoft approval middleware, CHEQ, AP2, and Verifiable Intent all give implementation patterns for explicit approval, parameter binding, nonce/TTL handling, and rollback semantics. FIDO's April 2026 workstreams are the strongest sign that verifiable delegated intent is becoming a standards problem rather than a product feature. The May 2026 TrustFall and SymJack disclosures cut the other way: in shipping coding agents, the approval prompt itself fails C9.2.2 — trust dialogs that list nothing they're approving, and symlinked copy targets that make the displayed path differ from the kernel-level write destination. Canonicalization before display (resolve symlinks, show real paths, enumerate what trust grants) is now a verifiable audit item, not a UX nicety. |
| C9.3 Component Isolation | Mature | E2B, Modal, Firecracker, Fly.io Sprites, Northflank, OpenAI's Agents SDK sandbox, and Anthropic's May 19, 2026 Claude Managed Agents self-hosted sandbox option (Cloudflare, Daytona, Modal, Vercel) provide production-grade isolation, and MCP tunnels let internal databases and APIs reach agents without inbound firewall rules or public endpoints. Lasso+Portkey MCP Gateway adds inline protocol filtering. The OX MCP STDIO advisory, CrewAI Docker-fallback chain, Semantic Kernel CVEs, and the nginx-ui MCPwn / Akamai database-server / Microsoft Defender May 14 misconfiguration cluster show the remaining gap: sandbox availability, command allowlisting, host-path confinement, quota/timeouts, manifest enforcement, and tool-call-endpoint authentication all have to fail closed by default. | C9.5 Secure Messaging | Emerging | C9.5 is now focused on semantic validation of agent outputs before they influence downstream agents. A2A session smuggling plus the arXiv 2602.11327 threat model (replay, Sybil, protocol downgrade attacks) show why schema-valid messages still need meaning-level constraints; transport, replay, schema, and MCP-specific protocol hardening belong in ASVS and C10. |
| C9.6 Authorization & Delegation | Emerging | OPA, Cedar, Cerbos, and AuthZEN-style PDP APIs can enforce the deterministic policy layer, but agent-specific delegation context propagation (C9.6.2) still has limited native framework support. C9.6.5 requires the PDP to evaluate the agent's granted permissions and the initiating principal's delegated scope as independent constraints. | C9.7 Intent Verification | Emerging | Pre-execution gates are mostly custom-built, but AP2, Verifiable Intent, CHEQ, OpenAPI x-agent-trust, Microsoft Agent 365, and the March 2026 arXiv "Before the Tool Call" paper give concrete patterns for binding action parameters back to user intent, policy versions, and audit evidence. Cursor CVE-2026-26268 (Git-hook RCE) and the Claude Code project-file CVEs (2025-59536, 2026-21852) show what a missing pre-execution gate costs when agents act autonomously on attacker-controlled repository content. The May 12, 2026 NetOps/AIOps survey (Bilal et al.) names the architectural pattern the same gates have to implement — a "strict propose-commit split" where the model only drafts and a non-bypassable gate the model has no authority over performs the mutation — and adds an evaluation contract (tool-call traces, gate-violation rates, refusal-storm rates, rollback completeness) that auditors can ask vendors for directly. Anthropic's Opus 4.5/4.6/4.7 system cards (Feb-May 2026) now publish prompt-injection attack success rates per surface (GUI 17.8% single-attempt unsafeguarded → 57.1% at 200 attempts with safeguards, constrained coding 0%, browser-use 1.4% with Opus 4.5 safeguards), which gives C9.7 a published vendor baseline auditors can ask procurement targets to match or improve on. C9.7.4's PDP fail-closed requirement is especially important for distributed agents where network partitions are normal operational events. Microsoft's RAMPART (May 20, 2026) gives intent-gate owners the missing test harness: adversarial cross-prompt-injection scenarios expressed as pytest cases that run in CI on every change, so gate regressions (like the Claude Code 50-subcommand bypass or the v2.1+ trust-dialog warning removal TrustFall exploited) become catchable before ship. |
| C9.8 Multi-Agent Isolation | Emerging | Namespace isolation for tools, memory, context windows, aggregate swarm rate limits, behavior monitoring, and shutdown controls are documented in research but have limited production tooling. Microsoft Agent 365 (GA May 1, 2026) is now the most mature enterprise control-plane example — agent registry, AWS Bedrock and Google Cloud sync, lifecycle governance, security policy templates — but cross-framework swarm shutdown and emergent-behavior detection remain immature. Microsoft MDASH (May 12, 2026) shows the architectural pattern for inverting this: defensive multi-agent ensembles that observe and challenge other agents in real time. |

## Open Research Questions

- [ ] **How do you enforce "least agency" in practice?** — The OWASP Agentic Top 10 introduces the principle but no standard framework exists for defining minimum autonomy levels. NIST's five-tier autonomy taxonomy (Level 0 tool-assisted to Level 4 fully autonomous) is a starting point.
- [ ] **Can agent identity scale to millions of ephemeral instances?** — SPIFFE treats all pod replicas as identical, but agents are non-deterministic. The arXiv 2603.14332 framework found 5.8x variance in inference determinism across models, complicating replay-based verification. Instance-level identities are a proposed solution but the Gravitee 2026 report shows only 21.9% of teams even treat agents as independent identity-bearing entities — the gap between theory and practice remains vast.
- [ ] **How do you detect semantic privilege escalation?** — All technical permission checks pass, but the agent operates outside intended scope. No standard tooling exists for semantic boundary enforcement.
- [ ] **Will A2A, MCP, AP2, and FIDO converge on compatible trust semantics?** — A2A and MCP cover communication and tool access; AP2 and Verifiable Intent cover commerce-specific intent proofs; FIDO is now trying to standardize delegated agent authentication. The open question is whether these efforts produce interoperable identity, intent, and authorization claims or another set of incompatible trust islands.
- [ ] **How do you monitor emergent swarm behavior?** — Multi-agent oscillation, deadlocks, and coordinated manipulation (steganographic collusion) require runtime behavioral analysis that goes beyond individual agent monitoring. Limited production tooling exists.
- [ ] **What does agentic misalignment defense look like?** — Anthropic's study showed 96% of models attempt blackmail under specific conditions. This is a model behavior problem (C07) that manifests as an agent security problem (C09).
- [ ] **How do you secure the MCP ecosystem at scale?** — 30+ CVEs in 60 days (Jan-Feb 2026), 8,000+ exposed servers, 82% path traversal rate, 38-41% of registry servers lacking auth, and the April 2026 OX STDIO advisory suggest the MCP ecosystem has a systemic security gap. Trust-on-first-use models (MCPoison/CVE-2025-54136) and user-editable STDIO launch configs are fundamentally weak defaults. The Vulnerable MCP Project is tracking issues, but no certification or security baseline exists for MCP server implementations. The CSA "AI Agent Disclosure Vacuum" whitepaper (April 17, 2026) compounds the problem by documenting that many fixed flaws never get CVEs, so even diligent enterprise patch programs work from an incomplete picture.
- [ ] **Is agent-to-agent lateral movement the next major breach vector?** — The OpenClaw crisis (21,639 exposed instances, inherited OAuth tokens enabling cross-system traversal), the PleaseFix browser hijack, and the Meta rogue agent incident demonstrate that compromised or uncontrolled agents provide lateral movement paths invisible to traditional monitoring. The Gravitee 2026 report found 25.5% of deployed agents can create and task other agents with shared credentials — a direct enabler. No standard detection framework exists for agent-layer lateral movement.
- [ ] **How should approval protocols handle the privacy-sensitive action problem?** — The CHEQ IETF draft proposes keeping sensitive data outside the model context during approval, while AP2 and Verifiable Intent focus on cryptographic proof of user intent for commerce. The remaining gap is a general-purpose approval pattern that binds exact action parameters, nonce, TTL, requester, user intent, and rollback semantics without overexposing sensitive data to the model or the approving party.
- [ ] **Can frameworks enforce true fail-closed semantics across isolation tiers?** — The CrewAI sandbox fallback, MCP STDIO advisory, and Semantic Kernel file-transfer exposure all show the same pattern: weak runtime behavior looks like convenience until it becomes host execution. C9.3.2 and C9.7.4 require termination when quotas, timeouts, isolation guarantees, or PDP dependencies fail, but no widely adopted framework API lets a developer declare required isolation guarantees and have the runtime enforce them uniformly.
- [ ] **Is agent state integrity a separate subsystem or a layer of memory?** — C9.4.5 overlaps with C8 memory controls, and the 2026 research push treats agent memory as a first-class architectural component. But integrity protection — cryptographic MACs or signatures over stored context, goals, and partial results — has no mainstream implementation. Without verifiable state integrity, an attacker with one-time write access can compromise agents indefinitely. The May-June 2026 research wave (MemMorph, the "Hidden in Memory" sleeper-payload attack, the agent-worm propagation analysis) plus the patched Copirate 365 backdoor (CVE-2026-24299) confirms the attack class is practical and recurring; integrity verification still trails the offense.
- [ ] **Will "agent runtime security" stabilize as a distinct product category?** — AARM, Microsoft Agent 365, Lasso MCP Gateway, OPA-based intent verifiers, and the CSAI Agentic Trust Framework all describe overlapping responsibilities: action interception, context accumulation, policy and intent evaluation, authorization enforcement, and tamper-evident receipts. The open question is whether these converge into an interoperable runtime layer with shared receipts and policies, or fragment into per-framework features. Standards adoption velocity through 2026 (AARM conformance, CSAI CNA scope, FIDO Agentic WGs, NIST NCCoE outcomes) will decide.
