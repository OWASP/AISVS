# C02: User Input Validation

> **Source:** [`1.0/en/0x10-C02-Input-Validation.md`](https://github.com/OWASP/AISVS/blob/main/1.0/en/0x10-C02-Input-Validation.md)
> **Requirements:** 12 | **Sections:** 2

## Control Objective

Robust validation of all inputs is a first-line defense against some of the most damaging attacks on AI systems. Prompt injection attacks can override system instructions, leak sensitive data, or steer the model toward behavior that is not allowed. Unless dedicated filters and validation controls sit before model context, jailbreaks can still exploit shared context windows, encoding flexibility, and tool-connected workflows. In agentic and multi-step systems, tool outputs, retrieved documents, MCP server responses, memory reads/writes, CI/CD event context, logs, and sub-agent messages carry the same risks as direct user prompts, so the validation pipeline has to cover every path that can steer model behavior.

## Threat Landscape

Known attacks, real-world incidents, and threat vectors relevant to this chapter:

- **Direct prompt injection** — overriding system instructions via user input. Remains the #1 vulnerability in production LLM deployments, appearing in over 73% of assessed systems (OWASP LLM01:2025). Multi-turn jailbreak techniques now achieve 90%+ success rates against frontier models in under 60 seconds.
- **Indirect prompt injection** — injected instructions in retrieved content (RAG results, web pages, emails, tool outputs). Research demonstrates that just five carefully crafted documents can manipulate RAG-augmented AI responses 90% of the time.
- **Tool-output and trace injection** — attacker-controlled observations, code comments, issue text, document metadata, tool responses, or agent scratchpads can be interpreted as follow-up instructions unless separated and validated. OWASP's prompt-injection cheat sheet now calls out thought/observation injection, tool manipulation, and context poisoning as agent-specific attack patterns; Google DeepMind's Gemini defense report similarly frames untrusted tool data as a core risk for function-calling systems.
- **Log-borne prompt injection** — debugging and remediation agents increasingly consume cloud logs as if they were operational evidence. LogJack (April 2026) showed that log entries containing injected instructions could steer remediation agents into command execution; the benchmark reported `curl | bash`-style RCE on 6 of 8 tested models and poor detection when the same payloads were embedded in logs rather than sent directly.
- **Second-order prompt injection** — exploiting privilege hierarchies in multi-agent systems, where a low-privilege agent is tricked into escalating requests to a higher-privilege agent (demonstrated against ServiceNow Now Assist in 2025).
- **Unicode and encoding smuggling** — invisible characters (zero-width joiners, Unicode tags), homoglyphs, mixed-script payloads, Base64 encoding, mathematical symbol substitution, and "glitch tokens" that bypass production guardrails from major vendors while remaining interpretable by LLMs. ASCII smuggling attacks tested across multiple LLMs in September 2025 showed broad susceptibility.
- **Multi-modal injection** — instructions hidden in images (adversarial perturbations invisible to humans), audio (AudioJailbreak achieving 87-88% success even over speakers with room reverb, ACM CCS 2025), and documents. Cross-modal chaining (steganographic embedding + semantic manipulation) compounds attack effectiveness (Chain of Attack, CVPR 2025).
- **Cross-modal prompt injection** — CrossInject (ACM MM 2025) demonstrated that embedding aligned adversarial signals across both vision and text modalities boosts attack success by at least 30.1% over single-modality attacks. Embedding-space attacks that target only the vision encoder (e.g., CLIP) reduce attacker requirements — no direct LLM access needed.
- **Physical-environment injection** — CHAI (Command Hijacking against embodied AI) demonstrated physical-world prompt injection against embodied AI agents (January 2026, UC Santa Cruz). Related work showed typographic adversarial instructions on physical objects can hijack camera-equipped autonomous agents.
- **Crescendo and multi-turn attacks** — gradually escalating benign prompts toward harmful outputs, exploiting context window accumulation to shift model behavior over multiple turns.
- **Token smuggling** — bypassing input filters through non-standard encodings, emoji-based code payloads, and character obfuscation techniques that evade tokenizer-level safety checks.
- **Zero-click indirect injection** — a particularly dangerous variant where the victim never interacts with the malicious payload directly. EchoLeak (CVE-2025-32711, CVSS 9.3) demonstrated this against Microsoft 365 Copilot: a single poisoned email triggers data exfiltration when the user queries Copilot about unrelated topics. The payload executes in natural language space, rendering traditional defenses (antivirus, firewalls, static scanning) ineffective.
- **AI framework code injection** — vulnerabilities in AI development frameworks (Langflow, Semantic Kernel, LangChain, ModelScope ms-agent) are increasingly exploited as entry points. CVE-2026-33017 (Langflow, CVSS 9.8) was exploited within 20 hours of disclosure in March 2026 — attackers built working exploits from the advisory alone, with no PoC required. Microsoft's May 2026 Semantic Kernel research shows the same pattern inside agent frameworks: once untrusted language can influence tool parameters, prompt injection can become RCE, arbitrary file write, or credential exfiltration rather than only unsafe text output. The pattern repeats in ModelScope's `ms-agent` (CVE-2026-2256, Feb 2026), where a regex denylist on the Shell tool was bypassed through shell-syntax obfuscation reachable from agent-ingested documents, web pages, and tickets.
- **Agent-config-as-input** — repository-level settings consumed by coding agents are an unvalidated input surface in their own right. Check Point Research's January 2026 Claude Code disclosure (CVE-2025-59536 + CVE-2026-21852, fixed in v2.0.65) showed that a malicious repo's `.claude/settings.json` could set `ANTHROPIC_BASE_URL` to an attacker-controlled endpoint, and Claude Code would issue authenticated API calls — leaking Anthropic API keys — *before* the trust-on-first-use prompt ever fired. Any file the agent reads at startup (`devcontainer.json`, hook configs, MCP server lists, `pyproject.toml` plugin entries) is part of the C2.1.3 untrusted-input surface; the same logic that screens prompts must screen the agent's own configuration plane.
- **Chat-template injection** — ChatInject (ICLR 2026) showed that payloads formatted to mimic the model's native chat-template tokens (role markers, turn boundaries, tool-call envelopes) are accepted by the LLM as if they were genuine framework turns, including across multi-turn persuasion sequences. Plain-text-only input sanitizers miss this because the malicious string only becomes structurally meaningful once the chat-template renderer concatenates it; auditors should test that tool outputs, retrieved documents, and memory reads cannot inject literal template delimiters.
- **Coding-agent context injection** — "Comment and Control" (April 2026, A. Guan with Johns Hopkins) demonstrated that a single PR title, issue body, or HTML-commented payload was enough to make Claude Code Security Review, Gemini CLI Action, and GitHub Copilot Agent exfiltrate secrets from CI runners. Critically, the payload was hidden in HTML comments that GitHub renders invisibly in the UI but feeds verbatim to the agent — markup that hides content from human reviewers while remaining readable by the agent is now a dedicated attack surface. One affected vendor's published system card had already predicted this failure mode; the agent shipped to production without enforcement.
- **Agentic workflow injection in CI/CD** — 2026 research on GitHub Actions found that issue bodies, PR descriptions, comments, workflow logs, and agent outputs can become prompt-to-agent or prompt-to-script channels. TaintAWI analyzed 13,392 real-world agentic workflows and confirmed 496 exploitable workflow-injection cases, while GitInject exercised live GitHub workflows across four providers and found every default configuration susceptible to at least one attack class. For C2 reviewers, repository event context is now a first-class input surface, not operational background noise.
- **Browser-extension injection (ShadowPrompt pattern)** — attackers chain an overly permissive origin allow-list in a vendor's own browser extension with a DOM-based XSS in a third-party subdomain (e.g., a CAPTCHA widget) to deliver prompts that the LLM interprets as user-authored. The Claude Chrome extension flaw (disclosed Dec 26, 2025; patched v1.0.41 on Jan 15, 2026; Arkose CAPTCHA XSS fully patched Feb 19, 2026) showed that simply visiting a malicious page was enough to steal access tokens, read conversation history, and act on the victim's behalf. Critically, the attack subverts instruction hierarchy — the injected content appears indistinguishable from a legitimate user turn — which is the failure mode C2.1.6 targets.
- **URL-parameter prompt injection** — Reprompt showed that a legitimate assistant URL with a prefilled query parameter can become a one-click injection vector. Treat deep links, share links, callback URLs, and query parameters that populate agent prompts as untrusted input sources, not as user intent.
- **Patch-survives-attack pattern** — Capsule Security's "ShareLeak" (CVE-2026-21520, Microsoft Copilot Studio, disclosed Nov 24 2025, patched Jan 15 2026) was followed three months later by a near-identical exfiltration via a slightly mutated payload. The lesson is structural: patching the literal injection string only closes the specific syntax; any agent that combines untrusted input, sensitive data access, and an outbound communication tool remains exposed to the same *class* of attack. Action-level enforcement at the governance plane (outbound-domain allow-lists, pre-execution tool gating, recipient restrictions) is increasingly framed as a "kill switch" that is categorically different from a model-side safety filter, because it runs independently of whatever the model was instructed to do.
- **Encoding-obfuscated injection with financial impact** — the Grok/Bankrbot "Morse code heist" (May 4–8, 2026) showed encoding tricks producing real-world theft: an attacker first sent Grok's wallet a Bankr Club NFT to elevate it to transfer-capable status, then publicly replied with a Morse-code-encoded instruction. Grok helpfully decoded the Morse, tagged @bankrbot, and the bot executed a ~$150K–$200K token transfer. Root cause was the system's inability to distinguish public conversational data from executable commands — and the Morse encoding sailed past plaintext filters, making this a combined C2.1.2/C2.1.3 failure with direct monetary loss.
- **Text-to-SQL injection chains** — Langroid's SQLChatAgent (CVE-2026-25879, May 2026) let an LLM generate and execute SQL without validation; attacker-shaped input (direct, or indirect via returned database rows) could make the model emit `COPY ... FROM PROGRAM` and reach OS command execution on the database host. Fixed in 0.63.0 with a SELECT-only allow-list. Database rows are an input surface: content *returned* by a tool can steer the next generation.
- **AI middleware as an injection surface** — the LiteLLM proxy CVE cluster (May–June 2026) shows that the gateway layer fronting LLM traffic is itself an input-validation target: CVE-2026-42208 (CVSS 9.3, unauthenticated SQL injection via a crafted Authorization header, exploited in the wild from May 8) and CVE-2026-42271 (CVSS 8.7, command execution via MCP stdio test endpoints, exploited from June 8), plus CVE-2026-42203 (server-side template injection in the `/prompts/test` prompt-template endpoint yielding code execution in the proxy process). All affect 1.81.16–1.83.6; fixed in 1.83.7. Prompt templates and proxy admin surfaces deserve the same validation rigor as prompts themselves.
- **Promptware C2 ("Agent Commander")** — CSA's March 2026 research note documents Johann Rehberger's demonstration that three different agent platforms, each compromised via a different injection vector (document analysis, email, website visit), can be enrolled into a single command-and-control dashboard — with persistence via built-in heartbeat configurations and evasion via response-suppression flags. Injection is no longer just a one-shot exfiltration trick; it can establish durable remote control over agent fleets.
- **Hidden-state cross-modal attacks** — CrossMPI (arXiv:2605.16090, May 2026) optimizes image-only perturbations in the LVLM's hidden-state space, targeting the middle layers where multimodal fusion happens. The perturbation steers the model's interpretation of *both* visual and textual inputs without modifying the text at all, outperforming embedding-space baselines. AudioHijack (arXiv:2604.14604, April 2026) achieved 0.89–0.95 prompt-injection success across 13 audio-language models with context-agnostic, imperceptible auditory payloads, including hijacks of production voice agents into tool misuse.
- **The "Lethal Trifecta"** — systems that combine (1) access to private data, (2) exposure to untrusted tokens from external sources, and (3) an exfiltration vector (ability to make external requests) are inherently vulnerable to indirect injection chains. As of early 2026, most enterprise AI copilot deployments exhibit all three characteristics.
- **Scale of the problem** — a meta-analysis of 78 recent studies (2021–2026) found that attack success rates against state-of-the-art defenses exceed 85% when adaptive strategies are employed, and 73% of tested platforms fail to adequately enforce boundaries between agents, tools, and sessions. Wiz Research tracked a 340% YoY increase in documented prompt injection attempts in Q4 2025 and a 190% rise in successful attacks causing measurable data exfiltration. VentureBeat's April 2026 survey of 100 technical decision-makers found only **34.7%** of organizations have deployed dedicated prompt injection defenses — the remaining 65.3% either haven't purchased the tooling or cannot confirm they have. Google's April 2026 Common Crawl scan of the public web flagged a **32% relative increase** in malicious indirect-injection content between November 2025 and February 2026, including fully-specified PayPal transaction strings hidden invisibly in ordinary HTML and aimed at browsing agents with payment authority — a concrete "in-the-wild" baseline for C2.1.3 screening of retrieved content.

## Implementation Maturity Overview

A chapter-level view of tooling maturity and adoption status for C02 controls:

| Section | Area | Maturity | Notes |
|---------|------|:--------:|-------|
| C2.1 | Prompt Injection Defense | Medium-High | Commercial solutions (Lakera Guard, AWS Bedrock Guardrails) and open-source layers such as PromptGuard 2/AlignmentCheck improve coverage, but no tool guarantees complete prevention. Instruction hierarchy enforcement, tool-argument validation, and agent trace monitoring remain provider- and architecture-dependent. Pre-tokenization input normalization is part of this surface: Unicode normalization is straightforward, but statistical anomaly detection and adversarial training are research-grade, and zero-width character and homoglyph attacks still bypass production guardrails from major vendors. |
| C2.2 | Content & Policy Screening | Medium | Content classifiers (OpenAI Moderation, Perspective API, Lakera Guard) are production-ready and attribute-based per-user policy resolution is maturing, but multi-modal coverage lags: image and audio adversarial detection remains largely research-grade. CSA's March 2026 review recommends combining visual input screening, sandboxing, least-privilege tool access, behavioral monitoring, and HITL gates because typographic, steganographic, perturbation-based, and physical-world variants require different defenses. |

> **Overall:** As of May 2026, only ~20% of enterprises have mature AI governance models (Deloitte 2026 AI Report). Cisco's State of AI Security 2026 report found that while 83% of organizations plan to deploy agentic AI, only 29% feel ready to secure it. Input validation tooling is strongest for text-based prompt injection and weakest for multi-modal and adaptive scenarios. Notably, HiddenLayer researchers bypassed OpenAI's guardrails framework using straightforward techniques in October 2025, and Cisco found multi-turn prompt attacks achieve ~60% success rates on average (one model: 92.78%), underscoring that no single guardrail layer is sufficient. The agent-framework RCE disclosures in 2026 raise the maturity bar: auditors should now test whether untrusted prompts, retrieved content, or URL parameters can influence tool names, tool arguments, local file paths, shell commands, network destinations, or memory writes. The first independent validation of AI firewall vendors (SecureIQLab, 32 scenarios across input/output/retrieval layers, OWASP- and MITRE ATLAS-aligned) begins April 2026 with results expected at Black Hat USA 2026 — this should provide the first empirical, vendor-independent efficacy data for the market. Two June 2026 signals sharpen the picture: the AgentDyn benchmark (560 indirect-injection test cases including *benign* third-party instructions) found that almost all existing defenses are either insufficiently secure or suffer significant over-defense — blocking legitimate third-party content — so auditors should measure false-positive rates alongside detection; and both the Five Eyes agentic-AI guidance (May 1, 2026) and a formal impossibility analysis (arXiv:2605.17634) now treat prompt injection as a risk to be managed with layered detection, gating, and HITL rather than a problem any single control can solve.

Finds almost all existing defenses are either not secure enough or block legitimate third-party content. The successor to AgentDojo-style evaluation for C2.1.3 screening — test both detection and false-positive rates.
- Abdelnabi & Bagdasarian, "[AI Agents May Always Fall for Prompt Injections](https://arxiv.org/abs/2605.17634)" (May 2026) — formal argument from Contextual Integrity theory that an adversary can always construct a context making a blocked flow appear legitimate, while tightening norms blocks genuine flows; data-instruction separation can therefore never fully eliminate injection. Strong rationale for pairing C2.1.6 hierarchy enforcement with system-level gating (C9.2 high-impact action approval, egress allow-lists) rather than treating any model-side defense as sufficient.
- "[AgentSentry: Mitigating Indirect Prompt Injection in LLM Agents via Temporal Causal Diagnostics and Context Purification](https://arxiv.org/abs/2602.22724)" (Feb 2026) — treats multi-turn indirect injection as a temporal causal takeover at tool-return boundaries, then purifies context so the agent can continue with task-relevant evidence. Useful evidence pattern for C2.1.3 and C2.1.6: verify why a later tool call happened, not just whether the initial input looked suspicious.
- "[AttriGuard: Defeating Indirect Prompt Injection in LLM Agents via Causal Attribution of Tool Invocations](https://arxiv.org/abs/2603.10749)" (Mar 2026) — action-level causal attribution for tool calls; proposed tool invocations are replayed under an attenuated view of untrusted observations to check whether the call survives without attacker-controlled content. Maps directly to C2.1.6 hierarchy enforcement and C9 tool approval.
- "[LogJack: Indirect Prompt Injection Through Cloud Logs Against LLM Debugging Agents](https://arxiv.org/abs/2604.15368)" (Apr 2026) — cloud-log payload benchmark for debugging agents; demonstrates that guardrails detecting direct malicious prompts can miss the same payload once embedded in operational logs. Extends C2.1.3's "all inputs that could steer model behavior" language to observability data.
- "[ARGUS: Defending LLM Agents Against Context-Aware Prompt Injection](https://arxiv.org/abs/2605.03378)" (May 2026) — AgentLure benchmark plus provenance-aware decision auditing; ARGUS builds an influence graph showing how untrusted context propagates into agent decisions and blocks decisions not justified by trusted evidence. Strong design reference for provenance logging and multi-step verification.
- "[Demystifying and Detecting Agentic Workflow Injection Vulnerabilities in GitHub Actions](https://arxiv.org/abs/2605.07135)" (May 2026) — defines prompt-to-agent and prompt-to-script workflow injection patterns, analyzes 13,392 agentic workflows, and reports 496 confirmed exploitable cases. Treat GitHub event context, action outputs, and workflow logs as C2-controlled input surfaces.
- "[Measuring Real-World Prompt Injection Attacks in LLM-based Resume Screening](https://arxiv.org/abs/2605.28999)" (May 2026) — measurement study over roughly 200K resumes found about 1% contained hidden prompt injections, with prevalence rising over the prior one to two years and more than 90% avoiding explicit instruction phrasing. Good reminder that C2.1.3 screening should include weak, oblique, and hidden instructions, not only "ignore previous instructions" strings.
- "[GitInject: Real-World Prompt Injection Attacks in AI-Powered CI/CD Pipelines](https://arxiv.org/abs/2606.09935)" (June 2026) — live GitHub workflow benchmark across four providers documenting eleven attack classes, including config-file injection, credential exfiltration, judgment manipulation, and availability attacks. The main mitigations are workflow-level: reduce token permissions, isolate untrusted event context, and gate agent-triggering workflows by actor trust.
- "[CrossMPI: Cross-Modal Prompt Injection via Hidden-State Optimization](https://arxiv.org/abs/2605.16090)" (May 2026) — image-only perturbations optimized in the LVLM's hidden-state middle layers (where multimodal fusion happens) steer interpretation of both visual *and* textual inputs without touching the text. Raises the bar for C2.2 non-text screening: embedding-space and pixel-space detectors miss attacks that only manifest at fusion layers.
- "[AudioHijack: Hijacking Large Audio-Language Models via Context-Agnostic and Imperceptible Auditory Prompt Injection](https://arxiv.org/html/2604.14604v1)" (April 2026) — 0.89–0.95 prompt-injection success across 13 audio-language models (Qwen2-Audio, Kimi-Audio, GLM-4-Voice, Phi-4-Multimodal, etc.), including hijacks of production voice agents into tool misuse. Audio-side evidence for C2.2's Level 3 non-text screening — detection tooling lags the attack literature badly.
- [OWASP Top 10 for Agentic Applications (2026)](https://genai.owasp.org/resource/owasp-top-10-for-agentic-applications-for-2026/), published Dec 19, 2025 — companion framework to the LLM Top 10 that addresses agent-specific failure modes. Prompt Injection appears as **ASI02**, with Agent Goal Hijack (**ASI01**) leaning on "Aegis" prompt-injection detection as its primary control. Key architectural patterns include the "Intent Capsule" (a signed, immutable envelope binding the agent's original mandate to each execution cycle), a Zero-Trust Tooling model with strict schema validation before tool invocation, and the Least-Agency principle (agents get only the minimum autonomy required). These shape how C2.1 and C2.2 should be verified in agentic deployments.

## Community Notes

_Space for contributor observations, discussion, and context that doesn't fit elsewhere._

**Open questions worth tracking:**
- No input validation tool guarantees complete prompt injection prevention. Defense-in-depth (combining multiple layers) remains the only viable strategy. How should auditors assess "sufficient" defense depth? The HiddenLayer bypass of OpenAI's guardrails (October 2025) reinforces this — guardrail LLMs are themselves susceptible to the attacks they're meant to detect.
- Multi-modal adversarial detection tooling is still largely research-grade. Organizations deploying vision or audio models should treat C2.2 non-text screening requirements as aspirational L2/L3 targets until tooling matures.
- The October 2025 MITRE ATLAS update added 14 agentic-specific techniques — these may drive future requirement additions to C2.1 (indirect injection via agent chains) and C2.2 (MCP argument validation).
- As of March 2026, zero-click indirect injection (EchoLeak pattern) represents a step change in threat severity. Organizations using AI copilots with access to email, documents, and external communication should audit for the "Lethal Trifecta" (private data access + untrusted token exposure + exfiltration vector) and implement image/link fetching restrictions as an immediate mitigation.
- The CrowdStrike 2026 report's finding of prompt injection at 90+ organizations suggests this is no longer a theoretical or red-team-only concern — it's an active adversary technique at scale. System prompt extraction was the most common objective in Q4 2025, giving attackers role definitions, tool descriptions, and workflow logic.
- Langflow CVE-2026-33017 (exploited in 20 hours, March 2026) is a watershed moment for AI framework security — it demonstrates that vulnerabilities in AI pipeline tooling are now targeted with the same speed as traditional web application zero-days. Organizations running AI orchestration frameworks should treat them with the same patching urgency as internet-facing web servers.
- The SecureIQLab independent validation initiative (April 2026) will be the first empirical, vendor-neutral assessment of AI firewall efficacy. Until those results land, organizations should remain skeptical of vendor detection-rate claims and test products against their own threat models.
- Cross-modal attacks (CrossInject achieving +30.1% success) and embedding-space attacks that require only vision encoder access (not full LLM access) are lowering the bar for multimodal exploitation. C2.2 multi-modal controls are becoming more urgent, not less.
- The current testing bar should include every input path that can steer behavior: first-party user prompts, externally retrieved documents, web pages, email, tool outputs, MCP arguments, memory reads/writes, agent scratchpads, and extracted text from images or audio. A control that only scans text before the first model call leaves the most important 2026 agentic attack surfaces uncovered.
- The ShadowPrompt disclosure (Jan–Feb 2026) is a reminder that **even the vendor's own browser extension is part of the input surface area** that C2.1.3 covers. Origin allow-lists should use exact strings, not wildcards, and every iframe/postMessage path that can deliver content into an LLM context should be treated as a potential untrusted input vector. This also validates C2.1.6 hierarchy testing across tool-augmented and multi-step workflows — once injected content is indistinguishable from a user turn, hierarchy alone may not recover the original trust boundary.
- The C2.1.4 wording (**reject** over-length inputs rather than silently truncate) is a direct response to context-window flooding and many-shot jailbreaking. Anthropic's many-shot study dropped one measured attack success rate from 61% to 2% using pre-model prompt classification and modification, but requires the system to see the complete input's length and structure — silent truncation obscures both. Auditors should expect an explicit rejection for oversize prompts and test the exact token boundary rather than assume a particular HTTP status.
- The May 2026 Semantic Kernel write-up gives auditors a concrete C2.1.3/C2.1.6 test pattern: trace whether any model-controlled text can reach framework sinks such as filter expressions, file paths, shell commands, network destinations, plugin names, or schema fields. A guardrail that says "safe" before the model call is not enough if the agent runtime later passes attacker-controlled arguments into a powerful tool.
- Enterprise readiness lags sharply. McKinsey's 500-deployment survey reports average system downtime of 72 hours per successful prompt-injection incident, a 23% drop in customer-trust scores for affected organizations, and a $340K average incident-response cost. Recorded Future pegs 2025 global losses from prompt-injection attacks at ~$2.3B, with 67% of incidents targeting customer-service chatbots and AI-backed trading systems. Only 34.7% of surveyed organizations have dedicated prompt-injection defenses deployed (VentureBeat, April 2026).
- **Patch the class, not just the syntax.** The ShareLeak follow-up (Microsoft Copilot Studio, April 2026) is the cleanest demonstration to date that vendor patches against a specific injection string do not prevent re-exploitation through a mutated payload of the same shape. Any agent that combines untrusted input, sensitive data access, and an outbound communication path is structurally exposed. C2 reviews should pair input-side controls with action-level enforcement (outbound-domain allow-listing, recipient policy, tool-call pre-execution checks) implemented in a governance plane that runs independently of model instructions — what some practitioners now call a "kill switch" — rather than relying solely on output filters that can be steered by the same injection they are meant to block.
- **Coding agents are credential-bearing input sinks.** The Comment and Control finding (April 2026) means GitHub issue bodies, PR titles, and HTML-commented diff content must be treated as untrusted inputs to any agent running in CI. Reviewers should specifically test whether comments invisible in the rendered Markdown UI (HTML comments, hidden references, collapsed details blocks) are passed verbatim to the agent — and whether the agent's CI runner exposes secrets that the agent has a path to print, copy, or commit. The fact that one affected vendor's own system card had already documented this failure class is a reminder that *predicted-but-unmitigated* risks need enforcement, not just disclosure.
- **MITRE ATLAS v5.3.0 raises the bar for control mapping.** The January 2026 update added `AML.T0086 Exfiltration via AI Agent Tool Invocation` and `AML.T0110 AI Agent Tool Poisoning`. C2 controls should now be mapped to these techniques alongside the original `AML.T0051` LLM Prompt Injection entry, particularly for agentic deployments where the realized impact (data exfiltration, persistent tool poisoning) is more relevant than the injection vector itself.
- **Agent configuration is now an input vector.** The Claude Code CVE-2026-21852 / CVE-2025-59536 chain (Check Point, January 2026) makes it concrete: a malicious repo's `.claude/settings.json` ran authenticated API traffic to an attacker endpoint *before* the trust prompt fired, exfiltrating Anthropic API keys. Any per-repo file that an agent reads during initialization — settings, hook scripts, `devcontainer.json`, MCP server lists, plugin entries — should be in scope for C2.1.3 untrusted-input screening, and trust gates must precede *any* outbound network call (not run alongside one). Reviewers should test the precise order of "trust prompt vs first network call" rather than only confirming a prompt exists.
- **Instruction-hierarchy enforcement is a Level 2, best-effort control.** C2.1 keeps normalization, representation-smuggling mitigation, untrusted-input screening, length rejection, and charset allow-listing at Level 1; instruction hierarchy is C2.1.6 at Level 2, while reserved-token encoding is C2.1.7 at Level 2 and many-shot detection is C2.1.8 at Level 3. The May 2026 evidence supports treating hierarchy as a model-layer control that still needs system-level gating, not as a complete security boundary.
- **Over-defense is now measurable and it's bad.** AgentDyn's inclusion of benign third-party instructions exposed that many injection defenses "pass" by blocking legitimate retrieved content wholesale. A C2.1.3 verification that only counts blocked attacks rewards over-blocking; ask vendors for false-positive rates on benign third-party content, not just detection rates.
- **The gateway is part of the attack surface.** The actively exploited LiteLLM cluster (May–June 2026) and the Langroid SQL chain show that "input validation" failures increasingly live in the middleware between user and model — prompt-template endpoints, API-key verification paths, MCP test endpoints, and LLM-generated SQL. C2 reviews should inventory every component that touches prompt or tool-call content in transit, not just the model-facing screening layer.
- **Encodings keep winning.** The Grok/Bankrbot heist payload was ordinary Morse code — no Unicode trickery, no adversarial optimization — and it cleared plaintext filters because the model itself volunteered the decoding. C2.1.2 representation-smuggling controls need to account for the model as a *decoder* of obfuscated instructions: screening that runs only on the raw input misses content the model will happily de-obfuscate downstream.
- **Chat-template rendering is part of the input plane.** ChatInject (ICLR 2026) shows that string-level sanitizers run before template assembly miss payloads that only become structurally meaningful once the chat-template renderer concatenates them with role markers. C2.1 verification should include round-trip tests that re-screen the fully assembled, post-template prompt — not only the raw user/tool string — and should treat any retrieved content that contains literal template delimiters as adversarial regardless of surrounding context.
