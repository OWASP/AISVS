# C08: Memory, Embeddings & Vector Database Security

> **Source:** [`1.0/en/0x10-C08-Memory-Embeddings-and-Vector-Database.md`](https://github.com/OWASP/AISVS/blob/main/1.0/en/0x10-C08-Memory-Embeddings-and-Vector-Database.md)
> **Requirements:** 11 | **Sections:** 3

## Control Objective

Embeddings and vector stores act as semi-persistent and persistent "memory" for AI systems via Retrieval-Augmented Generation (RAG). This memory can become a high-risk data sink and data exfiltration path. This control family hardens memory pipelines and vector databases so that access is least-privilege, data is sanitized before vectorization, retention is explicit, and systems are resilient to embedding inversion, membership inference, and cross-tenant leakage.

## Threat Landscape

Known attacks, real-world incidents, and threat vectors relevant to this chapter:

- RAG poisoning — injecting malicious content into knowledge bases retrieved at inference time; PoisonedRAG (USENIX Security 2025) showed 5 documents achieve 90% attack success
- Embedding inversion attacks — reconstructing original text from embedding vectors; Vec2Text achieves up to 92% reconstruction of 32-token inputs; transfer attacks (ACL 2024) work without access to the original embedding model
- Cross-tenant data leakage through shared vector indices and metadata filter injection
- Memory persistence attacks — injecting instructions that persist across conversations; MINJA (NeurIPS 2025) achieves >95% injection success via query-only interaction
- Agent memory poisoning — OWASP ASI06 (2026) formally recognizes this as a top agentic risk; ZombieAgent (January 2026) showed persistent cross-session attacks via ChatGPT connectors
- Environment-injected agent memory poisoning — eTAMP (April 2026) showed that a single contaminated observation, such as a manipulated product page, can persist across sessions and activate on different websites without direct memory access
- Long-horizon agent attacks — AgentLAB (February 2026) includes memory poisoning as one of five adaptive attack families across 28 tool-enabled environments and 644 security test cases; one-shot defenses did not reliably transfer to long-horizon settings
- AI Recommendation Poisoning — Microsoft (February 2026) documented 50+ real-world examples across 31 companies and 14 industries
- PII accumulation in vector stores without adequate deletion mechanisms; embedding inversion makes this a concrete exfiltration path
- Membership inference — determining whether a specific document exists in an embedded corpus
- Feedback loop poisoning — agent outputs written back to memory create self-reinforcing hallucinations; A-MemGuard found LLM classifiers miss 66% of poisoned entries
- Metadata filter injection — manipulating vector query filters to bypass tenant isolation
- Single-document RAG poisoning — CorruptRAG (Jan 2026) showed a single optimized document achieves 98.2% attack success by pushing its vector representation close to target queries, far stealthier than multi-document approaches
- Knowledge graph RAG poisoning — KEPo (Mar 2026) inserts adversarial triples that complete misleading inference chains in structured knowledge graphs, reducing QA accuracy by up to 80%
- Corpus-adaptive poisoning — Semantic Chameleon (Mar 2026) tailors poison documents to the specific corpus composition, defeating static detection
- Universal embedding inversion — ZSinvert demonstrated zero-shot inversion across any embedding model without per-model training, eliminating the attacker's need for model-specific knowledge
- Vector collision attacks — crafting documents whose embeddings collide with high-value query vectors via gradient-based optimization; the 2025 Supabase Cursor incident demonstrated real-world exploitation via support ticket injection
- Vector worms — theoretical future variant where poisoned embeddings instruct models to re-embed and reintroduce poisoned data elsewhere, creating cascading contamination
- Delayed tool invocation (Rehberger, January 2025) — conditional instructions in chat context trigger memory writes when users respond with natural affirmations, bypassing Google Gemini's guardrails
- Data loader poisoning — the ACM AISec "Hidden Threat in Plain Text" study demonstrated 74.4% attack success across 357 scenarios using 19 stealthy injection techniques (font poisoning, homoglyphs, zero-size fonts, out-of-bound text) against popular RAG data loaders including LangChain, LlamaIndex, and Docling; validated on black-box services like NotebookLM and OpenAI Assistants
- Provider-side RAG recommender poisoning — subtle token edits in item descriptions can promote or demote targeted items in embedding-based recommendation pipelines while staying under naive text-difference and semantic-similarity checks
- Training-free embedding inversion — Zero2Text (Feb 2026) uses recursive online alignment with LLM priors to invert embeddings without any training, achieving 1.8x ROUGE-L and 6.4x BLEU-2 improvements over baselines against OpenAI embeddings in black-box settings
- Zero-click exfiltration — EchoLeak demonstrated corporate data exfiltration from Microsoft 365 Copilot via crafted emails
- Centrality-driven hubness poisoning — Black-Hole Attack (April 2026) exploits a fundamental geometric defect of high-dimensional embedding spaces: vectors placed at or near the global or cluster centroid become disproportionate nearest neighbors. A handful of malicious vectors seeded near these centroids surface in up to 99.85% of top-10 results with no knowledge of user queries, and existing hubness-mitigation heuristics either crater retrieval accuracy or provide only partial protection
- Few-shot cross-model inversion — ALGEN (ACL 2025) reaches partial inversion from a single leaked (text, embedding) pair and near-optimal reconstruction with 1k samples by aligning the victim embedding space to an attacker-controlled encoder in one optimization step; transfers across domains and languages, and the authors show standard defenses (noise, quantization, DP) do not defeat it
- Compound query + corpus RAG attack — PIDP-Attack (March 2026) bolts a universal query-path suffix onto traditional passage poisoning, reaching 98.125% attack success on Natural Questions, HotpotQA and MS-MARCO across eight LLMs without needing any foreknowledge of user queries
- Agentic runtime supply-chain poisoning — SOK work in April 2026 frames persistent memory poisoning as an inference-time data supply-chain attack and describes a Viral Agent Loop where agents become carriers for self-propagating payloads without code-level vulnerabilities
- Agent-facing vector database write surfaces — vector stores exposed through MCP or similar agent protocols shift from passive retrieval infrastructure to active long-term memory; a compromised tool session can become a privileged memory-write path unless write authorization, provenance, and tenant scope are enforced before storage
- Vector database control-plane exposure — Milvus fixed CVE-2026-26190 in February 2026, a critical authentication bypass on the metrics port that could expose REST API and sensitive operations when deployed with weak network isolation. The Milvus 2.6.x line then continued shipping dependency-chain security releases through April and May 2026: v2.6.14 (April 7, 2026) upgraded Go to 1.25.8 to address the critical CVE-2025-68121 plus CVE-2026-27142 and CVE-2026-25679; v2.6.15 (April 24, 2026) upgraded `go-jose/v4` to v4.1.4 and refreshed `gnupg` in the GPU image to remediate CVE-2026-34986 and CVE-2025-68973; v2.6.16 (May 14, 2026) upgraded `lz4_flex` to 0.11.6 in the Tantivy binding to fix CVE-2026-32829. The cadence highlights that vector engines now inherit a steady stream of upstream Go, crypto, and storage-codec CVEs that have to be tracked alongside vector-store-specific advisories
- Experience-replay memory poisoning — MemoryGraft (arXiv:2512.16962, December 2025) targets the semantic imitation heuristic in agents that retrieve "successful" prior experiences; benign-seeming artifacts captured during execution are stored, then surface as malicious procedure templates on semantically similar future tasks. On MetaGPT's DataInterpreter with GPT-4o, a small number of poisoned entries accounted for up to 47.9% of retrievals on benign workloads, producing durable behavioral drift that conventional prompt and RAG defenses miss
- Architecture-dependent poisoning susceptibility — "Architecture Matters" (arXiv:2605.05632, May 2026) showed that under CorruptRAG-AK adversarial knowledge-base poisoning, attack success ranges from 81.9% on vanilla RAG to 24.4% on Recursive Language Models across 921 Natural Questions QA pairs — a 58-point spread on architectures that score comparably (~92%) on clean inputs. Agentic RAG and MADAM-RAG land between, making architectural choice a more powerful lever than any single ingestion-time filter
- Language-aware inversion under DP — BeamClean (arXiv:2505.13758, May 2025) jointly estimates the noise parameters and decodes token sequences with a language-model prior; against Laplacian and Gaussian obfuscation it achieves 77% recovery versus 17% for naive distance-based baselines at DP ε≈9.6, eroding the safety margin previously assumed for noise-only embedding privacy defenses
- Single-document knowledge dominance — One Shot Dominance (arXiv:2505.11548, May 2025) showed that a single injected document can dominate retrieval ranking and downstream answers for targeted queries, complementing CorruptRAG with a different optimization pathway and confirming that the 1-document poisoning regime is now a robust attack class rather than a single result
- Bias-amplification poisoning — "Bias Amplification in RAG" (arXiv:2506.11415, June 2025) frames poisoning as steering the retrieval geometry via subspace projection rather than crafting individual high-similarity documents, letting attackers nudge LLM outputs along a chosen bias axis without obvious factual contradictions
- RAG-triggered serialization injection — CVE-2025-68664 ("LangGrinch", CVSS 9.3, disclosed December 2025) made LangChain's `dumps()` / `dumpd()` re-hydrate dictionaries whose `lc` marker had been controlled via LLM response fields (`additional_kwargs`, `response_metadata`). A poisoned RAG document that nudges the model to emit such metadata can chain into class instantiation, Jinja2 template execution, and env-var exfiltration — leaking cloud, model, and vector-store credentials. Fixed in `langchain-core` 0.3.81 and 1.2.5 with allowlisted deserialization and `secrets_from_env=False` by default. Disclosed alongside CVE-2026-34070 (LangChain path traversal via prompt template loader, CVSS 7.5, fixed in `langchain-core` ≥ 1.2.22) and CVE-2025-67644 (LangGraph SQLite checkpoint SQL injection through metadata filter keys, fixed in `langgraph-checkpoint-sqlite` 3.0.1)
- Chroma MCP server SQL injection — April 2026, unpatched: eight of the Chroma official MCP server's thirteen tools (62%) pass collection-name parameters unsanitized into SQLite queries, enabling a prompt-injection → SQL-injection chain through agent tool calls. Combined with "ChromaToast" (CVE-2026-45829, April 2026) — a pre-authentication remote code execution vulnerability — Chroma deployments exposed through agent protocols become both a writeable memory surface and a code-execution surface
- Self-vs-injected experience confusion — agents that retrieve and replay their own past trajectories cannot, by default, distinguish records they themselves produced from artifacts injected by tool output, observation, or environment, which is the core invariant MemoryGraft and AgentPoison exploit and is the focus of the cryptographic provenance attestation work emerging in early 2026
- Vector store query-language injection — Spring AI's April 28, 2026 CVE bundle showed that filter expressions and document IDs flowing into vector store query languages reproduce the SQL/LDAP-style injection patterns from relational systems. CVE-2026-40967 (CVSS 8.6) covers unescaped keys/values in the `FilterExpressionConverter` implementations across multiple Spring AI VectorStore backends, CVE-2026-40978 covers SQL injection in `CosmosDBVectorStore.doDelete()` via crafted document IDs, and CVE-2026-40966 covers cross-tenant memory exfiltration where conversation IDs are interpolated into filter expressions without isolation, letting an attacker read chat history and long-term memory belonging to other conversations. Fixed in Spring AI 1.0.6 and 1.1.5. The same bundle also disclosed CVE-2026-40979 (ONNX model cache in world-writable `/tmp` with predictable paths — model-substitution surface for any agent that embeds with cached models) and CVE-2026-40980 (PDF-parsing memory exhaustion in `ForkPDFLayoutTextStripper`, a denial-of-service path for ingestion pipelines that accept attacker-supplied PDFs)
- Filter-expression code execution in agent vector stores — CVE-2026-26030 (CVSS 9.8, disclosed May 7, 2026) in Microsoft Semantic Kernel's Python SDK `InMemoryVectorStore` builds the default filter expression as a Python lambda and executes it via `eval()`. Attacker-controlled vector-store fields (e.g., the `city` field used in Microsoft's own samples) break out of the lambda string context and run arbitrary Python in the agent process. Any RAG document, agent memory write, or tool output that lands such a field in the vector store can lift prompt injection into remote code execution against the agent host. Fixed in `semantic-kernel` 1.39.4 by blocking dangerous attribute names in filter expressions; the advisory explicitly recommends not using `InMemoryVectorStore` in production until patched. Disclosed alongside CVE-2026-25592 (.NET SDK, `SessionsPythonPlugin` exposes a sandbox-to-host file download as a kernel function the model can call to write attacker-controlled files outside the Azure Container Apps sandbox; fixed in `Microsoft.SemanticKernel` 1.71.0)
- Conversational trojan memory implants — MemPoison (arXiv:2605.29960, May 28, 2026) plants triggerable backdoors in an agent's long-term memory purely through dialogue, using semantic bridges to bind trigger and payload, "entity masquerading" to survive the agent's own memory-rewriting and summarization step, and embedding-level optimization for stealth; attack success reaches 0.95 across multiple agent domains and explicitly bypasses selective memory-write filters — the consolidation step that many memory architectures treat as a defense is itself the carrier
- Sleeper memory poisoning — "Hidden in Memory" (arXiv:2605.15338, May 2026) shows a single exposure to a manipulated document or webpage plants fabricated "sleeper" memories that lie dormant across conversations: poisoned memories were written in 99.8% of trials on one frontier model and 95% on another, and once retrieved induced attacker-intended actions in 60–89% of evaluations even after the malicious context was long gone — a one-to-many compromise where one ingestion corrupts many later sessions
- Exam-style RAG membership inference — E-MIA (arXiv:2605.00955, May 2026) converts verifiable details of a target document into natural-looking quiz queries (four question types) to decide whether the document is in a RAG knowledge base, improving member/non-member separation while remaining hard to distinguish from benign traffic because the probes carry no perplexity anomalies that retrieval-side filters could flag
- Production copilot memory hijack — CVE-2026-24299 ("Copirate 365," disclosed October 2025, patched March 5, 2026, public writeup May 2026): prompt injection drives Microsoft 365 Copilot's `record_memory` tool — enabled by default enterprise-wide and, per Microsoft's own documentation, without audit logs — to plant persistent malicious memories; chained with CSS/`@font-face` HTML-preview exfiltration ("SpAIware") this becomes a persistent backdoor leaking data in later sessions. A textbook real-world failure of memory write authorization (8.2.4) with evidence and alerting falling back to C12 monitoring controls
- Developer-tool memory persistence — Cisco's MemoryTrap research (OWASP GenAI blog, May 13, 2026, by the ASI06 entry lead) showed malicious content injected via a routine developer workflow persisted across coding-agent sessions because the agent treated memory, hooks, and config files as trusted state, turning a one-time compromise into persistent prompt injection
- Multi-modal RAG knowledge poisoning — M³Att (arXiv:2605.10253, May 2026) extends knowledge-base poisoning to medical multi-modal RAG, a reminder that 8.2.2's crafted-content screening applies to image and mixed-modality corpora, not just text
- Epistemic Bias Injection (EBI) — Wu & Saxena (arXiv:2512.00804, November 2025, revised March 2026) showed that adversaries can crowd out alternative viewpoints by inserting factually correct but ideologically slanted passages into the RAG corpus. Because the passages are linguistically coherent and truthful, fact-checking and toxic-content filters do not flag them, and the attack evaded existing retrieval-based sanitization defenses in the authors' evaluation. They propose BiasDef, a lightweight prototype defense that computes a geometric epistemic-bias metric directly on retrieved-passage embeddings and substantially reduces adversarial retrieval and downstream bias. This complements the per-document optimization of CorruptRAG and the subspace-projection attack of Bias Amplification in RAG: EBI's signal is the *distribution* of viewpoints among truthful passages, not similarity to any single query vector
- Multi-session memory poisoning — SMSR (arXiv:2606.12703, June 2026) formalizes runtime poisoning of persistent RAG-agent memory as a separate threat from static-corpus RAG poisoning. The key result for 8.2.4 is that provenance-free retrieval filters cannot give a useful worst-case certificate; the proposed HMAC-signed memory plus randomized ablation pattern cuts unsigned attack success from 93-100% to 0% and reduces query-only poisoning from 65.3% to 5.3% in the authors' live-agent evaluation
- Agent-memory data-model gap — "Is Agent Memory a Database?" (arXiv:2605.26252, May 2026) argues that long-term memory correctness is about state trajectory, not isolated vector records. The paper's Governed Evolving Memory framing maps cleanly to C08's ingestion, revision, forgetting, and retrieval obligations: 8.2.4 write authorization, 8.2.5 contradiction handling, and 8.3.1-8.3.3 expiry, reset, and quarantine

## Open Research Questions

- [ ] How do you enforce RBAC in vector databases that don't natively support it? (Qdrant and Weaviate now offer native solutions; Chroma and FAISS still lack RBAC)
- [x] What's the practical risk of embedding inversion for modern high-dimensional embeddings? — Partially answered: Vec2Text achieves 92% reconstruction on 32-token inputs; OWASP LLM08:2025 estimates 50-70% word recovery; ACM RecSys 2025 reproducibility study provides nuanced benchmarks per model. Transfer attacks (ACL 2024) mean attackers do not need the original model.
- [ ] How should memory expiry policies differ for conversation memory vs. knowledge base memory? Especially given MINJA's temporal decoupling where poison planted today executes weeks later.
- [ ] What sanitization is adequate for content entering RAG pipelines? Sunil et al. (2025) showed no single signal is sufficient — composite trust scoring needed. A-MemGuard found LLM classifiers miss 66% of poisoned entries. CorruptRAG (Jan 2026) raised the bar further: a single optimized document achieves 98.2% success, and Semantic Chameleon (Mar 2026) showed corpus-adaptive poisoning defeats static detectors. SDAG (Feb 2026) offers a promising inference-time defense via sparse attention.
- [ ] Can homomorphic encryption or encrypted search schemes enable similarity search on encrypted vectors at practical latency?
- [x] What is the optimal noise calibration for embedding privacy that preserves retrieval utility above acceptable thresholds? — Partially answered: SPARSE (Feb 2026) uses Mahalanobis mechanism with elliptical noise calibrated by dimension sensitivity. DPPN (2025) perturbs only privacy-sensitive neurons, reducing leakage 5-78% while improving utility 14-40%.
- [ ] How should contradiction detection scale for large, rapidly-evolving knowledge bases? Microsoft's Feb 2026 report shows 50+ real-world slow-burn poisoning campaigns, increasing urgency.
- [ ] How can write-ahead validation effectively prevent MINJA-style query-only memory injection while maintaining agent usability?
- [ ] What architectural patterns best defend against "vector worms" — self-propagating poisoned embeddings that instruct models to re-embed and spread contamination?
- [ ] How should memory quarantine and forensic investigation workflows be standardized across different vector database backends?
- [ ] How effective are inference-time defenses (e.g., SDAG sparse attention, cross-encoder re-ranking) compared to ingestion-time defenses against single-document attacks like CorruptRAG? Can they be layered effectively? Early evidence suggests hybrid BM25+vector retrieval eliminates gradient-guided attacks but joint sparse+dense optimization circumvents at 20–44%.
- [ ] As of March 2026, both ZSinvert and Zero2Text enable training-free embedding inversion across any model in black-box settings — what minimum privacy transform (SPARSE, DPPN, Eguard, or combinations) is needed to stay ahead of model-agnostic inversion?
- [ ] How should ingestion pipelines defend against format-level poisoning (font poisoning, homoglyphs, zero-size fonts) that bypass all tested data loaders at 74.4–100% ASR? Is format-aware sanitization per file type (DOCX, HTML, PDF) the only viable approach, or can embedding-level defenses catch these?
- [ ] Can RAGuard's adversarial retriever training generalize to unseen poisoning strategies, or does it require continuous retraining as new attack methods (e.g., Semantic Chameleon's corpus-adaptive approach) emerge?
- [ ] As of April 2026, the Black-Hole Attack shows centrality-driven hubness is a geometric property of high-dimensional embedding spaces, not a bug in any one model. Can retrievers be engineered (hub-aware indexing, isotropy-regularized embeddings, NeighborRetr-style rebalancing) to close this class of attack without trading away retrieval accuracy? HubScan detects it post-hoc but does not eliminate it at the index level.
- [ ] Ingestion-only sanitization assumes attackers can't shape the query path. PIDP-Attack (March 2026) breaks that assumption with a universal query suffix. What architectural patterns jointly defend corpus and query paths without requiring users to accept measurable reductions in retrieval recall?
- [ ] ALGEN (ACL 2025) demonstrated that one leaked (text, embedding) pair bootstraps cross-model inversion and defeats the standard privacy transforms. What minimum combination of privacy-preserving embeddings (SPARSE, DPPN, Eguard) plus operational controls (limiting embedding egress, rotating embedding models) keeps inversion risk acceptable for regulated data?
- [ ] How should Agent Memory Guard-style cryptographic baselines (SHA-256 hashes, snapshot + rollback, declarative YAML policies) scale to continuously-written conversational memory and large RAG corpora without creating excessive review or storage overhead?
- [ ] How should memory write gates detect environment-injected poisoning like eTAMP when the malicious payload arrives as ordinary website content rather than as a direct memory write or explicit prompt injection?
- [ ] Can Agent-BOM-style graph audit trails be made lightweight enough for production agents while still preserving enough semantic state to reconstruct cross-session memory poisoning and tool misuse?
- [ ] How should MCP-exposed vector databases constrain agent read/write operations so that a compromised tool session cannot silently create trusted long-term memory outside tenant, user, or task scope?
- [ ] Can zero-trust runtime controls and cryptographic provenance break Viral Agent Loop-style propagation without making ordinary agent memory sharing unusable?
- [ ] How should agents distinguish self-generated experiences from externally injected procedural artifacts when "successful" demonstrations can be implanted via MemoryGraft (Dec 2025) and surface only on semantically similar future tasks? Can MemoryGraft's proposed Cryptographic Provenance Attestation and Constitutional Consistency Reranking be deployed with bounded key-management, enclave, and latency overhead on production agents with high write volumes?
- [ ] "Architecture Matters" (May 2026) shows Recursive Language Models cut CorruptRAG-AK attack success from 81.9% to 24.4% on Natural Questions versus vanilla RAG with comparable clean accuracy. What is the compute, latency, and cost trade-off of moving production retrieval from vanilla or agentic RAG to RLM-style architectures, and can the gains be reproduced on long, multi-turn agent workloads?
- [ ] BeamClean (May 2025) achieves 77% token recovery at DP ε≈9.6 against Laplacian and Gaussian obfuscation by jointly estimating noise parameters with a language-model prior. Can input-dependent privacy transforms (SPARSE, DPPN, Eguard) survive against jointly estimating attackers, or do regulated workloads now need encrypted inference or trusted-execution-environment retrieval rather than noise-based defenses alone?
- [ ] The Mnemonic Sovereignty survey (arXiv:2604.16548, April 2026) identifies nine governance primitives — write authorization, read authorization, integrity, confidentiality, availability, forget, audit, recovery, and benign-persistence — and finds no published architecture covers all of them. What is the minimum coherent baseline (and which existing tools cover which primitives) for an enterprise vector store plus agent memory deployment?
- [ ] How should organizations integrate LangChain's CVE-2025-68664 (LangGrinch) deserialization, CVE-2026-34070 (path traversal), and CVE-2025-67644 (LangGraph SQLite SQL injection) into routine RAG infrastructure hygiene — for example, version pinning, runtime allowlists for `load()`, and automated scanning of poisoned RAG documents that target `additional_kwargs` and `response_metadata` fields?
- [ ] How should vector databases exposed through MCP or A2A tool protocols be hardened so that prompt-injection-driven SQL injection (Chroma MCP, April 2026) and pre-auth RCE (ChromaToast, CVE-2026-45829) cannot escalate to long-term memory corruption or credential theft? Is a separate "tool-safe" deployment profile (parameter sanitization, scoped credentials, isolated indices) needed for any vector store reachable from an agent runtime?
- [ ] The Spring AI April 28, 2026 bundle (CVE-2026-40967 vector-store `FilterExpression` injection, CVE-2026-40978 `CosmosDBVectorStore` SQL injection, CVE-2026-40966 cross-tenant memory exfiltration via conversation IDs) and the May 7, 2026 Microsoft Semantic Kernel `InMemoryVectorStore` `eval()` RCE (CVE-2026-26030) collectively reframe vector store query languages as a first-class injection sink. What is the right canonical defense pattern — typed query builders, parameterized filter APIs, deny-by-default attribute allowlists, fuzz harnesses against `FilterExpressionConverter` implementations — and which orchestration frameworks already implement which pieces?
- [ ] MemPoison (May 2026) shows payloads engineered to survive memory rewriting and summarization — the consolidation step many architectures implicitly trust. Should memory consolidation be modeled as an untrusted transformation with its own pre- and post-write validation, and can provenance survive a rewrite that legitimately changes the text?
- [ ] Sleeper memory poisoning (arXiv:2605.15338) achieves near-perfect write rates because the poisoned content looks benign at ingestion and only misbehaves when semantically triggered later. What retrieval-time or periodic re-screening cadence catches dormant memories that write-time gates (8.2.2, 8.2.4) structurally cannot, and how does that interact with 8.3.1 expiry windows?
- [ ] E-MIA-style exam queries (May 2026) make document-level membership inference look like benign traffic. Can C12 retrieval logs plus C8.1 scope evidence and poisoning-anomaly monitoring be tuned to flag quiz-pattern probing without drowning in false positives from legitimate factual Q&A workloads?
- [ ] Epistemic Bias Injection (Wu & Saxena, arXiv:2512.00804) bypasses fact-checking and toxicity filters because the injected passages are truthful. BiasDef's geometric metric is a promising direction, but how do you set epistemic-bias thresholds for contested topics without suppressing legitimate viewpoint diversity, and can the metric survive adaptive attackers who optimize against it?
- [ ] Provably Secure RAG (SAG, arXiv:2508.01084, August 2025) gives C08 a cryptographic regression target via pre-storage encrypted documents and embeddings plus retrieval-time verification, but the original evaluation does not characterize end-to-end retrieval latency at production index sizes, key management at multi-tenant scale, or compatibility with ANN indexes (HNSW, IVF-PQ, ScaNN) under deletion and re-indexing. What is the smallest viable deployment shape that preserves the formal guarantees without falling back to plaintext for performance?
- [ ] How should production RAG threat models adopt the "Securing RAG" (arXiv:2604.08304, April 2026) abstraction of six stages, three trust boundaries, and four security surfaces — and the formal threat model in arXiv:2509.20324 — so that audits cleanly distinguish RAG-introduced risks from inherited LLM risks instead of conflating them with prompt-injection and model-leakage findings?
- [ ] Vector engines now inherit a steady upstream-dependency CVE stream (Milvus v2.6.14–v2.6.16 patched CVE-2025-68121, CVE-2026-27142, CVE-2026-25679, CVE-2026-34986, CVE-2025-68973, CVE-2026-32829 across Go, `go-jose/v4`, `gnupg`, and `lz4_flex` in roughly six weeks). What SBOM, exposure-window, and forced-upgrade SLA looks reasonable for managed and self-hosted vector stores so that downstream RAG and agent platforms can rely on a predictable patch cadence rather than ad hoc release tracking?
