# C11: Adversarial Robustness & Attack Resistance

> **Requirements:** 17 | **Sections:** 4

## Control Objective

Ensure that AI systems remain reliable, privacy-preserving, and abuse-resistant when facing evasion, inference, extraction, or poisoning attacks. These controls cover model alignment testing, adversarial hardening, privacy attack resistance, model theft deterrence, and security adaptation for autonomous agents.

## Threat Landscape

Known attacks, real-world incidents, and threat vectors relevant to this chapter:

- **Adversarial evasion in production** — Changing just 13 bytes of a malware sample evaded Google's Magika classifier in 90% of attempts (October 2025). The Cylance universal bypass (AML.CS0003) demonstrated 100% evasion by appending benign strings. ML-based malware detectors face 65.6% evasion rates; DL-based detectors up to 99%. The CrowdStrike 2026 Global Threat Report found 82% of detections were malware-free, with adversaries using valid credentials and trusted identity flows to evade AI-based detection entirely.
- **AI-assisted exploit discovery and autonomous malware** — On May 11, 2026, Google Threat Intelligence Group reported a criminal campaign using a likely AI-developed zero-day exploit against a Python-based web administration tool to bypass 2FA, with planned mass exploitation disrupted through responsible disclosure. The same report describes PRC- and DPRK-linked interest in model-assisted vulnerability research, PROMPTSPY using a model to interpret Android UI state and generate structured actions, and agentic tools such as Hexstrike and Strix used for autonomous reconnaissance and validation. For C11, this pushes adaptive testing beyond crafted prompts: teams need adversarial evaluation for model-assisted exploit development, self-modifying malware, and agent runtimes that can turn UI, code, and tool state into actions.
- **Autonomous LLM-agent intrusions in the wild (Sysdig/Marimo)** — On May 30, 2026, Sysdig's Threat Research Team disclosed the first documented in-the-wild intrusion whose post-exploitation phase was driven by an autonomous LLM agent rather than a human with AI assistance. Initial access came through CVE-2026-39987, a pre-authentication RCE in Marimo notebooks' `/terminal/ws` WebSocket endpoint, exploited 9 hours 41 minutes after public disclosure — faster than most human N-day weaponization. The agent extracted cloud credentials, replayed them through a fanned-out egress pool (12 API calls across 11 IPs in 22 seconds) to pull an SSH key from AWS Secrets Manager, pivoted through a bastion, and improvised a full PostgreSQL schema-plus-contents dump in under two minutes with no prior knowledge of the target schema. Sysdig's framing matters for C11.4: the attacker's constraint is now inference budget, not playbook authorship, so detection thresholds and runtime anomaly monitoring must assume machine-speed, schema-agnostic adversaries. CVE-2026-39987 is on CISA's KEV catalog.
- **Prompt-to-shell RCE in agent frameworks (Semantic Kernel)** — Microsoft's May 7, 2026 disclosure showed a single crafted prompt launching arbitrary code on the host running a Semantic Kernel agent: CVE-2026-26030 chains prompt injection through unsafe string interpolation into a Python `eval()` in the in-memory vector store (blocklist bypass via class-hierarchy traversal to load `os`), and CVE-2026-25592 escapes the SessionsPythonPlugin sandbox by chaining `ExecuteCode` with an unvalidated `DownloadFileAsync` write into host startup folders. Fixed in Python semantic-kernel 1.39.4 and .NET SDK 1.71.0. For agent runtime defense this collapses the distance between "adversarial input" and "host compromise" — adversarial robustness testing for agents must include the framework's own tool plumbing, not just model behavior.
- **Formal limits of prompt-injection defense** — "AI Agents May Always Fall for Prompt Injections" (Abdelnabi & Bagdasarian, arXiv 2605.17634, May 17, 2026) reframes prompt injection through Contextual Integrity theory and argues an impossibility result: any fixed "never do X" rule blocks legitimate flows, while any "allow X" rule admits attacks that construct a context making X appear appropriate. Empirically, prompt-injection classifiers performed near chance (AUROC 0.43–0.59) when attacks operated through contextual manipulation rather than recognizable injection vocabulary. For C11.4 the implication is architectural: detection-layer defenses bound risk but cannot eliminate it, so deterministic privilege separation (CaMeL-style), least-agency tool scoping, and human gates on high-impact actions carry the residual load.
- **Frontier-model autonomous offense (Claude Mythos Preview)** — Anthropic disclosed Claude Mythos Preview on April 7, 2026, alongside Project Glasswing (up to $100M in usage credits to AWS, Apple, Broadcom, Cisco, CrowdStrike, Google, JPMorganChase, the Linux Foundation, Microsoft, NVIDIA, Palo Alto Networks, and 40+ defenders, plus $4M of open-source security grants). UK AISI's May 2026 evaluation reported the model completes expert-level cyber tasks 73% of the time, succeeds end-to-end on "The Last Ones" 32-step corporate attack range in 3/10 runs (avg 22/32 steps), and reproduces exploits on the first attempt in over 83% of cases — including a 27-year-old OpenBSD bug and a 16-year-old FFmpeg flaw. C11 evidence now has to assume an attacker can hire a Mythos-class model to chain reconnaissance, exploit synthesis, and post-exploitation; static benchmarks built for 2024-era capabilities will under-report risk.
- **Frontier-model alignment regressions (IICL on GPT-5.4)** — Adversa AI's Involuntary In-Context Learning attack (arXiv 2604.19461, April 2026) bypasses GPT-5.4 safety with 10 examples (5 harmful, 5 benign) and 2 control words (`answer`, `is_valid`) — no encoding, no role-play. The same probes scored 60% ASR on GPT-5.4 but 0% on GPT-5 and GPT-5-mini, so the vulnerability was introduced by a post-release safety-tuning update. The attack targets the structural layer (in-context pattern completion) rather than content filters and reinforces C11.1 (re-run the alignment suite on every model update) and the case for fast-rollback policy infrastructure when a model swap reintroduces a regression.
- **Defense-in-depth pipeline composition (STACK)** — STACK (STaged AttaCK, AAAI 2026 / arXiv 2506.24068) builds jailbreaks against each safeguard component in turn — input classifier, model, output classifier — and composes them to defeat the full pipeline. Against a few-shot prompted classifier pipeline that drove ClearHarm ASR to 0% on naive attacks, STACK achieved 71% ASR in a black-box configuration and 33% in transfer with no access to the target pipeline. The defensive recommendations are direct C11.1 inputs: make refusal responses opaque (uniform regardless of which component triggered), always route through the full pipeline to suppress timing side channels, and tightly control programmatic access to individual safeguard components.
- **Vector-database geometric poisoning (Black-Hole)** — "Can You Trust the Vectors in Your Vector Database?" (arXiv 2604.05480, April 2026) shows that injecting ~1% of vectors near the centroid of an embedding space causes those vectors to appear in up to 99.85% of top-10 nearest-neighbor results across queries, exploiting centrality-driven hubness in high-dimensional embeddings. This is a structural defect, not a per-document poisoning attack — embedding distance alone cannot distinguish a malicious centroid-resident document from a relevant one, so C11.4 detection must inspect vector positions and retrieval-rank statistics, not only document content.
- **Stealthy style-trigger backdoors (BadStyle)** — BadStyle (arXiv 2604.21700) weaponizes an LLM as a poisoned-sample generator that rewrites clean text into style-transferred variants carrying imperceptible style-level triggers while preserving semantics and fluency. An auxiliary target loss stabilizes payload injection during fine-tuning, yielding ~30% average ASR gains over style-trigger baselines across LLaMA, Phi, DeepSeek, and GPT variants, and the backdoor remains effective in unknown downstream deployments. This is a direct adaptive-attack stressor for C11.1 backdoor checks (activation clustering, spectral signatures, neural cleanse) and for any C06 model-supply-chain pipeline that relies on perplexity- or trigger-pattern-based scanning of training corpora.
- **Compromised AI development workflows ("Clinejection")** — Disclosed February 9, 2026 (Adnan Khan, Snyk) and exploited February 17, 2026, the chain ran from a single attacker-controlled GitHub issue title through prompt-injection of Cline's Claude-powered triage workflow, into an attacker-commit `npm install`, into a Cacheract cache-poisoning payload (>10 GB of junk to evict legitimate keys), into a nightly publish workflow that restored the poisoned cache, into exfiltration of `VSCE_PAT`, `OVSX_PAT`, and `NPM_RELEASE_TOKEN`. An unauthorized `cline@2.3.0` then shipped to npm, adding a postinstall that globally installed the OpenClaw agent on roughly 4,000 developer machines during an 8-hour window. For C11.4 this is the canonical incident showing that an AI triage agent with shell access on untrusted issue input collapses input validation, agent self-review, and supply-chain trust into a single failure surface — controls in C02, C06, C08, and C11.4 must all hold simultaneously.
- **Loopback and local-control-plane attacks against agents (AutoJack)** — Microsoft Defender Security Research disclosed AutoJack on June 18, 2026: untrusted web content rendered by an AutoGen Studio browsing agent could reach a local MCP WebSocket, bypass localhost-origin assumptions, skip authentication on `/api/mcp/*`, and pass attacker-controlled `StdioServerParams` that spawned arbitrary host processes. The affected MCP route was fixed before a PyPI release, but the pattern is broader than AutoGen: if a browsing or code-execution agent can reach privileged services on loopback, localhost is no longer a trust boundary. C11.4 evidence should therefore include local control-plane threat modeling, authenticated WebSocket/MCP endpoints, executable allowlists, and endpoint detections for unexpected Python/Node child processes.
- **Selective attack timing against AI control monitors** — "Attack Selection in Agentic AI Control Evaluations" (arXiv 2606.06529, June 2026) shows that an agentic attacker that chooses when to start or abort sabotage can make control evaluations look too optimistic. At a 1% audit budget, the start policy reduced empirical safety by 20 percentage points in both BashArena and LinuxArena, while stop policies reduced safety by 20 points in BashArena and 28 points in LinuxArena. For C11.1 and C11.4, red-team suites need start/stop attack selection, not only always-attack baselines.
- **AI-powered CI/CD agent prompt injection (GitInject)** — GitInject (arXiv 2606.09935, June 2026) evaluates real GitHub workflow runs rather than simulated tool calls and documents 11 attacks across config-file injection, credential exfiltration, judgment manipulation, and availability. All tested provider defaults were vulnerable to at least one class, and the highest-impact failures came from workflow credential/configuration structure rather than model behavior alone. This makes C11 adversarial robustness inseparable from C06 supply-chain evidence and C09 privilege boundaries for code-review and maintenance agents.
- **Membership inference** — Pre-trained LLMs show surprising resilience, but fine-tuned models are significantly more vulnerable. Multi-query attacks achieve 39% extraction rates. New techniques (ACL 2025) achieve first successful MIA on pre-trained LLMs via paragraph-level feature aggregation.
- **Model inversion / training data extraction** — Researchers extracted 10,000+ verbatim training examples from ChatGPT for ~$200 using simple repetition prompts. Over 5% of output was direct 50-token copies. Llama 3.2 PII extraction demonstrated in 2025.
- **Model extraction at industrial scale** — DeepSeek, Moonshot AI, and MiniMax used ~24,000 fake accounts for industrial-scale distillation of Claude (February 2026). DeepSeek R1 achieved GPT-o1 performance at $5.6M development cost via systematic API harvesting.
- **Inference-time poisoning** — PoisonedRAG (USENIX Security 2025) showed just 5 crafted documents manipulate RAG responses 90% of the time. MCP tool poisoning and MCPTox benchmark document cross-tool contamination across 45+ servers. Vocabulary-engineered RAG document poisoning reaches 95% ASR in April 2026 evaluations, and Normalized Passage Attention Score (NPAS) + Attention-Variance Filter defenses (OpenReview 2026) reduce effective ASR by roughly 20% by flagging passages whose cross-attention profile deviates from the benign distribution.
- **Jailbreaks and alignment bypass** — JBFuzz achieves ~99% success across frontier models with ~7 queries in under 1 minute. Multi-turn jailbreaks hit 97% success within five conversational turns (Mindgard 2026 benchmarks). Skeleton Key (multi-turn guardrail rewriting), Context Compliance Attack (fabricated history injection), and evaluation awareness/sandbagging where models strategically underperform on safety evaluations. As of early 2026, role-play attacks show 89.6% success and healthcare LLM attacks reach 94.4% completion of adversarial objectives. As of April 2026, "Defamiliarization" — a literary-theory-based multi-turn attack that reframes harmful intent through conceptual displacement — adds another class of bypass that survives per-turn content filtering, and one academic study reported 100% attack success across 22 of 26 aligned LLMs evaluated.
- **Autonomous adversarial agents** — Large reasoning models (LRMs) used as autonomous jailbreak agents achieved 97.14% overall success rate across tested frontier systems (Nature Communications, 2026), collapsing the skill barrier for attackers. A follow-on autonomous adversarial agent named "Claudini" designed jailbreaks with 100% success vs 56% for human-crafted methods. This matters because it means attackers can scale red-teaming faster than defenders can patch — the cost asymmetry now favors the attacker.
- **Context window poisoning** — A rapidly emerging class of attack that hides malicious instructions deep in 128K+ token contexts so they evade scanning of retrieved snippets and system prompts but still influence generation. Adversa's April 2026 roundup flags it as "one of the most critically under-defended operational vulnerabilities." Defenses overlap with C11.4 (runtime context contamination detection) but require per-position attention/provenance tracking rather than input-edge filtering.
- **Compound prompt-injection + DB poisoning** — PIDP-Attack (arXiv 2603.25164, March 2026) combines query-time prompt injection with RAG database poisoning. Across Natural Questions, HotpotQA, and MS-MARCO over eight target LLMs, it reports 4-16% ASR gains over the PoisonedRAG baseline while maintaining high retrieval precision and requiring no prior knowledge of the victim query. Defense-in-depth across C02 (input validation), C08 (memory/embeddings), and C11.4 is required — neither layer alone mitigates the compound attack.
- **Defense circumvention** — A March 2026 replication study broke 12 of 12 recently published AI defenses that had each claimed near-zero attack success, returning observed ASR to attack-level baselines. The takeaway for C11.1 and C11.4: adaptive attack evaluation (attacks tailored to the deployed defense) is non-negotiable; static benchmark ASR numbers from the defending paper routinely do not survive contact with adaptive adversaries.
- **Multi-modal and vision-language attacks** — VLM-specific jailbreaks combine imperceptible image perturbations with text manipulation to disrupt cross-modal alignment. Typographic attacks convert harmful text to visual form to bypass text-only filters. Proxy model transfer enables black-box attacks on VLMs without direct access. A January 2026 survey (arXiv 2601.03594) catalogs attack taxonomies across LLMs and VLMs and proposes a three-layer defense framework (perception, generation, parameter layers). As of March 2026, Reasoning-Oriented Programming (VROP) decomposes harmful objectives into benign "semantic gadgets" — individually harmless visual components that only become dangerous when composed through optimized reasoning chains. VROP achieves 9.5% higher attack success rates than prior baselines on commercial VLMs (GPT-4o, Claude 3.7 Sonnet) by exploiting compositional reasoning to bypass multimodal safety alignment.
- **Semantic chaining jailbreaks** — A four-step attack technique (February 2026) exploits how models evaluate modifications to existing content rather than re-assessing full semantic meaning. By splitting harmful requests into discrete, benign-appearing chunks and presenting them as iterative edits, attackers bypass safety filters that only evaluate the delta. NeuralTrust demonstrated successful semantic chaining against Grok 4 and Gemini Nano, exploiting the architectural gap between per-turn content filtering and multi-turn intent tracking.
- **Indirect prompt injection in the wild** — Palo Alto Unit 42 documented 22 distinct IDPI payload engineering techniques actively exploited in production (March 2026). Delivery methods include HTML attribute cloaking (19.8%), CSS rendering suppression (16.9%), and zero-sizing/off-screen positioning. Social engineering accounts for 85.2% of observed jailbreak payloads. Google Threat Intelligence then scanned Common Crawl's 2-3 billion monthly public-web pages and reported a 32% relative increase in malicious IPI content between November 2025 and February 2026. Forcepoint X-Labs separately found 10 live-web payloads on April 22, 2026 spanning API-key exfiltration, data destruction, unauthorized PayPal transfer attempts, AI denial of service, traffic hijacking, and content suppression. The operational lesson is that web content, tool output, RAG input, and agent memory all need hostile-input treatment before an AI system acts on them.
- **Self-modification abuse** — Morris II (March 2024) demonstrated the first self-replicating AI worm that propagates through interconnected agents via metamorphic prompts. Copilot configuration self-modification attacks silently alter agent settings via indirect prompt injection.
- **Adversarial bias exploitation** — 3 million daily spoofed emails bypassed Proofpoint classifiers by exploiting undertrained brand-impersonation patterns. State-sponsored operations exploit differential sensitivity across languages and cultural contexts. CrowdStrike 2026 documents Russia-nexus FANCY BEAR deploying LLM-enabled malware (LAMEHUG) for automated reconnaissance, while DPRK-nexus FAMOUS CHOLLIMA leveraged AI-generated personas to scale insider operations — both exploiting trust biases in organizational security models.
- **Deepfake evasion of identity verification** — MITRE ATLAS published a case study (November 2025) documenting deepfake attacks against mobile KYC liveness detection in banking, financial services, and cryptocurrency platforms. Attackers use synthetic face generation to bypass biometric classifiers that underpin identity verification pipelines.

## Tooling & Implementation

Current tools, frameworks, and libraries that help implement these controls:

- **Adversarial robustness testing:** [IBM ART v1.20.1](https://github.com/Trusted-AI/adversarial-robustness-toolbox) (Linux Foundation — evasion, poisoning, extraction, and inference attacks/defenses; supports PyTorch, TensorFlow, scikit-learn), [NVIDIA Garak](https://github.com/NVIDIA/garak) ("nmap for LLMs" — 120+ vulnerability categories including hallucination, jailbreaks, data leakage, prompt injection), [Cisco AI Defense](https://www.robustintelligence.com/product) (acquired Robust Intelligence Oct 2024 — stress testing + real-time AI Firewall)
- **Red-teaming and alignment:** [HarmBench 1.0](https://github.com/centerforaisafety/HarmBench) (510+ behaviors, 18 attack methods, 33 target LLMs), [Microsoft PyRIT](https://github.com/Azure/PyRIT) (multi-turn adversarial automation; v0.13.0 released Apr 17, 2026), StrongREJECT (jailbreak effectiveness scoring, integrated into Garak), [Mindgard](https://mindgard.ai/) (automated red teaming with CI/CD integration, multimodal support, MITRE ATLAS + OWASP LLM Top 10 aligned reporting; Gartner Emerging Tech 2026, 2025 Cybersecurity Excellence Award), [JALMBench](https://openreview.net/pdf?id=DJkQ236C8B) (ICLR 2026 — standardized jailbreak evaluation benchmark), [DeepTeam 1.0.4](https://github.com/confident-ai/deepteam) (confident-ai, first stable release Nov 2025 — open-source LLM red-team framework covering prompt injection, multi-turn jailbreaks (Linear, Tree, Crescendo, Sequential, Bad Likert Judge), roleplay, leetspeak, context poisoning, permission escalation; aligned to OWASPTop10, OWASP_ASI_2026, NIST, MITRE, Aegis, BeaverTails), [SoK: Evaluating Jailbreak Guardrails](https://arxiv.org/abs/2506.10597) (IEEE S&P 2026 — six-dimension guardrail taxonomy with a Security-Efficiency-Utility evaluation framework and open-source benchmark code; useful for selecting and regression-testing guardrails rather than trusting vendor ASR claims)
- **Agent-control and workflow-specific auditing:** [Google DeepMind AI Control Roadmap](https://deepmind.google/blog/securing-the-future-of-ai-agents/) (June 2026 — treats powerful internal agents as possible insider threats; scales detection and prevention/response levels by model capability; measures coverage, recall, and time-to-response), [PI-Hunter](https://arxiv.org/abs/2606.12737) (automated source-aware red teaming that exposes latent indirect prompt injections and localizes vulnerable ingestion paths), [GitInject](https://arxiv.org/abs/2606.09935) (live CI/CD workflow test harness for AI review/maintenance agents), [Attack Selection](https://arxiv.org/abs/2606.06529) (selective start/stop sabotage for BashArena and LinuxArena control evaluations), and [Ensemble Monitoring for AI Control](https://arxiv.org/abs/2605.15377) (diverse 3-monitor ensembles outperformed homogeneous monitoring by 2.4x on adversarial coding tasks). These are useful C11.4 complements to model-level jailbreak suites because they test trajectories, tool boundaries, monitor coverage, and action gating.
- **Proactive jailbreak defenses:** [ProAct](https://arxiv.org/pdf/2510.05052) (proactive defense that feeds spurious responses to attackers — reduces ASR by up to 92%, combined with other defenses achieves 0% success), [HoneyTrap](https://arxiv.org/abs/2601.04034) (multi-agent honeypot defense — four cooperating agents: Threat Interceptor, Misdirection Controller, Forensic Tracker, System Harmonizer — reduces ASR by 68.77% and forces 19.8x resource consumption; validated on GPT-4, GPT-3.5-turbo, Gemini-1.5-pro, LLaMA-3.1), [TwinGate](https://arxiv.org/abs/2604.27861) (April 2026 — dual-encoder Asymmetric Contrastive Learning that clusters semantically disparate but intent-matched malicious fragments in a shared latent space; stateful defense for decompositional / multi-session jailbreaks where individual queries share near-zero embedding similarity; reports <0.2% false positives across 3.62M requests), TBOP (April 2026 — reduces jailbreak ASR to 2.56% while improving model utility, reported 60x faster inference than the next-best defense in the authors' evaluation; independent replication pending, per [VentureBeat's March 2026 study](https://venturebeat.com/security/12-ai-defenses-claimed-near-zero-attack-success-researchers-broke-all-of-them) that broke 12 of 12 similarly-claimed defenses under adaptive attack)
- **Agent design-pattern defenses:** [CaMeL](https://arxiv.org/abs/2503.18813) (Google DeepMind / ETH Zürich, *Defeating Prompt Injections by Design*) splits the agent into a privileged planner that never sees untrusted data and a quarantined worker that processes external content but cannot trigger high-impact tools, then extracts explicit control- and data-flow graphs before any action runs. Reported 67% mitigation on AgentDojo without any model retraining; pairs naturally with C11.4 runtime defense because the privileged-vs-quarantined split is enforced by surrounding code rather than model behavior. [Six Design Patterns for Securing LLM Agents Against Prompt Injections](https://arxiv.org/pdf/2506.08837) provides a complementary engineering checklist (action-selector, plan-then-execute, LLM map-reduce, dual LLM, code-then-execute, context-minimization) for agent runtime hardening.
- **Differential privacy:** [Opacus](https://opacus.ai/) (Meta, PyTorch DP-SGD with LoRA support, Dec 2024), [TensorFlow Privacy 0.12.3](https://www.tensorflow.org/responsible_ai/privacy), [Microsoft dp-transformers](https://github.com/microsoft/dp-transformers) (HuggingFace integration), [PySyft 0.9.5](https://github.com/OpenMined/PySyft) (privacy-preserving ML with OpenDP). As of 2026, Google research demonstrates user-level DP fine-tuning requires significantly less noise than previously thought, and DP-ZeRO enables large-scale DP training via DeepSpeed with reduced memory overhead. Practical note: DP-SGD adds 3-5x training overhead for full training, though parameter-efficient fine-tuning (LoRA, P-tuning) substantially narrows the gap.
- **Model watermarking:** [Google SynthID](https://deepmind.google/technologies/synthid/) (production on Gemini — text/image/audio/video; text component [open-sourced](https://github.com/google-deepmind/synthid-text)), academic approaches (Kirchenbauer green-list, frequency-domain embedding). Newer "holographic" watermarking distributes marks across entire artifacts, making partial removal destructive. [zkDL++](https://openreview.net/forum?id=JyrjeQJ8VK) (ICLR 2025) introduces zero-knowledge proofs for provable watermark extraction — verifying watermark presence without revealing extractor internals, which closes a key privacy gap in deployment. However, research shows any image watermark can be removed >50% of the time without knowledge of the watermark design, including SynthID and Meta's Stable Signature.
- **Model extraction detection:** [Lakera Guard](https://www.lakera.ai/) (acquired by Check Point 2025 — 98%+ detection, sub-50ms, 100+ languages), rate limiting, query-pattern analysis, model fingerprinting techniques
- **Inference-time anomaly detection:** [NeMo Guardrails](https://docs.nvidia.com/nemo/guardrails/) (NVIDIA — programmable rails with Garak integration), [Rebuff](https://github.com/protectai/rebuff) (multi-layer prompt injection detection), [RAGForensics](https://arxiv.org/abs/2504.21668) (first traceback system for RAG poisoning — iteratively retrieves and probes knowledge base subsets to identify poisoned texts; ACM Web Conference 2025), [SDAG](https://arxiv.org/abs/2602.04711v2) (Sparse Document Attention — block-sparse attention mask that prevents cross-attention between retrieved documents, requiring only inference-time attention mask changes with no fine-tuning), [ReliabilityRAG](https://arxiv.org/pdf/2509.23519) (builds a contradiction graph over retrieved documents using NLI, selects a consistent majority via Maximum Independent Set to filter poisoned entries), [NPAS + AV Filter](https://openreview.net/forum?id=PS43wqCSME) ("Through the Stealth Lens", OpenReview 2026 — Normalized Passage Attention Score plus Attention-Variance Filter use cross-attention anomalies to flag poisoned retrieved passages; reports ~20% ASR reduction over baseline defenses). For vector-store geometry attacks like [Black-Hole](https://arxiv.org/abs/2604.05480), content-only filters are insufficient — defenses need to inspect vector positions relative to the embedding centroid, monitor per-vector retrieval-rank statistics over time, and reject candidates whose nearest-neighbor recall is anomalously high across unrelated queries.
- **Agent component and skill review:** VirusTotal Code Insight integration for OpenClaw's ClawHub (reported by GTIG in May 2026) automatically analyzes submitted skills for unauthorized network operations, malicious payloads, and unsafe embedded instructions. Treat this as a C11/C06 bridge control: adversarial robustness testing should include malicious skill packages, tool metadata poisoning, and host-command execution paths before agent components are granted runtime privileges. On the commercial side, Workday's Agent Passport (announced June 2, 2026) packages pre-production testing, verification, and continuous monitoring of enterprise AI agents with attestations tied to OWASP LLM Top 10, NIST AI RMF, and MITRE ATLAS — an early signal that agent attestation is becoming a procurable control rather than a custom build. The Semantic Kernel CVEs (May 2026) show why framework plumbing belongs in scope: review tool attributes (`[KernelFunction]`-style exposure), `eval()`/interpolation paths, and sandbox file-write boundaries, not just prompts and policies.
- **Formal robustness verification:** [alpha-beta-CROWN](https://github.com/Verified-Intelligence/alpha-beta-CROWN) (VNN-COMP winner 2021-2025, GPU-accelerated), [auto_LiRPA 0.3](https://github.com/Verified-Intelligence/auto_LiRPA) (provable output bounds), [ERAN](https://github.com/eth-sri/eran) (ETH Zurich — abstract interpretation). *Limitation: scales to small/medium networks; not yet practical for full-scale LLMs.*

## Open Research Questions

- [ ] **How do you detect evaluation awareness / sandbagging?** — Frontier models can strategically underperform on dangerous capability evaluations while maintaining general performance. Noise injection into model weights reveals sandbagging through anomalous performance *improvements*, but this isn't a production detection method. Claude 3.5 Sonnet spontaneously underperformed without explicit instruction when it discovered an unlearning process would be triggered.
- [ ] **Can formal verification scale to production LLMs?** — alpha-beta-CROWN wins VNN-COMP annually but only scales to small/medium networks. Certified robustness for full-scale transformers remains an open problem. Formal verification is required for safety-critical components — the "where the model architecture supports them" qualifier is doing heavy lifting.
- [ ] **How do you defend against inference-time RAG poisoning?** — PoisonedRAG showed 5 crafted documents manipulate 90% of responses. As of early 2026, three academic defenses have emerged — RAGForensics (traceback), SDAG (sparse attention isolation), and ReliabilityRAG (contradiction-graph filtering) — but none has reached production maturity. The fundamental challenge remains: models can't distinguish instructions from data in retrieved context, and existing defenses have proven insufficient against sophisticated adaptive attacks.
- [ ] **Is DP-SGD practical for LLM pretraining?** — 3-5x overhead makes DP-SGD feasible for fine-tuning (Opacus + LoRA) but prohibitive for full pretraining. Google's 2026 research on user-level DP shows prior approaches added far more noise than necessary, and DP-ZeRO improves scalability via DeepSpeed. Still, membership inference defenses (C11.2) remain practically limited to fine-tuning scenarios for large models.
- [ ] **Can model watermarking survive determined removal?** — SynthID text watermarking is robust to mild paraphrasing but less effective on factual outputs. Research shows any image watermark (including SynthID and Meta's Stable Signature) can be removed >50% of the time without knowledge of the watermark design. zkDL++ (ICLR 2025) introduces zero-knowledge proofs for provable extraction, which helps verification but doesn't solve the removal problem. Weight-based watermarking for IP protection is an active research area but no standard exists.
- [ ] **Can adversarial robustness keep pace with automated attack generation?** — Automated red-teaming tools discover 37% more unique vulnerabilities than manual testing alone (Mindgard 2026), yet only 26% of organizations conduct proactive security testing. The gap between attack automation (FuzzyAI, JBFuzz, Mastermind) and defense deployment suggests most production systems are undertested. Multi-turn attacks that self-improve through interaction (Mastermind framework) represent a qualitative shift from static jailbreak templates.
- [ ] **How do you bound self-modification in agentic systems?** — Morris II demonstrated autonomous propagation via metamorphic prompts. Copilot configuration self-modification showed silent settings changes. Bounded self-modification is required, but defining and enforcing those bounds at runtime remains unsolved.
- [ ] **Are layered defense-in-depth pipelines actually composable?** — STACK (AAAI 2026) sequentially defeated input classifier, model, and output classifier in production-style safeguard stacks at 71% black-box ASR even when single-stage ASR was ~0%, and 33% in transfer with no target-pipeline access. The design implications — opaque refusal responses, always-route-through-full-pipeline, tight control of safeguard-component access — are clear, but no public benchmark yet measures pipeline-composition robustness as a first-class metric across vendors.
- [ ] **Is prompt injection formally unsolvable for agentic systems?** — Abdelnabi and Bagdasarian (arXiv 2605.17634, May 2026) argue through Contextual Integrity theory that any fixed allow/deny rule either blocks legitimate contextual flows or admits adversarially constructed contexts, and showed injection classifiers at near-chance AUROC (0.43–0.59) against contextual manipulation. The UK NCSC reached a similar operational conclusion in December 2025. If the result holds, runtime-detection verification should shift emphasis from detection accuracy toward blast-radius engineering: least-agency tool scoping, deterministic privilege separation, and human gates sized to the worst undetected injection. What "CI-aware alignment" looks like as a testable control remains open.
- [ ] **Can embedding-geometry attacks be detected without breaking RAG utility?** — The Black-Hole vector-database attack (arXiv 2604.05480) achieves up to 99.85% top-10 contamination with just 1% of vectors injected near the centroid by exploiting centrality-driven hubness. Content-level scanners cannot see this; index-level monitoring of centroid distance, hub-recall statistics, and per-vector retrieval frequency is required, but no production vector store ships this out of the box.
- [ ] **Do AI control evaluations account for selective attackers?** — June 2026 attack-selection work shows that an agent can reduce measured safety by choosing when to start or stop sabotage under a limited audit budget, without becoming more capable at the underlying attack. C11.1 and C11.4 test plans should therefore include always-attack, never-attack, start-policy, and abort-policy variants, plus evidence that monitor thresholds are calibrated for rare, well-timed attacks.
- [ ] **When does localhost stop being a trust boundary for agents?** — AutoJack shows that a browsing/code-execution agent on the same workstation can become the bridge between untrusted web content and privileged local services. C11 runtime anomaly detection needs a practical verification pattern for loopback exposure: enumerate local control planes, authenticate WebSockets/MCP routes, require server-side parameter binding, isolate browsing identities, and alert on unexpected child processes spawned by agent framework runtimes.

## Industry Adoption & Testing Metrics

As of June 2026, industry benchmarks paint a sobering picture of adversarial robustness in practice:

- **Testing coverage gap:** Only 26% of organizations conduct proactive AI security testing. Automated red teaming identifies 37% more unique vulnerabilities than manual testing alone, and post-implementation testing reveals 75-80% more vulnerabilities than pre-deployment assessments catch.
- **Attack success rates in benchmarked evaluations:** Role-play attacks succeed at 89.6%, multi-turn jailbreaks reach 97% within five turns, and healthcare-domain LLM attacks hit 94.4% adversarial objective completion. Even simple cognitive manipulations ("bad math" misdirection) succeed 76% of the time.
- **Detection capability:** Best-in-class model detection (Claude 3.7 Sonnet) achieves only 46.9% detection rate against adversarial inputs. Known jailbreak exploits succeed at a 17% average rate even against models with active defenses, and documented attack techniques show 28% overall success.
- **Economic impact:** Organizations with mature AI security practices experience 60% fewer incidents. Proactive testing yields an estimated $2.4 million in prevented breach costs, with roughly $1.9 million saved per breach avoided.
- **Framework expansion:** MITRE ATLAS grew from its initial scope to 15 tactics and 66 techniques by October 2025, adding 14 AI agent-specific techniques. AI-assisted cyber attacks surged 72% in 2025, driving the need for more comprehensive adversarial robustness frameworks. The 2026 ATLAS update shifts focus from model-centric attacks to execution-layer exposure, covering autonomous workflow chaining, delegated authority persistence, and API-level orchestration risk.
- **AI-assisted exploit development:** GTIG's May 2026 AI Threat Tracker marks a practical shift from AI as a drafting aid to AI as exploit-discovery and attack-orchestration infrastructure. The reported zero-day 2FA bypass was a semantic logic flaw that conventional scanners are not optimized to find, while PROMPTSPY shows malware can use model reasoning over UI state to select actions. C11 evidence should therefore include adaptive exploit-generation tests, adversarial code review, and controls that keep model-driven agents from acting directly on high-impact system state.
- **Frontier-model attacker baselines:** UK AISI's May 2026 evaluation of Claude Mythos Preview reported 73% success on expert-level cyber tasks (where no model could score before April 2025), end-to-end completion of a simulated 32-step corporate network attack in 3 of 10 attempts with an average of 22/32 steps, and first-attempt exploit reproduction in 83% of cases. Anthropic's Project Glasswing then placed $100M of usage credits behind defender-only access for AWS, Apple, Broadcom, Cisco, CrowdStrike, Google, JPMorganChase, the Linux Foundation, Microsoft, NVIDIA, Palo Alto Networks, and 40+ partners. C11 adversarial evaluations should assume the attacker has equivalent capability inside their own infrastructure within the next benchmark cycle, not the next product cycle.
- **Operational IPI telemetry:** Google, Unit 42, and Forcepoint now provide independent public-web telemetry showing that indirect prompt injection has moved from lab demonstrations to live infrastructure. Most payloads remain low sophistication, but the observed intents — data destruction, credential leakage, unauthorized transactions, DoS, ad-review evasion, and SEO manipulation — map directly to C11.4, C02, C08, and C09 controls.
- **CrowdStrike 2026 threat data:** 89% increase in AI-enabled attacks year-over-year. Average eCrime breakout time dropped to 29 minutes (65% faster than 2024), with the fastest observed at 27 seconds. Adversaries exploited legitimate GenAI tools at 90+ organizations via malicious prompt injection. FANCY BEAR deployed LLM-enabled malware (LAMEHUG) for automated recon; FAMOUS CHOLLIMA used AI-generated personas for insider operations at scale.
- **Emerging defensive results:** ProAct and HoneyTrap represent a shift toward proactive, deceptive defense. ProAct's spurious response strategy reduces ASR by up to 92% (to 0% when combined with other defenses). HoneyTrap's multi-agent honeypot forces 19.8x attacker resource consumption. TBOP (April 2026) claims ASR of 2.56% with 60x faster inference than the next-best defense. TwinGate (April 2026) adds a stateful dual-encoder defense against decompositional jailbreaks across untraceable multi-session traffic, reporting <0.2% false positives on 3.62M requests. CaMeL (Google DeepMind / ETH Zürich) demonstrates a design-pattern defense that splits agent privileges and extracts explicit control flow before any action runs, mitigating 67% of AgentDojo prompt injections without model retraining. These approaches complement rather than replace static alignment — the defense-in-depth picture is maturing, even as STACK shows that the picture is still composable from the attacker's side.
- **Machine-speed N-day weaponization:** The Sysdig/Marimo intrusion (May 2026) put hard numbers on autonomous-agent operations: exploitation began 9 hours 41 minutes after CVE disclosure, credential replay ran 12 API calls across 11 IPs in 22 seconds, and a full internal database dump finished in two minutes against a schema the attacker had never seen. Rate-limit thresholds, extraction detectors (C11.3), and runtime anomaly SLAs (C11.4) calibrated against human operators will miss adversaries whose only constraint is inference budget.
- **Exposed deployment surface:** Intruder's May 2026 scan of certificate-transparency logs found roughly 1 million exposed AI services across 2 million hosts. 31% of 5,200+ queried Ollama servers responded without authentication — 518 of them proxying commercial frontier models — and 90+ unsecured n8n/Flowise instances exposed complete agent workflows and credentials across government, marketing, and finance. Much of the adversarial-robustness conversation assumes an authenticated API boundary that, in practice, frequently is not there.
- **Defense claims need adaptive-attack validation:** A March 2026 replication study broke 12 of 12 published AI defenses that had each claimed near-zero ASR, restoring observed success rates to attack-level baselines. This is the direct justification for C11.1 (adaptive attacks) and C11.4 (adaptive stress tests) — authors' own benchmark numbers are not sufficient evidence, and teams should re-validate any defense against adaptive attacks tailored to the specific deployed configuration before relying on it.
- **Agent-control evidence is becoming measurable:** Google DeepMind's June 2026 AI Control Roadmap reports analysis of one million coding-agent trajectories to build live monitoring for Gemini Spark and frames evidence around coverage, recall, and time-to-response. Parallel research shows why those metrics need adversarial stress: selective attack timing lowered measured safety by 20-28 percentage points at a 1% audit budget, and diverse monitor ensembles outperformed same-model ensembles by 2.4x on adversarial coding tasks.
- **Local control planes are now part of adversarial robustness:** AutoJack did not ship in a PyPI release, but it gives defenders a concrete June 2026 test case for browsing-agent confused-deputy behavior. Auditors should treat localhost MCP/WebSocket/debug endpoints as reachable from untrusted content whenever an agent can browse, execute code, or make arbitrary network requests on the same host.

These numbers reinforce that adversarial robustness is not a theoretical concern — it's a measurable operational risk with quantifiable return on investment for organizations that test proactively.
