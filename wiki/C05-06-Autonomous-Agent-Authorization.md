# C5.6: Autonomous Agent Authorization

> [Back to C05 Index](C05-Access-Control.md)
> **Last Researched:** 2026-03-29

## Purpose

Control permissions for AI agents and autonomous systems through scoped capability tokens and continuous authorization. Autonomous agents introduce a fundamentally different authorization model: unlike human users who authenticate once per session, agents may execute hundreds of actions requiring per-action or per-batch authorization. Each agent inherits every credential, permission, and delegated session it receives, making it a single point of high-value access. The OWASP Top 10 for Agentic Applications (2026) ranks Identity & Privilege Abuse as a top-3 risk and introduces the principle of "Least Agency" -- autonomy should be earned, not a default setting. Real-world incidents in 2025-2026 (OpenClaw credential theft at scale, PerplexedBrowser zero-click compromise, 88% of organizations reporting agent security incidents) demonstrate that agent authorization is no longer a theoretical concern.

> **Scope note:** Controls in this section govern capability token scoping, default-deny for high-risk capabilities, session binding, and continuous per-action authorization for autonomous agents. Identity and authentication for agents is in C5.1; authorization policy engines and externalized PDPs are in C5.2.

## Requirements

| # | Requirement | Level | Threat Mitigated | Verification Approach | Gaps / Notes |
|---|-------------|:-----:|-----------------|----------------------|--------------|
| **5.6.1** | **Verify that** autonomous agents receive scoped capability tokens that explicitly enumerate permitted actions, accessible resources, time boundaries, and operational constraints. | 1 | Overprivileged agents performing unintended actions; prompt injection causing agents to exceed intended scope; agents accumulating permissions across task boundaries. | Inspect token claims for action enumeration, resource scoping, and time boundaries; attempt out-of-scope actions and confirm denial; verify tokens cannot be escalated by the agent itself. | Principle of least privilege applied to autonomous agents. Capability-based security (as in object-capability model) is well-suited. Token format should enumerate positive permissions rather than relying on deny-lists. Frameworks like AutoGPT and CrewAI are beginning to add permission scoping. **[2026-03]** Grantex implements scoped JWT grant tokens with per-agent DIDs, granular revocation with cascade, and delegation chains with depth limits and scope narrowing. NIST's Feb 2026 concept paper explicitly calls for task-scoped privileges and action-level approvals for high-impact agent decisions. 93% of audited popular AI agent projects still use unscoped API keys (Grantex 2026). The OWASP Top 10 for Agentic Applications (2026) introduces the principle of "Least Agency" -- autonomy should be earned, not a default setting -- and ranks Identity & Privilege Abuse (ASI03) as a top-3 risk, noting that an agent's identity includes every credential, permission, and delegated session it inherits, making it a single point of high-value access. |
| **5.6.2** | **Verify that** high-risk capabilities (file system access, code execution, external API calls, financial transactions) are disabled by default and require explicit authorization. | 1 | Prompt injection or jailbreaking causing agents to execute arbitrary code, access the file system, or make external network calls; agents performing destructive actions without human approval. | Attempt high-risk actions with default agent configuration and confirm denial; verify each high-risk capability requires explicit opt-in; test that enabling one capability does not implicitly enable others. | Critical for agent safety. Default-deny for dangerous capabilities prevents prompt injection from escalating to system compromise. OpenAI function calling, LangChain tool permissions, and Anthropic tool use all support capability restriction but default configurations vary. **[2026-03]** OWASP identifies "Excessive Agency" as a top-10 LLM/AI agent risk; the new OWASP Top 10 for Agentic Applications (2026) ranks Tool Misuse & Exploitation (ASI02) as the #2 risk, where agents misuse legitimate tools due to prompt injection, misalignment, or unsafe delegation. ISACA (2025) warns that each agent spawns multiple identities, exponentially expanding the attack surface; revocation in one place does not automatically cut off access elsewhere. 48% of cybersecurity professionals identify agentic AI as the number-one attack vector heading into 2026. Indirect prompt injection attacks have achieved >90% success rates in controlled evaluations when agents have unrestricted capabilities. CVE-2026-25253 (OpenClaw, CVSS 8.8, Feb 2026) demonstrated real-world impact: a crafted URL stole authentication tokens from 17,500+ exposed AI agent gateway instances, granting access to stored credentials for Claude, OpenAI, and Google AI services. |
| **5.6.3** | **Verify that** capability tokens are bound to user sessions, include cryptographic integrity protection, and cannot be persisted or reused across sessions. | 2 | Token theft enabling unauthorized agent actions; agents persisting tokens to maintain access beyond the user's session; replay attacks using captured capability tokens. | Verify token-session binding (e.g., session ID in token claims); attempt token reuse after session termination and confirm rejection; test token integrity by modifying claims and verifying rejection; confirm tokens are not written to persistent storage. | Session-bound tokens prevent agents from accumulating long-lived credentials. HMAC or digital signature integrity protection prevents token tampering. Storage restrictions must be enforced at the runtime level since agents with file system access could persist tokens if not prevented. **[2026-03]** OAuth 2.0 on-behalf-of token exchange (ScaleKit) cryptographically binds the agent (actor) to the delegator, preserving the chain of responsibility. ARIA model records every delegation as a distinct, cryptographically verifiable relationship in a graph, enabling cascade revocation. The 2025 Salesloft-Drift breach (stolen OAuth tokens exposed 700+ organizations) demonstrates the risk of tokens that lack session binding and cascade revocation. |
| **5.6.4** | **Verify that** agent-initiated actions undergo authorization through a policy decision point that evaluates contextual attributes (e.g., user identity, resource sensitivity, action type, environmental context). | 3 | Agents bypassing authorization by acting autonomously without per-action policy checks; contextual risks (e.g., production vs. staging environment, business hours vs. off-hours) not factored into agent authorization decisions. | Verify each agent action triggers a PDP evaluation; test with varying contextual attributes (environment, time, resource sensitivity) and confirm decisions change appropriately; audit PDP logs for completeness of agent action authorization. | Continuous authorization for agents is an emerging pattern. Unlike human users who authenticate once per session, agents may execute hundreds of actions requiring per-action or per-batch authorization. Performance optimization (batched policy evaluation, pre-computed permission sets) is needed to avoid latency bottlenecks. **[2026-03]** ISACA identifies seven critical IAM gaps for agentic AI including: insufficient runtime context evaluation, interagent trust gaps (no peer-to-peer trust establishment across org boundaries), and revocation complexity where revoking access in one place does not propagate. Zero Trust Identity approach uses decentralized identifiers, verifiable credentials, and zero-knowledge proofs for continuous verification. Model Context Protocol (MCP) is emerging as a channel for agents to negotiate and verify authorization context. A key risk documented in early 2026: when agent-initiated actions are evaluated against the agent's identity rather than the requester's, users with limited access can indirectly trigger unauthorized actions. 88% of organizations reported confirmed or suspected AI agent security incidents in the past year (Gravitee State of AI Agent Security 2026). The IETF Secure Intent Protocol (`draft-goswami-agentic-jwt-00`) proposes JWT-compatible agentic identity and workflow management for continuous authorization chains. |
